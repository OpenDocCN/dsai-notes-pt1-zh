## 安装Python

首先，我们需要安装Python本身。我们可以通过在终端输入 `python --version` 来检查是否已安装Python。在macOS、Linux和Windows上，命令类似。

![](img/27d9e63635c240722240233451dbd5dd_19.png)

![](img/27d9e63635c240722240233451dbd5dd_20.png)

如果显示“Python not found”，则意味着没有以这种方式设置好Python命令。我通常还会尝试输入 `python3`。在我的系统上，这显示了一个Python 3.9版本，这比当前最新的Python 3.13落后了四个版本。这表明这是操作系统自带的旧版本Python。

**强烈建议不要修改或直接使用这个系统自带的Python版本**，因为这可能会破坏系统功能。我建议安装一个独立的Python版本。

![](img/27d9e63635c240722240233451dbd5dd_22.png)

![](img/27d9e63635c240722240233451dbd5dd_23.png)

安装Python有多种方式。官方方法是访问 `python.org` 下载。但请注意，我建议安装一个稍旧版本的Python（例如Python 3.11或3.12）。这是因为像PyTorch这样的库（我们将在本书中使用）并不总是立即支持最新的Python版本。通常需要几个月时间。为了稳妥起见，使用落后一两个版本的Python是安全的。本书的代码在Python 3.10、3.11、3.12和3.13上都能正常运行。

![](img/27d9e63635c240722240233451dbd5dd_25.png)

![](img/27d9e63635c240722240233451dbd5dd_27.png)

![](img/27d9e63635c240722240233451dbd5dd_28.png)

我个人在这台电脑上使用 **Homebrew** 包管理器来安装Python。你可以在终端输入类似 `brew install python@3.11` 的命令。这只是个人偏好，你也可以直接从官网下载安装。

![](img/27d9e63635c240722240233451dbd5dd_29.png)

## 安装包管理器 UV

![](img/27d9e63635c240722240233451dbd5dd_31.png)

![](img/27d9e63635c240722240233451dbd5dd_32.png)

![](img/27d9e63635c240722240233451dbd5dd_34.png)

下一步是设置包管理器 **UV**。UV是一个比原生Pip更快、更便捷的包管理器。Python社区对此非常兴奋，许多人都在使用它。

![](img/27d9e63635c240722240233451dbd5dd_36.png)

你可以通过Pip来安装UV，命令是 `pip install uv`。这意味着我们使用Python的原生包管理器（Pip）来安装另一个更快的包管理器（UV）。

安装完成后，你可以使用 `uv pip` 接口，它的语法与传统的 `pip` 非常相似，对Pip用户来说更熟悉。如果你愿意探索，也可以使用UV的原生语法。

![](img/27d9e63635c240722240233451dbd5dd_38.png)

## 创建虚拟环境

![](img/27d9e63635c240722240233451dbd5dd_40.png)

![](img/27d9e63635c240722240233451dbd5dd_42.png)

虽然并非绝对必要，但我**强烈推荐创建虚拟环境**。虚拟环境是你的计算机上一个独立的文件夹，所有包都安装在其中，并且它自带一个Python版本。这意味着如果你不小心搞乱了环境，可以直接删除这个文件夹重新开始，而不会影响整个计算机。

![](img/27d9e63635c240722240233451dbd5dd_44.png)

![](img/27d9e63635c240722240233451dbd5dd_46.png)

![](img/27d9e63635c240722240233451dbd5dd_48.png)

![](img/27d9e63635c240722240233451dbd5dd_49.png)

在创建虚拟环境之前，让我们先下载本书的代码。你可以访问GitHub仓库，点击“Download ZIP”将代码下载到本地。为了演示方便，我将下载的文件夹移动到桌面。

![](img/27d9e63635c240722240233451dbd5dd_51.png)

![](img/27d9e63635c240722240233451dbd5dd_53.png)

![](img/27d9e63635c240722240233451dbd5dd_55.png)

现在，让我们在代码目录中创建虚拟环境。命令是 `uv venv --python 3.11`。这将在当前目录下创建一个名为 `.venv` 的文件夹（在macOS/Linux上，以点开头的文件夹默认是隐藏的）。

![](img/27d9e63635c240722240233451dbd5dd_57.png)

创建虚拟环境只需做一次（除非你删除它）。但每次要使用它时，都需要激活它。激活命令是 `source .venv/bin/activate`（在Windows上是 `.venv\Scripts\activate`）。

![](img/27d9e63635c240722240233451dbd5dd_59.png)

![](img/27d9e63635c240722240233451dbd5dd_60.png)

激活后，终端提示符通常会变化。此时输入 `python` 命令，使用的就是虚拟环境中的Python版本了。你可以通过 `which python`（在Windows上是 `where python`）来确认。

![](img/27d9e63635c240722240233451dbd5dd_62.png)

![](img/27d9e63635c240722240233451dbd5dd_63.png)

## 安装依赖包

![](img/27d9e63635c240722240233451dbd5dd_65.png)

![](img/27d9e63635c240722240233451dbd5dd_66.png)

在激活的虚拟环境中，你可以使用 `uv pip install` 命令来安装包。例如，安装PyTorch的命令是 `uv pip install torch`（注意PyTorch的包名是`torch`，而不是`pytorch`）。

本书各章节通常会告诉你需要安装哪些包。但为了更方便，你可以一次性安装所有必需的包。在代码仓库的根目录下，有一个 `requirements.txt` 文件，它列出了运行本书所需的所有包。

你可以使用以下命令一次性安装所有依赖：
```bash
uv pip install -r requirements.txt
```

![](img/27d9e63635c240722240233451dbd5dd_68.png)

![](img/27d9e63635c240722240233451dbd5dd_69.png)

![](img/27d9e63635c240722240233451dbd5dd_71.png)

### Windows 用户的注意事项

在Windows上安装时，你可能会遇到与 `tensorflow-io-gcs-filesystem` 相关的问题。我们需要这个包是因为OpenAI发布的GPT-2模型权重最初是用TensorFlow格式保存的。虽然本书使用PyTorch，但为了加载这些权重文件，需要短暂使用TensorFlow。

如果遇到问题，你有几个选择：
1.  编辑 `requirements.txt` 文件，暂时删除 `tensorflow` 这一行，先安装其他包，稍后再单独处理TensorFlow。
2.  尝试不使用UV，直接用原生Pip安装：`pip install -r requirements.txt`。
3.  使用备份方案：**Google Colab**。

## 在 Google Colab 中运行

![](img/27d9e63635c240722240233451dbd5dd_76.png)

![](img/27d9e63635c240722240233451dbd5dd_78.png)

如果你在本地设置遇到困难，Google Colab 是一个极佳的免费替代方案。它是一个云端的Jupyter Notebook环境。

在Colab的新笔记本中，你可以执行以下步骤：
1.  安装UV：`!pip install uv`
2.  直接从GitHub仓库的原始链接安装所有依赖：
    ```bash
    !uv pip install -r https://raw.githubusercontent.com/rasbt/LLMs-from-scratch/main/requirements.txt
    ```
    在Colab中，由于每次会话本身就是一个干净的虚拟环境，因此不需要再单独创建虚拟环境。

安装完成后，你可以使用 `!uv run jupyter lab` 在Colab中启动Jupyter Lab，然后创建新的Notebook或打开已有的章节代码文件。

![](img/27d9e63635c240722240233451dbd5dd_80.png)

**学习建议**：为了获得最佳学习效果，我强烈建议你**新建一个Notebook并亲手输入书中的代码**，而不是直接打开已有输出的Notebook。这能让你更仔细地观察和理解每一行代码。如果你打开了已有的Notebook，可以先选择“清除所有单元格的输出”，以避免剧透。
