# 4：从零实现GPT模型以生成文本 🧠

在本章中，我们将最终实现GPT模型架构，该架构将在后续章节中进行预训练和微调。本章的重点是编写我们将要使用的LLM架构代码。

## 概述

在之前的章节中，我们已经实现了数据准备和采样，将词元ID嵌入到多维嵌入空间。在上一章中，我们实现了注意力机制，它是LLM进行核心计算的“引擎”。现在，我们将完成整个架构的搭建。

如果把注意力机制比作汽车的发动机，那么本章的任务就是装上所有剩余的部件：车轮、方向盘以及发动机周围的一切，比如座椅等。

我们将以循序渐进的方式完成这个任务，因为实现整个LLM架构是一个庞大的主题。理解架构的最佳方式是先展示一个“虚拟”类，其中包含一些占位符，我们稍后将用实际代码替换它们。

上一节我们介绍了本章的目标，本节中我们来看看GPT模型的整体架构。

以下是GPT模型的“虚拟”类，它展示了架构的主要组成部分：

```python
class DummyGPTModel(nn.Module):
    def __init__(self, cfg):
        super().__init__()
        self.tok_emb = nn.Embedding(cfg["vocab_size"], cfg["emb_dim"])
        self.pos_emb = nn.Embedding(cfg["context_length"], cfg["emb_dim"])
        self.drop_emb = nn.Dropout(cfg["drop_rate"])
        self.trf_blocks = nn.Sequential(
            *[DummyTransformerBlock(cfg) for _ in range(cfg["n_layers"])]
        )
        self.final_norm = DummyLayerNorm(cfg["emb_dim"])
        self.out_head = nn.Linear(cfg["emb_dim"], cfg["vocab_size"], bias=False)

    def forward(self, in_idx):
        batch_size, seq_len = in_idx.shape
        tok_embeds = self.tok_emb(in_idx)
        pos_embeds = self.pos_emb(torch.arange(seq_len, device=in_idx.device))
        x = tok_embeds + pos_embeds
        x = self.drop_emb(x)
        x = self.trf_blocks(x)
        x = self.final_norm(x)
        logits = self.out_head(x)
        return logits
```

![](img/2b7ae7eb4d07123e4c0e4529c48bfd6d_8.png)

这个模型接收经过词元化的文本，然后经过嵌入层（词元嵌入和位置嵌入），这些层我们在第2章已经编写过，但技术上它们是模型的一部分。嵌入层的输入大小是词汇表大小（训练语料库中所有唯一词的数量），位置嵌入的输入大小是上下文长度（一个上下文中可以容纳的词元数量）。

![](img/2b7ae7eb4d07123e4c0e4529c48bfd6d_10.png)

接下来，我们将添加更多组件，其中核心之一是Transformer块。一个LLM由多个这样的Transformer块组成，每个块包含多个元素，我们将在本章后面讨论。其中一个核心元素是我们在上一章实现的掩码多头注意力模块。最后，我们会添加一些输出层。这就是LLM架构在概念上的样子。

回到我们的虚拟GPT模型，它编码了图中展示的内容：我们有嵌入层、可选的Dropout，然后是Transformer块。Transformer块会重复多次，重复次数由配置中的`n_layers`指定。

以下是配置字典的示例：

```python
model_config = {
    "vocab_size": 50257,      # 输入/训练集中的唯一词元数量
    "context_length": 1024,   # LLM支持的上下文长度
    "emb_dim": 768,           # 词元嵌入的维度
    "n_heads": 12,            # 多头注意力模块中的头数
    "n_layers": 12,           # 要重复的Transformer块数量
    "drop_rate": 0.1,         # Dropout率
    "qkv_bias": False         # 是否在查询、键、值线性层中使用偏置
}
```

我们有这些Transformer块，然后是一个称为层归一化（LayerNorm）的组件，最后是输出层。有趣的是，输入层将词汇表大小映射到嵌入空间，而输出层则从嵌入空间映射回词汇表大小。这使得模型能够生成文本，因为我们是将词元ID（单词）转换为中间表示（嵌入空间，即向量表示），然后再转换回原始的词汇表空间（本质上是词元ID），这样LLM就能生成文本了。

在`forward`方法中，我们使用这些元素：创建嵌入，可选地添加Dropout，然后应用Transformer块（得益于`nn.Sequential`，它会顺序通过所有不同的Transformer块），最后是输出层。

如果我们将此视为我们的LLM，它看起来非常简单，因为我们使用了占位符。为了让代码在填充实际代码之前能够运行，我们也有占位符代码。目前，我们称之为虚拟Transformer块，它实际上什么都不做，只是返回输入。稍后，我们将用实际代码填充这个块，解释Transformer块的具体功能。正如你已经看到的，Transformer块的一部分是掩码多头注意力模块，我们稍后会添加它，以及其他一些东西。层归一化我们也会单独讨论，它本质上是一个归一化层，在Transformer块和最终层中都会使用。目前，只需将其视为一个宏观概念。

## 4.2：使用层归一化归一化激活值 📊

上一节我们介绍了GPT模型的整体架构，本节中我们来看看其中一个关键组件：层归一化。

在宏观计划中，我们之前已经介绍了GPT模型架构，其中包含几个组成部分：第2章介绍的嵌入层，第3章讨论的掩码多头注意力模块，现在还有其他几个东西。我们有了这个目前是占位符的虚拟GPT模型，并说我们将在接下来的章节中填充空白。这就是我们现在要对其中一个组成部分做的事情：层归一化。

![](img/2b7ae7eb4d07123e4c0e4529c48bfd6d_28.png)

![](img/2b7ae7eb4d07123e4c0e4529c48bfd6d_30.png)

![](img/2b7ae7eb4d07123e4c0e4529c48bfd6d_32.png)

层归一化（LayerNorm）是2016年引入的一个经典深度学习概念，它关乎归一化给定层的输出（这些输出将进入下一层），以便为优化提供良好的属性。

为了让这一点更清晰，让我们看一下我准备的代码示例。为了简单起见，在这个视频中，我将使用一个更概念化、更简单的例子，而不是在真实LLM的上下文中使用层归一化。

我设置了一个随机种子，然后创建了一个更小的例子。让我们使用 `torch.randn` 生成一个 `2 x 5` 的随机样本。我们有两个样本，每个样本有五个维度。我们可以将其视为一个嵌入在五维空间中的词元，类似于第2章中我们讨论的嵌入在三维空间中的词元。在真实的LLM案例中，嵌入维度可能更大（例如768），但为了简单起见，让我们使用一个更小的例子，以便可以将其放在图表中。

本质上发生的是，对于一个给定的样本，我们有这五个输入值。现在我们要做的是，将它们通过一个小型神经网络，然后得到输出值。我们希望确保这些输出值在某个特定的值范围内。如果我们只是应用神经网络层，我们可能会得到这些值的均值为0.13，方差为0.39，这实际上还不错。但如果你熟悉经典的深度神经网络，可能会存在梯度消失或梯度爆炸等问题。将给定层的输入（以及下一层的输入）归一化有助于稳定训练。

一个给定层的输出（例如这些值）将进入深度神经网络中的下一层，因为网络中有许多层。如果我们有非常小的值或非常大的值，这可能不是最优的，可能导致训练不稳定。因此，我们尝试在这里添加一个称为层归一化的特定层，以归一化进入下一层的输出，使它们具有均值0和方差1。这是我们的目标。

![](img/2b7ae7eb4d07123e4c0e4529c48bfd6d_34.png)

![](img/2b7ae7eb4d07123e4c0e4529c48bfd6d_35.png)

考虑我们的输入，然后也实现一个小型层。我们使用 `nn.Sequential`，然后使用 `nn.Linear` 创建神经网络层。输入是5维，输出使用6以匹配我们展示的内容。然后我们应用这个层到批次样本。技术上我们还可以添加一个非线性激活函数，例如ReLU。在深度神经网络中，我们通常将线性层与非线性激活函数配对，因为否则，如果神经网络中只有线性层，线性层的组合将是一个线性函数，那将不是一个非常有用的神经网络。因此，我们通常添加非线性，以便网络可以学习更复杂的东西。

![](img/2b7ae7eb4d07123e4c0e4529c48bfd6d_37.png)

![](img/2b7ae7eb4d07123e4c0e4529c48bfd6d_39.png)

现在看看输出。我们得到一些输出值，两个样本。第一行对应第一个样本，第二行对应第二个样本，现在通过这个小型神经网络进行了转换。第一行就是我们在这里看到的，如果我们关注一个样本的话。

在开始任何层归一化或归一化之前，让我们先看看这个输出的均值和方差。我们可以调用 `.mean()`。但默认情况下，`.mean()` 会计算整个张量的平均值，这不是我们想要的。我们实际上想要第一个样本的均值。有几种方法可以计算。我们可以计算整个东西的均值，但这会混合多个样本，不是我们想要的。另一种方法是使用 `dim=0`，但这会计算每列（跨样本）的均值，同样混合了不同的样本。这实际上是另一种称为**批量归一化**的归一化方案。批量归一化实际上并不推荐用于LLM，因为当我们训练神经网络并希望跨多个GPU分割它们时，一种方法是将样本跨多个GPU分割，那么每个GPU最终得到的样本量很小，这对于批量归一化来说非常不稳定。

长话短说，我们使用的是称为**层归一化**的概念，其中层归一化与样本大小无关，它是在特征维度上进行归一化。为了得到这个维度，我们使用列维度 `dim=1`。这将在此处进行归一化（计算均值）。我们也可以输入 `dim=-1`，这样更通用。例如，如果我们因为某种原因有一个额外的维度，使用 `-1` 仍然会给出相同的结果。但如果我们写 `dim=1`，当有额外维度时，它会再次计算跨维度0和1的均值，这不是我们想要的。我们想要的是跨特征维度的均值，假设特征维度始终是最后一个维度，我们可以只使用更通用的 `-1`。

另外，我们可以通过设置 `keepdim=True` 来保持原始维度，这样会保留这两个维度（两行），看起来更清晰。这就是均值。我们可以对方差做同样的事情。

我们的目标是将数据归一化，使得均值为0，方差为1。我们实现这一点的一种方法是围绕均值进行中心化。如果我们有一个输出并减去均值，这将给出以均值为中心的输出。例如，一个简单的例子，如果值只是5,5,5,5,5，减去平均值（也是5），5-5=0，这样做应该得到0。这里我们得到的结果非常接近0（科学计数法表示）。为了看起来更清晰，我们可以设置打印选项。

![](img/2b7ae7eb4d07123e4c0e4529c48bfd6d_41.png)

![](img/2b7ae7eb4d07123e4c0e4529c48bfd6d_42.png)

现在我们已经以0为中心了。我们还想做的是使其方差为1。为此，我们除以标准差。标准差是方差的平方根。为了简单起见，你可以假设这是归一化的公式：减去均值并除以标准差。这样应该仍然均值接近0，并且方差应该为1。这就是我们在这里看到的。到目前为止一切顺利。我们实现了这一点。这是层归一化的一部分。

![](img/2b7ae7eb4d07123e4c0e4529c48bfd6d_44.png)

![](img/2b7ae7eb4d07123e4c0e4529c48bfd6d_46.png)

层归一化实际上还有更多内容。它实际上是一个神经网络层，会学习一些参数。让我在这里复制我从头开始实现的层归一化类，向你展示我的意思。

我们看到的是一个神经网络类，有一个初始化方法（构造函数），其中有一些东西。一个是参数。正如你可能从第3章回忆起的，参数是可训练的神经网络权重。这里我们有相同的大小和嵌入维度。参数将使这六个值可训练，我们称之为缩放参数。类似地，我们有一个偏移参数，默认初始化为零。我稍后会解释为什么初始化为1和0。目前，暂时忽略 `eps`。

目前，我们在这里拥有的正是我们之前拥有的归一化：我们计算了均值和方差，然后减去均值并除以标准差。这正是我们之前所做的归一化。

现在我们添加了一个概念：`eps`。`eps` 是做什么的？它防止除以0的错误。如果我们有0的平方根（基本上是0），我们通过添加一个很小的值（例如0.00001）来防止除以0。这就是我们在这里做的。

![](img/2b7ae7eb4d07123e4c0e4529c48bfd6d_48.png)

![](img/2b7ae7eb4d07123e4c0e4529c48bfd6d_50.png)

到目前为止一切顺利。我们有这些东西，现在有一些新的东西：这里的缩放和偏移。为了解释发生了什么，让我们这样处理。我们暂时忽略这个。因为我们先进行除法... 不，顺序不对。让我们这样做，也许更容易。我们在这里做的是减去均值，然后我们可选地加回一些值。如果我们检查偏移的原始值，它是零，所以我们本质上是在加零，这类似于什么都不加。然而，网络在后续训练中（第5章）可能会学习到对归一化更有益的某些值。从技术上讲，它可能学会撤销均值中心化。我的意思是，如果我减去值5，而网络学会偏移为5，那么我加回5，如果这里什么都没发生，那么这里就什么都没发生。这给了网络撤销这种归一化的选项。长话短说，然后类似地，我们可以对除以标准差做同样的事情。这里的缩放是乘以值。默认情况下，这是1。默认情况下，我们乘以1，这什么都不做，就像保持这个值一样。然而，网络再次，它是一个参数，可能会学习不同的值。它可能学习这个值而不是缩放，然后它基本上会撤销这个除法。这只是给网络提供了学习某些值以略微不同地转换值的选项。这本质上就是层归一化层的样子，默认情况下，由于这里的1和0的设置方式，它将与我们的归一化相同。

现在，让我们在实践中尝试一下。看看它是如何工作的。我们做层归一化，然后我们有嵌入维度参数。然后我们将其应用到我们的输出上。让我们检查一下它们是否确实具有我们期望的属性：均值0和标准差/方差为1。我们可以看到确实是均值接近0，标准差接近1（由于浮点误差）。我在这里注释掉的一件事是 `unbiased`。这只是一个微小的细节。如果你熟悉统计学中的总体统计量和样本统计量，有一个概念叫贝塞尔校正项：当计算方差或标准差时，你不是除以样本大小 `n`，而是除以 `n-1`。`unbiased=True` 基本上是总体统计量，而 `unbiased=False` 是样本统计量。这本身又是一整堂课，我曾经是统计学教授并教授这些内容，但我不想在这里把它变成统计学课。我们在这里拥有的是，从技术上讲，我们可以将其设置为 `False`。你会看到值略有不同，这是因为我们的样本量非常小。如果我增加这个样本量，使其非常大，那么无论是否使用校正项，差异都应该很小。在神经网络训练中，正如我之前提到的，我们的嵌入维度是768，所以是否使用校正项实际上根本无关紧要。我在这里明确设置这个的原因是为了模仿原始GPT团队在使用TensorFlow训练GPT时的行为。长话短说，如果这个 `unbiased` 设置让你感到困惑，不必太担心，这只是一个实现细节，并不那么重要。

前馈网络是包含在更大神经网络（LLM）中的一个小型神经网络，而GELU激活是非线性激活函数。这本质上是深度学习中的一个概念，我们有线性层和非线性激活函数。

在宏观计划中，我们的目标仍然是实现在下一章中要训练的最终GPT架构。然而，GPT架构由多个构建块组成，我们正在这里按顺序处理它们。我们已经介绍了层归一化：首先我们编写了虚拟架构，然后我们介绍了层归一化，编写了位于其他神经网络层之间的归一化层。现在我们将编写GELU激活，它也将位于线性单元或神经网络层之间。
