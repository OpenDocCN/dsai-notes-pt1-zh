# 069：Python列表与元组

![](img/a34c62f14205780e033595fc4fd1d8bc_1.png)

在本节课中，我们将学习Python中的两种复合数据类型：**列表（List）** 和 **元组（Tuple）**。它们是Python中关键的数据结构，用于存储有序的元素序列。我们将从定义、创建、访问元素开始，逐步深入到它们的特性、操作以及核心区别。

---

## 📦 元组（Tuples）

元组是一种**有序的序列**。它使用圆括号 `()` 表示，其中的元素用逗号分隔。元组可以包含不同类型的数据，如字符串、整数、浮点数等，但其自身的类型始终是 `tuple`。

![](img/a34c62f14205780e033595fc4fd1d8bc_3.png)

以下是一个元组的例子：
```python
ratings = (1, 2, 3, 4, 5)
```

![](img/a34c62f14205780e033595fc4fd1d8bc_4.png)

### 访问元组元素

![](img/a34c62f14205780e033595fc4fd1d8bc_6.png)

每个元组元素可以通过**索引**进行访问。索引从 `0` 开始。

![](img/a34c62f14205780e033595fc4fd1d8bc_8.png)

以下是索引与元素的对应关系示例：
| 索引 | 元素 |
|------|------|
| 0    | 1    |
| 1    | 2    |
| 2    | 3    |
| 3    | 4    |
| 4    | 5    |

访问第一个元素：
```python
ratings[0]  # 返回 1
```

![](img/a34c62f14205780e033595fc4fd1d8bc_10.png)

![](img/a34c62f14205780e033595fc4fd1d8bc_11.png)

访问第二个元素：
```python
ratings[1]  # 返回 2
```

![](img/a34c62f14205780e033595fc4fd1d8bc_12.png)

![](img/a34c62f14205780e033595fc4fd1d8bc_13.png)

Python也支持**负索引**，从末尾开始计数（-1表示最后一个元素）：
```python
ratings[-1]  # 返回 5
```

![](img/a34c62f14205780e033595fc4fd1d8bc_15.png)

![](img/a34c62f14205780e033595fc4fd1d8bc_17.png)

### 元组操作

我们可以通过加法来**拼接**两个元组：
```python
tuple1 = (1, 2, 3)
tuple2 = (4, 5, 6)
result = tuple1 + tuple2  # 结果为 (1, 2, 3, 4, 5, 6)
```

![](img/a34c62f14205780e033595fc4fd1d8bc_19.png)

![](img/a34c62f14205780e033595fc4fd1d8bc_20.png)

如果我们想获取元组的一部分，可以使用**切片**。切片时，结束索引是**不包含**的。

![](img/a34c62f14205780e033595fc4fd1d8bc_22.png)

获取前三个元素：
```python
ratings[0:3]  # 返回 (1, 2, 3)
```

获取最后两个元素：
```python
ratings[3:5]  # 返回 (4, 5)
# 或使用负索引
ratings[-2:]  # 返回 (4, 5)
```

![](img/a34c62f14205780e033595fc4fd1d8bc_24.png)

使用 `len()` 函数获取元组长度：
```python
len(ratings)  # 返回 5
```

![](img/a34c62f14205780e033595fc4fd1d8bc_26.png)

### 元组的不可变性

元组是**不可变**的。这意味着一旦创建，就不能修改其内容。这个特性很重要，它保证了数据的稳定性。

![](img/a34c62f14205780e033595fc4fd1d8bc_28.png)

例如，尝试修改元组元素会导致错误：
```python
ratings[0] = 10  # 会引发 TypeError
```

如果我们将一个元组变量赋值给另一个变量，它们会**引用同一个元组对象**，而不是创建副本。因此，修改其中一个变量（通过重新赋值）不会影响另一个变量引用的原始元组。

![](img/a34c62f14205780e033595fc4fd1d8bc_30.png)

![](img/a34c62f14205780e033595fc4fd1d8bc_31.png)

```python
ratings1 = ratings
# ratings1 和 ratings 指向同一个元组对象
```

![](img/a34c62f14205780e033595fc4fd1d8bc_33.png)

如果我们想对元组进行排序等操作，必须创建一个**新的元组**：
```python
sorted_ratings = sorted(ratings)  # 返回一个新的排序后的列表
```

![](img/a34c62f14205780e033595fc4fd1d8bc_35.png)

### 嵌套元组

![](img/a34c62f14205780e033595fc4fd1d8bc_37.png)

元组可以包含其他元组或复杂数据类型，这称为**嵌套**。我们可以使用多层索引来访问嵌套元素。

![](img/a34c62f14205780e033595fc4fd1d8bc_38.png)

![](img/a34c62f14205780e033595fc4fd1d8bc_39.png)

```python
N_T = (1, 2, ("a", "b"), (3, 4))
```

![](img/a34c62f14205780e033595fc4fd1d8bc_41.png)

访问嵌套在索引2处的元组中的第一个元素：
```python
N_T[2][0]  # 返回 "a"
```

我们可以将嵌套结构想象成一棵树，通过逐层索引来访问深层数据。

![](img/a34c62f14205780e033595fc4fd1d8bc_43.png)

![](img/a34c62f14205780e033595fc4fd1d8bc_44.png)

## 📝 列表（Lists）

![](img/a34c62f14205780e033595fc4fd1d8bc_49.png)

列表也是一种**有序的序列**，但它使用方括号 `[]` 表示。列表在许多方面与元组相似，但一个关键区别是列表是**可变**的。列表可以包含字符串、浮点数、整数，并且可以嵌套其他列表、元组等数据结构。

以下是一个列表的例子：
```python
L = ["Michael Jackson", 10.1, 1982]
```

### 访问列表元素

![](img/a34c62f14205780e033595fc4fd1d8bc_51.png)

与元组类似，列表元素也可以通过索引访问，索引规则完全相同。

![](img/a34c62f14205780e033595fc4fd1d8bc_53.png)

访问第一个元素：
```python
L[0]  # 返回 "Michael Jackson"
```

使用负索引访问最后一个元素：
```python
L[-1]  # 返回 1982
```

![](img/a34c62f14205780e033595fc4fd1d8bc_55.png)

### 列表操作

![](img/a34c62f14205780e033595fc4fd1d8bc_57.png)

列表同样支持**切片**操作：
```python
L[-2:]  # 返回 [10.1, 1982]
```

我们可以通过加法来**拼接**两个列表：
```python
L1 = L + ["pop", 10.5]
# L1 变为 ["Michael Jackson", 10.1, 1982, "pop", 10.5]
```

![](img/a34c62f14205780e033595fc4fd1d8bc_59.png)

### 列表的可变性

![](img/a34c62f14205780e033595fc4fd1d8bc_61.png)

由于列表是**可变**的，我们可以直接修改其内容。以下是修改列表的几种方法：

使用 `extend()` 方法添加多个元素（会修改原列表）：
```python
L.extend(["new", 2])
# L 变为 ["Michael Jackson", 10.1, 1982, "new", 2]
```

![](img/a34c62f14205780e033595fc4fd1d8bc_63.png)

使用 `append()` 方法添加单个元素：
```python
L.append("A")
# L 变为 ["Michael Jackson", 10.1, 1982, "new", 2, "A"]
```

![](img/a34c62f14205780e033595fc4fd1d8bc_65.png)

直接通过索引修改元素：
```python
L[0] = "hard rock"
# L 变为 ["hard rock", 10.1, 1982, "new", 2, "A"]
```

使用 `del` 命令删除元素：
```python
del(L[0])
# L 变为 [10.1, 1982, "new", 2, "A"]
```

### 字符串与列表的转换

![](img/a34c62f14205780e033595fc4fd1d8bc_67.png)

![](img/a34c62f14205780e033595fc4fd1d8bc_68.png)

使用 `split()` 方法可以将字符串按特定分隔符（默认为空格）分割成列表：
```python
"A,B,C,D".split(",")  # 返回 ["A", "B", "C", "D"]
```

### 别名与克隆

当我们将一个列表赋值给另一个变量时，两个变量会**引用同一个列表对象**，这称为**别名**。修改其中一个变量会影响另一个。

```python
A = ["hard rock", 10, 1.2]
B = A
A[0] = "banana"
print(B[0])  # 输出 "banana"
```

![](img/a34c62f14205780e033595fc4fd1d8bc_70.png)

为了避免这种副作用，我们可以创建列表的**克隆**（副本）：
```python
B = A[:]  # 或 B = A.copy()
A[0] = "hard rock"
print(B[0])  # 仍然输出 "banana"
```

![](img/a34c62f14205780e033595fc4fd1d8bc_71.png)

![](img/a34c62f14205780e033595fc4fd1d8bc_72.png)

## 🎯 总结

![](img/a34c62f14205780e033595fc4fd1d8bc_77.png)

本节课中我们一起学习了Python中两种核心的复合数据类型：**元组**和**列表**。

![](img/a34c62f14205780e033595fc4fd1d8bc_79.png)

我们了解到：
*   **元组**使用 `()` 定义，是**不可变**的，适用于存储不应被修改的数据序列。
*   **列表**使用 `[]` 定义，是**可变**的，提供了丰富的方法来修改、扩展和操作数据。
*   两者都支持**索引**、**切片**、**拼接**和**嵌套**。
*   理解**可变性**的区别以及**别名**与**克隆**的概念，对于避免程序中的意外错误至关重要。
