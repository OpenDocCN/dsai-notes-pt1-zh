## 概述

## 搜索结构的期望大小分析

![](img/4641725ed883cde75ebecba188ea2d91_13.png)

我们正在同时构建垂直分解和搜索结构。垂直分解本身不依赖于线段插入的顺序。然而，搜索结构的大小和形状可能高度依赖于插入顺序，其大小可能在**线性**和**二次**之间变化。我们的目标是证明其**期望大小是线性的**。

![](img/4641725ed883cde75ebecba188ea2d91_15.png)

![](img/4641725ed883cde75ebecba188ea2d91_17.png)

我们将通过分析在第 `i` 步中创建的新节点的期望数量来证明这一点。为此，我们将使用一种称为**逆向分析**的巧妙方法。我们不直接分析搜索结构节点，而是分析新创建的梯形数量。这是因为被移除的梯形数量、新插入的梯形数量以及其间创建的新节点数量彼此之间存在线性关系。因此，分析新创建的梯形数量的期望值就足够了。

![](img/4641725ed883cde75ebecba188ea2d91_18.png)

![](img/4641725ed883cde75ebecba188ea2d91_20.png)

### 逆向分析

逆向分析的核心思想是：我们不是从前向后看（即考虑第 `i` 次插入时可能发生的情况），而是从后向前看。我们观察在第 `i` 次插入**之后**的垂直分解。此时，所有线段都已存在。

![](img/4641725ed883cde75ebecba188ea2d91_22.png)

![](img/4641725ed883cde75ebecba188ea2d91_24.png)

在这种情境下，如果我们考虑最后一次插入，当前存在的每条线段都有**相同的概率**是最后被插入的那条。因此，我们可以通过以下方式分析最后一次插入创建的梯形数量：查看当前所有线段，并计算如果该线段是最后插入的，会创建多少个新梯形。

例如，对于某条线段，我们可以数出它会创建1、2、3、4、5、6个新梯形。我们对每条线段都进行这个计算。然后，由于每条线段成为最后插入者的概率相同，这些数量的**平均值**就是创建梯形数量的**期望值**。

### 从梯形角度计数

我们定义 `K_i` 为：对所有线段 `S_j` 求和，计算如果线段 `S_j` 是最后插入的，会创建的新梯形数量。

![](img/4641725ed883cde75ebecba188ea2d91_26.png)

![](img/4641725ed883cde75ebecba188ea2d91_28.png)

`K_i = Σ（对于所有线段 S_j） [如果 S_j 最后插入所创建的新梯形数量]`

逆向分析的主要技巧在于，我们可以从梯形的角度来重写这个求和。我们不再对线段求和并数梯形，而是对当前存在的所有梯形求和，并计算有多少条线段会导致该梯形在最后一步被创建。

对于一个给定的梯形，只有**至多四条**线段能导致它在最后一步被创建：定义其**顶部**、**底部**、**左端点**和**右端点**的线段。因此，每个梯形对总和的贡献最多为4。

于是，我们可以得到：
`K_i ≤ Σ（对于所有梯形 Δ） 4 = 4 * (梯形总数)`

### 计算边界

我们知道，对于 `i` 条线段，垂直分解中最多有 `3i + 1` 个梯形。因此：
`K_i ≤ 4 * (3i + 1) = 12i + 4`

`K_i` 是对所有线段求和的结果。因此，**平均值**（即期望值）为：
`E[第 i 次插入创建的新梯形数量] = K_i / i ≤ (12i + 4) / i`

当 `i ≥ 4` 时，可以简化为 `≤ 13`。这意味着，在第 `i` 次插入中，平均创建不超过13个新梯形。

![](img/4641725ed883cde75ebecba188ea2d91_30.png)

### 总期望大小

![](img/4641725ed883cde75ebecba188ea2d91_31.png)

![](img/4641725ed883cde75ebecba188ea2d91_33.png)

现在，我们对所有插入步骤求和（从 `i=1` 到 `n`）。由于每次插入平均创建不超过13个梯形，那么总体上，创建的梯形总数期望为 `O(n)`。由于搜索结构节点数与梯形数成线性关系，因此**搜索结构的期望大小是线性的**。

## 查询时间的期望分析

![](img/4641725ed883cde75ebecba188ea2d91_35.png)

![](img/4641725ed883cde75ebecba188ea2d91_36.png)

接下来，我们分析查询路径的长度，因为它对应于查询时间。考虑一个查询点 `Q`。

![](img/4641725ed883cde75ebecba188ea2d91_37.png)

![](img/4641725ed883cde75ebecba188ea2d91_38.png)

![](img/4641725ed883cde75ebecba188ea2d91_40.png)

我们再次使用逆向分析。我们问：在第 `i` 次插入时，`Q` 所在的梯形发生改变的概率是多少？只有当定义该梯形的某条线段恰好是第 `i` 次插入的线段时，这种情况才会发生。

![](img/4641725ed883cde75ebecba188ea2d91_42.png)

![](img/4641725ed883cde75ebecba188ea2d91_44.png)

对于一个梯形，最多有4条线段定义它（顶、底、左、右）。在 `i` 次插入后，每条线段是第 `i` 次插入的概率是 `1/i`。因此，`Q` 所在的梯形在第 `i` 次插入时发生改变的概率最多为 `4 / i`。

![](img/4641725ed883cde75ebecba188ea2d91_46.png)

![](img/4641725ed883cde75ebecba188ea2d91_47.png)

### 搜索路径长度期望

我们分析的是，经过 `n` 次插入后，搜索路径变长的总次数期望。这等于每次插入时路径变长的概率之和：
`E[搜索路径长度增量] ≤ Σ（i=1 到 n） (4 / i)`

![](img/4641725ed883cde75ebecba188ea2d91_49.png)

![](img/4641725ed883cde75ebecba188ea2d91_50.png)

这个和 `Σ (1/i)` 被称为调和级数，其数量级为 `O(log n)`。因此：
`E[搜索路径长度增量] ≤ 4 * O(log n) = O(log n)`

这意味着，从根节点到叶子的搜索路径的**期望长度是 `O(log n)`**。因此，**期望查询时间为 `O(log n)`**。

![](img/4641725ed883cde75ebecba188ea2d91_52.png)

![](img/4641725ed883cde75ebecba188ea2d91_53.png)

![](img/4641725ed883cde75ebecba188ea2d91_54.png)

基于我们的分析，在给定假设下，我们得到了点定位数据结构的完整性能描述：

![](img/4641725ed883cde75ebecba188ea2d91_61.png)

![](img/4641725ed883cde75ebecba188ea2d91_62.png)

*   **预处理时间**：构建整个结构需要 `O(n log n)` 的期望时间。这包括每次插入时在搜索结构中查找（`O(log i)`）和处理新梯形（线性总和）的成本。
*   **空间复杂度**：搜索结构具有**期望线性大小**，即 `O(n)`。
*   **查询时间**：执行一次点定位查询需要**期望对数时间**，即 `O(log n)`。

## 处理退化情况：符号扰动

![](img/4641725ed883cde75ebecba188ea2d91_66.png)

到目前为止，我们一直假设所有顶点的x坐标都是唯一的。如果不满足这个条件（例如存在垂直线段或顶点共享x坐标），我们在判断左右关系时就会遇到困难。

![](img/4641725ed883cde75ebecba188ea2d91_67.png)

![](img/4641725ed883cde75ebecba188ea2d91_68.png)

解决方案是使用一种**符号扰动**技术，具体来说是**仿射变换**。其思想是对平面进行一个极其微小的倾斜，使得所有x坐标变得唯一，同时保持点与线段的相对位置（如“在上方/下方”关系）不变。

![](img/4641725ed883cde75ebecba188ea2d91_70.png)

实际上，我们并不显式地进行这个几何变换。我们只是**在算法逻辑中采用字典序**来处理坐标比较。具体规则是：当比较两个点 `(x1, y1)` 和 `(x2, y2)` 时，我们先比较 `x` 坐标；如果 `x` 坐标相等，则比较 `y` 坐标。这等价于施加了一个使 `(x, y)` 变为 `(x + εy, y)` 的微小仿射变换（其中 `ε` 是一个非常小的正数）。

### 对算法操作的影响

我们的算法主要需要两种几何测试：
1.  测试一个点是在某条垂直线的左侧还是右侧。
2.  测试一个点是在某条线段的上方还是下方。

![](img/4641725ed883cde75ebecba188ea2d91_72.png)

如果使用字典序进行比较，这两种测试的结果将与对扰动后几何进行测试的结果一致。因此，算法在退化情况下也能正确运行。这种方法优雅地解决了所有退化情况，无需复杂特判。

## 最终定理

综合以上所有内容，我们得到了关于点定位的最终定理：

**给定一个由平面上 `n` 条互不相交的线段组成的平面细分，我们可以在 `O(n log n)` 的期望时间内对其进行预处理，构建一个期望大小为 `O(n)` 的数据结构。对于任何查询点，可以在 `O(log n)` 的期望时间内确定其所在的区域。**
