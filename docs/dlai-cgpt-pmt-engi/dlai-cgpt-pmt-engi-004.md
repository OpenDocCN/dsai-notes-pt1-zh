## 基础设置

![](img/7f0ef2bc5790ea569bc00bf4b63f552b_8.png)

首先，我们需要进行一些基础设置，包括导入必要的库和加载API密钥。以下是初始化代码示例：

```python
import openai

# 加载你的API密钥
openai.api_key = "你的API密钥"

![](img/7f0ef2bc5790ea569bc00bf4b63f552b_10.png)

def get_completion(prompt, model="gpt-3.5-turbo"):
    messages = [{"role": "user", "content": prompt}]
    response = openai.ChatCompletion.create(
        model=model,
        messages=messages,
        temperature=0,
    )
    return response.choices[0].message["content"]
```

![](img/7f0ef2bc5790ea569bc00bf4b63f552b_12.png)

这段代码定义了一个辅助函数 `get_completion`，它将帮助我们向模型发送提示并获取回复。

## 生成基础摘要

![](img/7f0ef2bc5790ea569bc00bf4b63f552b_14.png)

现在，让我们尝试总结一段产品评论。假设我们运营一个电子商务网站，拥有大量用户评论。总结这些长评论可以帮助我们快速浏览更多反馈，从而更好地了解客户的想法。

以下是一个生成简短摘要的提示词示例：

**提示词：**
```
你的任务是为一个电子商务网站的产品评论生成简短摘要。
请总结以下用三重反引号分隔的评论，摘要字数不超过30字。

![](img/7f0ef2bc5790ea569bc00bf4b63f552b_16.png)

评论：```女儿生日得到了这个熊猫毛绒玩具，她非常喜欢并且走到哪里都带着它。玩具柔软可爱，价格也适中，而且比预期提前一天送达。```
```

![](img/7f0ef2bc5790ea569bc00bf4b63f552b_18.png)

运行上述提示后，模型可能会生成如下摘要：

> 柔软可爱的毛绒玩具，女儿很喜欢，价格适中且提前送达。

![](img/7f0ef2bc5790ea569bc00bf4b63f552b_20.png)

这是一个相当不错的总结。通过调整提示词中关于字数或句子数的要求，我们可以轻松控制摘要的长度。

![](img/7f0ef2bc5790ea569bc00bf4b63f552b_21.png)

## 生成有针对性的摘要

![](img/7f0ef2bc5790ea569bc00bf4b63f552b_23.png)

有时，我们生成摘要可能带有特定目的。例如，如果我们想向运输部门提供反馈，我们可以修改提示词来反映这一侧重点。

![](img/7f0ef2bc5790ea569bc00bf4b63f552b_25.png)

以下是针对运输部门的提示词示例：

**提示词：**
```
你的任务是为一个电子商务网站的产品评论生成简短摘要，以向运输部门提供反馈。
请总结以下用三重反引号分隔的评论，专注于提及产品运输和交付方面的内容。摘要字数不超过30字。

![](img/7f0ef2bc5790ea569bc00bf4b63f552b_27.png)

评论：```女儿生日得到了这个熊猫毛绒玩具，她非常喜欢并且走到哪里都带着它。玩具柔软可爱，价格也适中，而且比预期提前一天送达。```
```

![](img/7f0ef2bc5790ea569bc00bf4b63f552b_29.png)

运行后，摘要可能更侧重于交付信息：

> 产品比预期提前一天送达。

![](img/7f0ef2bc5790ea569bc00bf4b63f552b_31.png)

同样地，如果我们想为定价部门生成摘要，可以要求模型专注于与价格和价值感知相关的方面。

**提示词：**
```
你的任务是为一个电子商务网站的产品评论生成简短摘要，以向定价部门提供反馈。
请总结以下用三重反引号分隔的评论，专注于提及产品价格和价值感知相关的方面。摘要字数不超过30字。

评论：```女儿生日得到了这个熊猫毛绒玩具，她非常喜欢并且走到哪里都带着它。玩具柔软可爱，价格也适中，而且比预期提前一天送达。```
```

生成的摘要可能如下：

![](img/7f0ef2bc5790ea569bc00bf4b63f552b_33.png)

> 价格适中，但相对于玩具尺寸可能偏高。

通过这种方式，我们可以为不同的业务部门（如产品部门）生成定制化的摘要，提取对他们最有价值的信息。

![](img/7f0ef2bc5790ea569bc00bf4b63f552b_35.png)

## 提取信息而非总结

![](img/7f0ef2bc5790ea569bc00bf4b63f552b_37.png)

在某些情况下，我们可能不需要完整的总结，而是希望直接提取特定信息。例如，如果只想知道与运输相关的具体事实，可以使用提取信息的提示词。

![](img/7f0ef2bc5790ea569bc00bf4b63f552b_39.png)

**提示词：**
```
你的任务是从以下用三重反引号分隔的产品评论中，提取与运输部门相关的信息。
请仅提取事实，不要进行总结。

评论：```女儿生日得到了这个熊猫毛绒玩具，她非常喜欢并且走到哪里都带着它。玩具柔软可爱，价格也适中，而且比预期提前一天送达。```
```

![](img/7f0ef2bc5790ea569bc00bf4b63f552b_41.png)

模型可能会回复：

![](img/7f0ef2bc5790ea569bc00bf4b63f552b_43.png)

> 比预期提前一天送达。

![](img/7f0ef2bc5790ea569bc00bf4b63f552b_44.png)

这种方法在需要获取非常具体的事实时非常有用。

## 批量处理多个评论

![](img/7f0ef2bc5790ea569bc00bf4b63f552b_46.png)

在实际工作中，我们经常需要处理大量文本。以下是如何使用循环来批量总结多个产品评论的示例。

![](img/7f0ef2bc5790ea569bc00bf4b63f552b_48.png)

首先，我们定义几个评论：

```python
review_1 = """女儿生日得到了这个熊猫毛绒玩具...（此处为完整评论内容）"""
review_2 = """卧室的灯具有三档亮度...（此处为完整评论内容）"""
review_3 = """我的牙医推荐了这款电动牙刷...（此处为完整评论内容）"""
review_4 = """这套17件的搅拌机系统在季节促销时购买...（此处为完整评论内容）"""

![](img/7f0ef2bc5790ea569bc00bf4b63f552b_50.png)

reviews = [review_1, review_2, review_3, review_4]
```

然后，我们遍历列表，为每个评论生成摘要：

![](img/7f0ef2bc5790ea569bc00bf4b63f552b_52.png)

```python
for i in range(len(reviews)):
    prompt = f"""
    你的任务是为一个电子商务网站的产品评论生成简短摘要。
    请总结以下用三重反引号分隔的评论，摘要字数不超过20字。

    评论：```{reviews[i]}```
    """
    response = get_completion(prompt)
    print(f"评论 {i+1} 摘要：{response}\n")
```

运行上述代码，我们可以快速获得所有评论的简短摘要。对于一个拥有数百条评论的网站，你可以利用这种方法构建一个仪表板，展示摘要，让管理者能够快速浏览客户反馈，并决定是否点击查看完整评论。
