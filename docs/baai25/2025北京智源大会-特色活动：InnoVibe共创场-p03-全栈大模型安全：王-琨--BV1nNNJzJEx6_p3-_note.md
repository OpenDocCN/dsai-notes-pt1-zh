## 概述：全栈安全视角

上一节我们介绍了大模型安全的基本背景，本节中我们来看看如何从全栈的视角来系统性地审视安全问题。

报告人系统梳理了从模型构建到应用落地的完整链条中的安全问题。这个链条类比于程序员的前端与后端，具体包括以下阶段：

*   **数据准备与预训练**：模型构建的初始阶段。
*   **模型后训练**：包括微调、模型编辑与遗忘学习。
*   **模型推理与部署**：模型参数固定后的应用阶段。
*   **智能体系统**：模型外接记忆、工具与通讯机制后的复合系统。

该研究整理了近千篇文献，是目前覆盖面最广的全栈安全综述之一。

## 研究重心转移：从静态模型到动态智能体

传统的、针对固定参数模型的攻击（如提示词破解）研究重要性正在下降。原因在于大模型更新迭代速度极快，针对特定版本设计的攻击方法可能很快因模型更新而失效。

相比之下，**智能体**的安全问题变得至关重要。智能体因集成了外部工具、存储和通讯机制，暴露了更多攻击面，并且更接近实际落地应用场景，因此其安全性研究具有更高的现实意义和紧迫性。

## 动参数安全：后训练与编辑技术

在模型参数可调整的范畴内，当前研究主要集中于后训练技术。除了常见的微调，**模型编辑**和**遗忘学习**也被纳入该研究体系。这两项技术旨在为模型的知识更新与冲突寻找精确、可控的解决方案，但目前仍处于早期探索阶段。

以下是该领域关注的核心问题：

*   **知识精确更新**：如何修改模型对特定事实的记忆（如将“美国总统是拜登”改为“是特朗普”）。
*   **最小化副作用**：在更新目标知识时，如何最大限度地保留模型原有的其他知识与能力。
*   **安全影响**：强制修改神经元可能无意中影响其他关联概念（如其他国家元首信息的置信度），这在某些敏感场景下是不可接受的。

## 智能体安全研究实践

基于对智能体安全重要性的判断，研究团队在该方向进行了一系列探索。

### 1. 智能体通讯效率与安全

智能体间通讯可能存在冗余。例如，信息通过多层社交关系传递会导致计算资源的浪费。研究团队将智能体网络建模为图结构，通过图学习技术对网络进行“剪枝”。

**目标**是最大化剪枝后子图与原始图的信息相似度，同时尽可能提高剪枝比例。核心思想是优化智能体系统的通讯架构，提升效率与推理速度。

```python
# 图剪枝核心目标示意（伪代码）
def graph_pruning(original_graph):
    pruned_graph = learn_to_prune(original_graph) # 学习剪枝策略
    # 目标1: 最大化信息保留
    info_similarity = calculate_similarity(original_graph, pruned_graph)
    # 目标2: 最大化剪枝比例
    prune_ratio = calculate_prune_ratio(original_graph, pruned_graph)
    return pruned_graph, info_similarity, prune_ratio
```

### 2. 智能体系统架构优化

针对不同复杂度的任务，所需的智能体数量与协作架构应动态调整。研究团队将其建模为图学习问题，让系统自动学习最优的智能体组织架构。为防止智能体在交互中遗忘核心任务，系统会通过持续提示来锚定任务目标。

### 3. 智能体拓扑结构与安全传播

该研究从拓扑学角度探索智能体安全。不同“身份”的智能体（如“市长”与“普通人”）在网络中具有不同的影响力。通过实验发现，连接度更高的智能体架构，其信息（包括错误信息或有害信息）传播影响力更大。而“身份”较弱的节点，其影响力传播则受限。

### 4. 分层记忆与信息安全

为不同安全等级的信息设计不同的流动通道。例如，个人日程可与朋友分享，但银行卡密码则不能。研究测试了分层的记忆架构，以确保信息能在系统中以可控、安全的方式流转。

### 5. 基于图学习的智能体安全预测

图神经网络具有强大的归纳偏置能力，即在小规模图上训练出的模型可以泛化到更大规模的图上。利用这一特性，可以训练一个图模型来预测单个智能体节点的安全性。当智能体系统规模扩大时，该模型能轻易地迁移，用于预测更大尺度下智能体系统的安全状态。

## 后训练安全实践：模型编辑与遗忘学习

在模型后训练阶段，团队重点关注模型编辑与遗忘学习中的安全问题。

### 1. 阿尔法编辑：最小化副作用的模型编辑

核心思路是将编辑操作对模型原始知识分布的破坏降到最低。

**方法**是先将待编辑的知识投影到一个“零空间”进行修改，然后再映射回原空间。公式化地看，这类似于寻找一个编辑向量 **Δ**，使得在改变目标知识的同时，与原始权重矩阵 **W** 的乘积在无关方向上的变化最小。

```python
# 阿尔法编辑核心思想示意（伪代码）
def alpha_edit(model_weights_W, knowledge_to_edit):
    # 1. 投影到“零空间”
    null_space_projection = project_to_null_space(knowledge_to_edit)
    # 2. 在零空间内进行编辑
    edited_knowledge = edit_in_null_space(null_space_projection)
    # 3. 映射回原空间，得到权重更新量Δ
    delta_W = map_back_to_original_space(edited_knowledge)
    # 目标：应用Δ后，对无关输出的影响最小
    new_W = model_weights_W + delta_W
    return new_W
```

该方法能有效保证在更新特定知识时，最大程度地保留模型的原有性能，相关论文被评为顶级会议的最佳论文。

### 2. 诱导式遗忘学习

模型编辑解决“时间上”的知识冲突（新知识覆盖旧知识），而遗忘学习解决“空间上”的实体冲突（要求模型忘记特定信息）。当前遗忘学习技术要么不精确，要么会对模型造成严重损伤。

团队提出一种新方法：
1.  在上游训练一个特殊的“诱导令牌”。
2.  当模型在推理中遇到需要遗忘的内容时，通过编辑技术将该令牌强制嵌入推理链。
3.  该令牌会诱导模型输出“我不知道”，从而实现精确且低损伤的遗忘。

实验表明，该方法性能显著优于现有基线模型。

## 推理阶段安全：幻觉问题初探

在模型参数完全固定的推理阶段，**幻觉**是核心安全问题。一项有趣的研究发现，大模型在进行长链推理时，有时会跳过关键的逻辑判断步骤（如 `if-else` 分支），直接输出结果。这种“判断机制”的缺失是产生幻觉的重要原因之一。这项研究无需修改模型参数，为理解幻觉根源提供了新视角。

## 总结与未来展望

本节课中我们一起学习了全栈大模型安全的框架与前沿研究。

**总结当前观点**：
1.  **智能体安全**和**模型后训练安全**（特别是编辑与遗忘）的重要性日益凸显。
2.  **推理模型的安全**，尤其是长链推理中不断加深的幻觉问题，值得深入研究。
3.  **多模态推理模型**的安全是新兴且重要的方向，随着该类模型在2024年的崛起，相关安全问题将成为研究热点。
