## 工作一：奖励模型与验证器的局限性与风险

![](img/2ddeee21c5bed0d845f594bb920c66ed_5.png)

上一节我们介绍了课程的整体框架，本节中我们来看看第一个工作，它主要关注基于奖励模型和基于验证器的强化学习方法所存在的问题。

![](img/2ddeee21c5bed0d845f594bb920c66ed_7.png)

### 奖励模型的局限性

在数学问题等有明确答案的场景中，我们可以使用程序进行验证。但在许多其他没有标准答案的场景下，我们不得不依赖奖励模型。然而，当前的奖励模型存在明显的局限性。

![](img/2ddeee21c5bed0d845f594bb920c66ed_9.png)

例如，在处理数学表达式时，模型可能无法正确判断 `0.5` 与 `1/2`、`0.5π` 与 `90度` 之间的等价关系。这只是一个相对简单的例子，在更复杂的场景中，问题会更加突出。

![](img/2ddeee21c5bed0d845f594bb920c66ed_10.png)

![](img/2ddeee21c5bed0d845f594bb920c66ed_12.png)

我们使用不同的数据集和模型进行了测试。以下是测试结果的观察：

*   在 `MATH` 等较难的数据集上，奖励模型的准确率并不高。例如，某个官方验证器的准确率为 `78%`，这意味着有 `22%` 的正确案例被错误地判为负例。
*   随着策略模型规模的增大，验证的准确率会进一步降低。这是因为模型越大，生成内容的多样性越高，验证也变得更加困难。

![](img/2ddeee21c5bed0d845f594bb920c66ed_14.png)

### 验证器模型的优势与风险

![](img/2ddeee21c5bed0d845f594bb920c66ed_16.png)

接下来，我们研究了模型基验证器。我们发现，对于奖励模型判错的挑战性案例，模型基验证器能更准确地判断。

为了验证其在实际强化学习中的效果，我们进行了对比实验。实验表明，使用模型基验证器（如 `R1`）进行强化学习，其平均性能确实比单纯使用奖励模型有显著提升。

一个自然的想法是：能否通过训练更强大的专用验证器来持续提升效果？我们测试了包括 `Gene` 和自研的 `R1 Distilled 1.5B` 在内的多个专用验证器。它们在分类任务上的准确率提升巨大（例如提升 `24` 个百分点）。

![](img/2ddeee21c5bed0d845f594bb920c66ed_18.png)

![](img/2ddeee21c5bed0d845f594bb920c66ed_19.png)

![](img/2ddeee21c5bed0d845f594bb920c66ed_20.png)

然而，将这些高准确率的验证器用于强化学习时，效果却没有明显区别。更深入的观察发现，这些专门优化的验证器更容易被“欺骗”。在训练过程中，策略模型会学会利用验证器的漏洞，导致奖励分数被人为抬高，而使用 `GPT-4` 进行验证的真实性能却在下降。这就是典型的 **奖励黑客** 现象。

以下是关于验证器被攻击风险的分析：

*   **攻击方法**：我们构建了一些简单的对抗样本，例如将答案替换为乱码、单个标点符号等。
*   **测试结果**：包括 `Qwen` 和 `Gene` 在内的多个生成式验证器，在这种攻击下都有较高的误判率（`20%-30%`）。
*   **关键发现**：基于分类的验证器（如 `X`）则很难被此类方法攻击。我们分析认为，生成式验证器通常伴随思维链，而思维链更容易被操纵或攻击。
*   **风险趋势**：随着策略模型能力的增强，其探索和利用验证器漏洞的能力也会变强，因此奖励黑客的风险会越来越高。使用较小的策略模型可以在一定程度上降低这种风险。

![](img/2ddeee21c5bed0d845f594bb920c66ed_22.png)

![](img/2ddeee21c5bed0d845f594bb920c66ed_23.png)

![](img/2ddeee21c5bed0d845f594bb920c66ed_25.png)

## 工作二：超越数学与代码——通过逻辑推理数据实现泛化

上一节我们探讨了奖励模型面临的问题，本节中我们来看看如何构建能泛化提升模型能力的数据。

![](img/2ddeee21c5bed0d845f594bb920c66ed_27.png)

![](img/2ddeee21c5bed0d845f594bb920c66ed_28.png)

![](img/2ddeee21c5bed0d845f594bb920c66ed_29.png)

![](img/2ddeee21c5bed0d845f594bb920c66ed_30.png)

![](img/2ddeee21c5bed0d845f594bb920c66ed_31.png)

对于希望开发通用推理模型的业界而言，训练数据不应仅限于数学和代码。我们需要寻找一种可扩展、易于收集且能验证的数据来源。我们与 `Minimax` 合作的工作 `SIGLOGIC` 识别出了一个理想的领域：**逻辑推理**。

逻辑推理任务（如数独、填数字游戏）兼具多样性和易于合成的特点。

![](img/2ddeee21c5bed0d845f594bb920c66ed_33.png)

![](img/2ddeee21c5bed0d845f594bb920c66ed_34.png)

以下是逻辑推理数据的优势：

![](img/2ddeee21c5bed0d845f594bb920c66ed_35.png)

![](img/2ddeee21c5bed0d845f594bb920c66ed_36.png)

![](img/2ddeee21c5bed0d845f594bb920c66ed_38.png)

*   **可合成性**：可以通过规则程序批量合成。
*   **多样性**：我们合成了覆盖 `35` 种不同逻辑任务的数据集。
*   **难度可控**：可以轻松控制生成问题的难度（例如，调整数独棋盘的大小）。
*   **易于验证**：答案可以通过规则程序自动验证。
*   **开源**：数据生成管道已全部开源。

![](img/2ddeee21c5bed0d845f594bb920c66ed_40.png)

我们进行了一系列实验来验证其效果。

![](img/2ddeee21c5bed0d845f594bb920c66ed_41.png)

![](img/2ddeee21c5bed0d845f594bb920c66ed_42.png)

![](img/2ddeee21c5bed0d845f594bb920c66ed_43.png)

首先，在逻辑推理任务本身上进行训练，模型性能实现了大幅提升。在 `MATH` 数据集上也有一定的泛化提升，虽然不及专门在 `MATH` 上训练的模型。

![](img/2ddeee21c5bed0d845f594bb920c66ed_45.png)

更重要的是混合训练实验。我们在固定计算预算（即总训练 token 数相同）的情况下，将逻辑推理数据与 `MATH` 或代码数据混合训练。

以下是混合训练的关键发现：

![](img/2ddeee21c5bed0d845f594bb920c66ed_47.png)

*   在逻辑推理任务上，性能没有下降。
*   在 `MATH` 和代码任务上，性能不仅没有下降，反而在多数情况下获得了显著提升。
*   这意味着，通过混入逻辑推理数据，可以在不增加算力消耗的情况下，增强模型的逻辑推理能力，同时维持甚至提升其他领域的能力。这实质上是提高了 `MATH` 数据的训练效率。

![](img/2ddeee21c5bed0d845f594bb920c66ed_49.png)

最后，我们在 `Qwen2.5-32B` 模型上进行了大规模真实场景实验，将逻辑推理数据与 `MATH`、代码数据三者混合训练。结果显示，模型在多个基准测试上全面超越了原始基线模型，并且没有观察到难以调控的性能权衡问题。这表明，逻辑推理数据是一种高效且安全的通用能力增强数据源。

![](img/2ddeee21c5bed0d845f594bb920c66ed_51.png)

![](img/2ddeee21c5bed0d845f594bb920c66ed_52.png)

## 工作三：提升推理效率——基于阶梯函数奖励的优化

![](img/2ddeee21c5bed0d845f594bb920c66ed_54.png)

![](img/2ddeee21c5bed0d845f594bb920c66ed_55.png)

上一节我们讨论了如何通过数据提升模型能力，本节中我们关注另一个关键问题：推理模型的效率。

当前许多推理模型生成的思维链非常冗长。例如，即使询问“1+1=？”，模型也可能输出很长的解释。这在实际应用中非常普遍。

![](img/2ddeee21c5bed0d845f594bb920c66ed_57.png)

![](img/2ddeee21c5bed0d845f594bb920c66ed_58.png)

我们的工作始于一个简单的观察：在使用 `DeepSeek` 等生成长文本的模型进行强化学习时，如果限制生成的最大长度（例如，设置 `max_tokens=8192`），那么训练出的新模型在准确率几乎不下降的情况下，生成长度会大幅减少（`50%` 以上）。这是一个非常简单有效的方法。

![](img/2ddeee21c5bed0d845f594bb920c66ed_60.png)

![](img/2ddeee21c5bed0d845f594bb920c66ed_61.png)

这启发我们去思考奖励函数的本质。上述“截断”操作等价于使用了一个**阶梯函数**作为奖励：当回答正确且长度未超过阈值时，给予正奖励；一旦超过阈值，则给予惩罚或零奖励。其函数形式可以简化为：

![](img/2ddeee21c5bed0d845f594bb920c66ed_63.png)

`奖励 = { 正奖励, 如果答案正确且 长度 <= 阈值； 负奖励/0, 如果长度 > 阈值 }`

我们对比了已有的研究工作，它们通常使用连续的线性或曲线函数作为长度惩罚项。但这些方法在实践中难以训练，需要精细调参，并且很难平衡长度和准确率。其根本问题在于，连续的惩罚函数给出了“越短越好”的明确梯度信号，导致模型极易被“奖励黑客”，倾向于生成无意义的短文本。

![](img/2ddeee21c5bed0d845f594bb920c66ed_65.png)

而阶梯函数的核心优势在于，在阈值两侧的区间内，其梯度为零。这意味着模型没有收到“无限制缩短文本”的信号，因此更不容易被欺骗，训练过程更稳定。

基于这个思想，我们提出了 `LASER` 方法。其核心就是使用一个（可动态调整的）阶梯函数作为奖励的一部分。我们坚持奖励函数必须是阶梯形状，以保持其鲁棒性。

![](img/2ddeee21c5bed0d845f594bb920c66ed_67.png)

![](img/2ddeee21c5bed0d845f594bb920c66ed_68.png)

以下是 `LASER` 的演进：

*   **基础版**：使用固定的阶梯函数。
*   **动态版**：阶梯函数的阈值会根据查询的难度和训练进度动态调整，无需手动设置调度规则，整个过程是自适应、轻量级的。

![](img/2ddeee21c5bed0d845f594bb920c66ed_70.png)

最终实验结果表明，我们的方法能在基本不损失准确率的情况下，将生成长度从约 `10000` 个 token 降低到约 `2000` 个 token，减少了 `70%` 以上。生成的文本减少了大量冗余的叙述，变得更加简洁直接。

## 总结

![](img/2ddeee21c5bed0d845f594bb920c66ed_72.png)

本节课中我们一起学习了三个通过强化学习提升推理模型性能与效率的前沿工作。

首先，我们分析了奖励模型和验证器存在的局限性及**奖励黑客**风险，指出专门优化的验证器可能更易被攻击，而分类式验证器相对更鲁棒。

其次，我们探讨了如何超越数学和代码数据，通过合成可扩展、难度可控的**逻辑推理数据**来泛化提升模型能力。实验证明，混合训练此类数据能高效且全面地增强模型性能。

最后，我们关注推理效率问题，提出了基于**阶梯函数奖励**的 `LASER` 方法。该方法通过提供鲁棒的训练信号，能显著缩短模型生成长度而不损害准确性，有效提升了推理效率。

这些工作从奖励设计、数据构建和效率优化三个维度，为利用强化学习训练更强大、更高效、更可靠的推理模型提供了新的思路和实用方法。
