# 特色活动：InnoVibe共创场-p01-通过学习机制激发大模型推理时扩展能力：刘子君

在本节课中，我们将学习如何通过设计合适的学习机制，来激发大语言模型在推理阶段的扩展能力。我们将探讨多种扩展维度及其对应的学习方法。

## 概述

推理时扩展能力已成为大语言模型发展的关键维度。自OpenAI的o1模型发布以来，业界认识到，仅靠增加模型参数（训练时扩展）可能面临瓶颈，而通过特定学习机制在推理阶段提升模型能力（推理时扩展）则展现出巨大潜力。本报告将系统介绍四种不同的推理式扩展方式及其对应的学习机制。

## 多智能体上下文学习

上一节我们介绍了推理时扩展的重要性，本节中我们来看看第一种扩展方式：多智能体上下文学习。这种方式旨在通过集成多个大模型智能体来协同完成任务，以期获得优于单个模型的效果。

理想情况下，系统中纳入的智能体越多，任务效果应越好。然而，现有基于上下文学习的多智能体系统存在缺陷：当智能体数量达到一定阈值后，性能提升会停滞。

以下是现有多智能体系统面临的问题：
*   性能随智能体数量增加而提升，但存在瓶颈。
*   达到瓶颈后，增加更多智能体无法带来效果提升。

为此，我们提出了一种**团队优化**方法。该方法通过智能体之间的**互监督**机制，从大量智能体中筛选出最优的少数智能体组合。

**核心机制**：`团队优化 = 互监督筛选`

我们发现，经过团队优化后，使用更少的智能体（例如从7个优化到4个）反而能在特定任务上取得惊人的效果提升。该方法在四类任务上得到验证，能够显著减少系统开销并提升推理效果。

**因此，第一个结论是**：通过上下文学习结合团队优化算法，可以实现**少数精英智能体团队 > 大量普通智能体群体**的效果。

## 多智能体强化学习

除了上下文学习，强化学习是另一种强大的学习机制。上一节我们看到了多智能体协作的静态优化，本节中我们来看看如何用强化学习动态增强多智能体系统的推理扩展能力。

强化学习不仅能让基础模型提升推理能力，在多智能体场景下，还能让模型学会适应系统协作模式和特定任务环境。传统方法需要为不同环境精心设计系统，缺乏泛化性。

我们发现，强化学习能为模型注入两类知识：
1.  **环境知识**：针对特定任务环境（如网页操作），可能泛化性较低。
2.  **合作行为**：智能体间如何协作，这类知识具有高泛化性。

通过设计恰当的强化学习算法（包含多智能体信用重分配策略），我们能够显著提升系统在训练环境（分布内）上的效果，并有效泛化到未见过的环境（分布外），例如从网页环境泛化到手机操作环境。

**核心发现**：在开源模型上应用多智能体强化学习后，其效果可以逼近甚至超过闭源大模型（如GPT-4、Claude）在单智能体模式下的表现。

**因此，第二个结论是**：**小模型智能体 + 强化学习 + 多智能体系统** 可以媲美甚至超过 **大模型单智能体**。

## 奖励模型的强化学习

强化学习作为一种后训练范式，依赖于准确、通用的奖励信号来指导模型优化。上一节我们探讨了多智能体强化学习，本节我们聚焦于强化学习中的一个关键组件——奖励模型——的扩展。

目前，推理模型在数学、代码等可验证任务上成果显著，因为这些任务的奖励信号（规则）清晰明确。但对于更复杂、通用的任务，如何获得高质量的奖励信号是一大挑战。

现有奖励建模方法（如输出单个标量分数）在推理时扩展能力有限。我们提出采用**生成式奖励建模**方法，并结合**基于规则的强化学习**。

**核心方法**：`生成式奖励模型 + 基于规则的强化学习 -> 推理时扩展能力`

我们发现，通过**并行采样**策略，小尺寸的生成式奖励模型能够大幅提升奖励信号的准确度。例如，一个27B参数的小模型，在32次并行采样下，其奖励准确度可以超过一个671B参数的大模型。

**因此，第三个结论是**：在强化学习中，使用**小的奖励模型 + 并行采样**，再结合基于规则的强化学习算法，其效果可以超过**大的奖励模型**。

## 基于外部知识输入的上下文学习

模型的输入信息量是推理扩展的另一个关键维度。上一节我们改进了奖励模型，本节我们关注如何让模型在推理时利用远超其上下文窗口的海量外部知识。

若想突破模型固有的上下文窗口限制，现有三种主要途径：
1.  **检索增强生成（RAG）**
2.  **信息压缩**
3.  **多智能体交流**

我们发现，前两种方法的推理扩展能力较弱。第三种方法理论上具备扩展能力，但在处理复杂知识任务（如长文本问答、学术综述写作）时效果不佳。当输入知识远超窗口大小时（如对128K窗口输入更多信息），现有方法性能会出现崩溃。

我们设计了一种新的上下文学习方法来解决此问题。该方法使模型能够有效利用远超上下文窗口的外部知识，持续提升在知识密集型任务上的性能。

**效果对比**：
*   **我们的方法**：随着输入知识量的增加（即使超过窗口限制），模型回复质量持续提升。
*   **现有基线方法（如RAG）**：输入知识量增加后，性能提升缓慢或停滞。
*   **现有多智能体方法**：输入知识超过窗口后，性能出现崩溃式下降。

**因此，第四个结论是**：通过有效的上下文学习方法，**大模型 + 海量外部知识** 的效果可以超过 **大模型 + 有限外部知识**。进一步地，**小模型 + 海量外部知识** 也能媲美 **大模型 + 有限外部知识** 的推理能力。

## 总结与展望

本节课中我们一起学习了激发大模型推理时扩展能力的四种学习机制。

最后，我们对训练时扩展与推理时扩展的关系进行展望。有观点认为，纯粹的预训练规模扩展可能面临终结，但新的研究表明，改进预训练方法本身可以为后训练乃至推理时扩展提供更好基础。例如，预训练阶段获得的能力决定了模型经过强化学习后所能达到的上限。

**未来展望**：预训练的目标或许可以从单纯扩大规模，转向为更好地服务于**推理时的扩展能力**。基于适当学习机制激发的强推理扩展能力，将是未来大模型及智能体系统迈向AGI（通用人工智能）的重要基石。

**一致发现**：**适当的学习机制**是激发大模型**强大推理时扩展能力**的关键。
