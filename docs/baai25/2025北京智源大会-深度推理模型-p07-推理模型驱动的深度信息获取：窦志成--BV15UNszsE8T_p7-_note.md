# 深度推理模型-p07-推理模型驱动的深度信息获取：窦志成

![](img/4b5512482cbdf296a26269d98f593a52_1.png)

在本节课中，我们将学习如何利用深度推理模型来驱动更复杂、更深入的信息获取任务。我们将探讨传统搜索引擎与大语言模型各自的优缺点，并介绍一种结合两者优势的新范式——深度搜索。

## 信息获取的挑战与演进

![](img/4b5512482cbdf296a26269d98f593a52_3.png)

上一节我们介绍了信息获取的基本背景，本节中我们来看看其发展历程与核心挑战。

互联网上的信息规模极其庞大，网站数量超过20亿，索引网页规模达百亿级别。然而，用户所需的具体信息往往只存在于少数几个页面中。这就像在浩瀚的信息宇宙中“大海捞针”。

![](img/4b5512482cbdf296a26269d98f593a52_5.png)

搜索引擎的出现有效缓解了这一问题。其核心架构包含离线与在线处理，在线部分的关键是**检索与排序**，最终以“10个蓝色链接”的形式返回结果。用户需要自行点击、浏览并整合信息。

![](img/4b5512482cbdf296a26269d98f593a52_6.png)

![](img/4b5512482cbdf296a26269d98f593a52_7.png)

![](img/4b5512482cbdf296a26269d98f593a52_8.png)

![](img/4b5512482cbdf296a26269d98f593a52_9.png)

这种方式的优点是：对于简单、确定的信息需求，响应速度快。但其缺点是：难以处理复杂的、需要多源信息整合的查询。

![](img/4b5512482cbdf296a26269d98f593a52_11.png)

![](img/4b5512482cbdf296a26269d98f593a52_12.png)

例如，查询“2024年CCF ADL办了几期？分别的标题、学术主任和报名网站是什么？”。传统搜索引擎无法在一个页面内提供所有答案，用户需要自行加工处理。

## 大语言模型带来的新范式

![](img/4b5512482cbdf296a26269d98f593a52_14.png)

![](img/4b5512482cbdf296a26269d98f593a52_15.png)

![](img/4b5512482cbdf296a26269d98f593a52_16.png)

上一节我们回顾了传统搜索的局限，本节中我们来看看大语言模型如何改变了信息获取的方式。

以ChatGPT为代表的大语言模型提供了一种新的交互方式：用户可以用自然语言直接提问，模型直接生成文本答案，而非链接列表。

![](img/4b5512482cbdf296a26269d98f593a52_18.png)

![](img/4b5512482cbdf296a26269d98f593a52_19.png)

![](img/4b5512482cbdf296a26269d98f593a52_20.png)

其优点是：
*   能够处理以自然语言描述的复杂信息需求。
*   可以“一步到位”地生成整合好的答案，节省了用户浏览、复制、粘贴、合成的时间。

这种方式恰好弥补了传统搜索的缺陷。于是，一个问题自然产生：大语言模型是否会取代搜索引擎，成为主要的信息获取方式？

![](img/4b5512482cbdf296a26269d98f593a52_22.png)

答案是否定的。因为信息获取的核心要求是**准确性**，而大语言模型存在固有缺陷。

## 大语言模型在信息获取中的固有缺陷

![](img/4b5512482cbdf296a26269d98f593a52_24.png)

![](img/4b5512482cbdf296a26269d98f593a52_25.png)

![](img/4b5512482cbdf296a26269d98f593a52_26.png)

![](img/4b5512482cbdf296a26269d98f593a52_27.png)

![](img/4b5512482cbdf296a26269d98f593a52_28.png)

上一节我们看到了大语言模型的优势，本节中我们重点分析其为何无法独立承担信息获取任务。

![](img/4b5512482cbdf296a26269d98f593a52_29.png)

大语言模型存在两个致命缺陷：

![](img/4b5512482cbdf296a26269d98f593a52_31.png)

**1. 幻觉问题**
幻觉指模型生成看似合理但不符合事实的内容。研究表明，只要训练目标是自回归生成式语言模型，幻觉就不可避免。模型本质是对训练数据的压缩，那些在语料中仅出现一次的“长尾事实”在压缩过程中极易被损失掉。

例如，询问“中国人民大学人工智能学院的研究方向”，模型可能列出“基础理论、交叉应用、伦理治理”，但遗漏了该学院实际火热的“大模型、信息检索”方向，并捏造了不存在的“数据隐私”研究方向。这属于“不懂装懂”式的错误。

![](img/4b5512482cbdf296a26269d98f593a52_33.png)

![](img/4b5512482cbdf296a26269d98f593a52_34.png)

**2. 时效性问题**
模型的知识受限于其训练数据截止日期。例如，询问“北京智源人工智能研究院院长是谁？”，如果模型数据截止到2023年10月，它可能给出过时的答案。对于信息获取任务，提供过时信息是不可接受的。

![](img/4b5512482cbdf296a26269d98f593a52_35.png)

这些缺陷在强调准确性和溯源的信息获取任务中是致命的。因此，需要一种结合双方优势的方案。

## 检索增强生成：初步的解决方案

![](img/4b5512482cbdf296a26269d98f593a52_37.png)

![](img/4b5512482cbdf296a26269d98f593a52_38.png)

上一节我们指出了大模型的缺陷，本节中我们介绍第一种结合方案：检索增强生成。

![](img/4b5512482cbdf296a26269d98f593a52_39.png)

![](img/4b5512482cbdf296a26269d98f593a52_40.png)

![](img/4b5512482cbdf296a26269d98f593a52_41.png)

检索增强生成的基本思想是：当模型遇到知识盲区时，通过检索从外部知识源获取相关信息片段，并以此为基础生成答案。

其流程可以概括为：
```
用户问题 -> 检索相关文档/片段 -> 将检索结果作为上下文输入给大模型 -> 大模型生成最终答案
```
这种方式部分缓解了模型的幻觉和知识陈旧问题。然而，它仍然存在局限。

例如，对于复杂查询“2024年CCF ADL办了几期？”，即使使用RAG，模型可能仍会返回不完整（如只列出4期）或包含错误（如混入2025年信息）的答案。这是因为简单的“一次检索+生成”模式，难以模拟人类处理复杂信息需求时的多步、迭代式搜索与推理过程。

![](img/4b5512482cbdf296a26269d98f593a52_43.png)

## 迈向深度搜索：以推理模型驱动

![](img/4b5512482cbdf296a26269d98f593a52_44.png)

![](img/4b5512482cbdf296a26269d98f593a52_45.png)

上一节我们看到RAG仍有不足，本节中我们探讨更先进的解决方案：深度搜索。

![](img/4b5512482cbdf296a26269d98f593a52_47.png)

人类如何处理复杂信息获取任务？我们会**拆解问题**、**进行多轮搜索**、**验证与整合信息**。例如，先搜索“CCF ADL”，发现结果以序列号（如147, 149）呈现，则会尝试补全缺失的序列号（如148）进行搜索，并判断时间范围（是否在2024年内）。

深度搜索的目标，正是利用深度推理模型来模拟这一过程。推理模型负责理解任务与意图，并协同调用搜索工具，最终完成信息整合。传统搜索引擎则转变为底层服务，上层由智能体进行深度、复杂的任务规划与执行。

现有的一些开源工具（如Glean的“Research”功能、Breadth）采用了**工作流**模式，即预先定义好任务分解、深度搜索、内容合成等固定阶段。

然而，工作流模式将模型调用拆分为多个分离步骤，难以让模型进行连贯、统一的思考。为此，我们提出了 **WebThinker** 框架。

## WebThinker：边思考、边搜索、边写作

上一节我们介绍了深度搜索的理念，本节中我们深入讲解WebThinker框架的核心设计。

WebThinker的核心思想是**边思考、边搜索、边写作**。它并非分阶段的工作流，而是构建一个端到端的推理链，让模型在思考过程中自主决定何时需要搜索、何时需要写作。

以下是其核心模块：

**1. 深度网页探索模块**
当主推理链思考到某一步，判定自身知识不足时，会生成一个搜索请求。该模块被激活后，不仅获取搜索结果，还会自动点击页面中的相关链接，进行深度浏览，并将核心信息摘要返回给主推理链。

![](img/4b5512482cbdf296a26269d98f593a52_49.png)

**2. 报告撰写与润色模块**
当推理链积累足够信息后，会触发写作请求。写作过程中若发现信息缺失，可再次激活搜索。完成部分章节后，模型会主动检查文章结构，进行润色和整合，但只将章节标题等结构化信息而非全部文本返回推理链，以避免上下文过长。

![](img/4b5512482cbdf296a26269d98f593a52_50.png)

![](img/4b5512482cbdf296a26269d98f593a52_51.png)

整个系统的训练采用**强化学习**方法。我们通过以下原则构建训练数据：
*   能得出正确答案的推理链优于不能的。
*   在都能得出正确答案的链条中，调用工具次数少的更优。
*   在调用工具次数相同的情况下，推理步骤少的更优。

实验表明，WebThinker在复杂问题求解和报告撰写任务上，性能均显著优于仅使用推理模型、推理模型+RAG或简单使用搜索工具的方法。

![](img/4b5512482cbdf296a26269d98f593a52_53.png)

## 总结与展望

本节课中，我们一起学习了信息获取从传统搜索到AI搜索的演进之路。

我们首先分析了传统搜索引擎与大语言模型各自的优缺点：搜索**快而准**，但**不智能**；大模型**智能而流畅**，但存在**幻觉与时效性问题**。

随后，我们探讨了结合两者的方案：从初级的检索增强生成，到更先进的、以深度推理模型驱动的**深度搜索**。我们重点介绍了 **WebThinker** 框架，它通过“边思考、边搜索、边写作”的端到端推理链，实现了对复杂信息需求的高效、准确获取。

尽管深度搜索已展现出巨大潜力，但仍面临诸多挑战，例如如何进一步优化搜索策略、如何更精准地评估生成报告的质量与洞察力等。深度推理模型有望推动搜索引擎全面升级，迈向深度智能信息获取的新时代。
