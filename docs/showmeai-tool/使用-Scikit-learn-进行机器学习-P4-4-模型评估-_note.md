## 置换检验：结果是否可靠？🎲

![](img/685f8a9a43bae69c2b04fb232b54014c_11.png)

为了判断模型得分是否显著，我们可以使用**置换检验**。其核心思想是：如果特征（X）和目标（y）之间没有真实关系，那么模型在随机打乱的数据上得到的分数应该与在原始数据上得到的分数相近。

![](img/685f8a9a43bae69c2b04fb232b54014c_12.png)

以下是置换检验的基本步骤：

![](img/685f8a9a43bae69c2b04fb232b54014c_14.png)

1.  在原始数据上训练模型，并记录得分（例如 R² 分数）。
2.  将目标变量 `y` 的值随机打乱（排列），从而破坏 `X` 与 `y` 之间的任何潜在关系。
3.  在打乱后的“垃圾数据”上重新训练模型，并记录新的得分。
4.  重复步骤 2 和 3 多次（例如 100 次），得到一系列“垃圾分数”。
5.  比较原始得分与这一系列垃圾分数的分布。如果原始得分远高于垃圾分数，则说明模型的结果是显著的。

![](img/685f8a9a43bae69c2b04fb232b54014c_16.png)

在 Scikit-learn 中，可以使用 `permutation_test_score` 函数来实现：

```python
from sklearn.model_selection import permutation_test_score

# model 是你的模型（如 LinearRegression）
# X 是特征数据
# y 是目标数据
score, permutation_scores, pvalue = permutation_test_score(model, X, y)
```

![](img/685f8a9a43bae69c2b04fb232b54014c_18.png)

该函数返回三个值：
*   `score`：模型在原始数据上的真实得分。
*   `permutation_scores`：多次打乱数据后得到的分数列表。
*   `pvalue`：**p值**。它表示“获得与原始得分一样好或更好的分数”的概率。p值越小（通常小于 0.05），说明原始得分越不可能是偶然得到的，结果越显著。

![](img/685f8a9a43bae69c2b04fb232b54014c_20.png)

如果原始得分（例如 0.8）远高于所有垃圾分数（例如都集中在 0 附近），并且 p 值非常小，那么我们可以有信心地说，模型确实学到了有意义的关系。

## 交叉验证：获得稳健的评估 🔄

![](img/685f8a9a43bae69c2b04fb232b54014c_26.png)

![](img/685f8a9a43bae69c2b04fb232b54014c_28.png)

当我们仅将数据分成一次训练集和测试集时，评估结果可能对这次特定的数据划分非常敏感（例如，测试集中恰好包含了一些容易预测或难以预测的样本）。为了获得更稳定、可靠的性能评估，我们使用**交叉验证**。

交叉验证的核心思想是：将数据集分成多个“折叠”，轮流将其中一个折叠作为测试集，其余作为训练集，最终取多次评估结果的平均值。

![](img/685f8a9a43bae69c2b04fb232b54014c_30.png)

以下是 K 折交叉验证（以 4 折为例）的流程：

1.  将数据均匀分成 4 份（折叠）。
2.  第 1 轮：使用第 2、3、4 份数据训练，在第 1 份数据上测试，得到分数 S1。
3.  第 2 轮：使用第 1、3、4 份数据训练，在第 2 份数据上测试，得到分数 S2。
4.  第 3 轮：使用第 1、2、4 份数据训练，在第 3 份数据上测试，得到分数 S3。
5.  第 4 轮：使用第 1、2、3 份数据训练，在第 4 份数据上测试，得到分数 S4。
6.  最终模型的性能评估为 `(S1 + S2 + S3 + S4) / 4`。

![](img/685f8a9a43bae69c2b04fb232b54014c_32.png)

![](img/685f8a9a43bae69c2b04fb232b54014c_33.png)

在 Scikit-learn 中，可以使用 `cross_val_score` 函数：

![](img/685f8a9a43bae69c2b04fb232b54014c_35.png)

```python
from sklearn.model_selection import cross_val_score

![](img/685f8a9a43bae69c2b04fb232b54014c_37.png)

# cv=5 表示使用 5 折交叉验证
scores = cross_val_score(model, X, y, cv=5)
mean_score = scores.mean()  # 平均得分
std_score = scores.std()    # 得分的标准差
```

交叉验证的优点：
*   **更稳健**：减少了因单次数据划分的随机性带来的评估波动。
*   **数据利用充分**：所有数据都既被用于训练，也被用于测试。
*   **提供方差信息**：通过多个分数可以计算标准差，了解模型性能的稳定性。

## 模型选择的标准流程 🏆

![](img/685f8a9a43bae69c2b04fb232b54014c_42.png)

在实际项目中，我们通常需要比较多个模型。一个严谨的模型选择和评估流程如下：

![](img/685f8a9a43bae69c2b04fb232b54014c_44.png)

1.  **数据划分**：首先将数据分为**训练集**和**独立的测试集**。测试集在整个模型开发过程中完全不动。
2.  **模型训练与交叉验证**：在训练集上，对每一个候选模型进行交叉验证，计算其平均交叉验证得分。
3.  **模型选择**：选择平均交叉验证得分最高的模型作为“获胜模型”。
4.  **最终评估**：使用之前预留的、从未参与任何训练的**独立测试集**，对获胜模型进行一次最终的性能评估。这个分数才是对外报告、代表模型泛化能力的最终指标。

![](img/685f8a9a43bae69c2b04fb232b54014c_46.png)

> **重要提示**：不能直接用获胜模型在交叉验证中得到的最高分数作为最终报告分数。因为我们在多个模型上进行了尝试，其中最好的分数可能包含“选择偏差”或“运气成分”。独立的测试集提供了对模型泛化能力的无偏估计。

## 总结 📝

![](img/685f8a9a43bae69c2b04fb232b54014c_48.png)

本节课我们一起学习了模型评估的两个核心工具：
1.  **置换检验**：用于判断模型得到的良好分数是否具有统计显著性，而非偶然。
2.  **交叉验证**：用于获得对模型性能更稳健、可靠的估计，减少单次划分的随机性影响。

![](img/685f8a9a43bae69c2b04fb232b54014c_49.png)

![](img/685f8a9a43bae69c2b04fb232b54014c_50.png)

最后，我们明确了模型选择和评估的标准流程：在训练集上通过交叉验证选择最佳模型，然后在完全独立的测试集上进行最终评估。遵循这一流程，能帮助我们建立对模型性能更真实、更可信的认识。
