## 准备工作 🔑

在开始推送模型之前，需要完成两项准备工作。

以下是必须完成的两个步骤：
1.  在 [Hugging Face 网站](https://huggingface.co/) 上注册一个账户。
2.  将你的身份验证令牌存储到本地。这可以通过在终端中运行以下命令轻松完成：
    ```bash
    huggingface-cli login
    ```
    如果你在Google Colab等笔记本环境中，可以在命令前加上感叹号执行：
    ```python
    !huggingface-cli login
    ```
    请注意，如果你在使用常规的Jupyter Notebook且无法访问终端，则需要将访问令牌直接复制到训练参数中，具体位置将在下文说明。

完成上述步骤后，你就可以使用推送API了。

## 使用Trainer API推送模型 📤

推送API允许我们将模型、其配置以及相关的分词器（Tokenizer）一起推送到Hub。

要在训练器（Trainer）中使用此功能，必须确保在训练参数中设置 `push_to_hub=True`。如果不指定模型ID（`model_id`），系统将默认使用输出目录的名称作为仓库名。

以下是关键参数说明：
*   **`push_to_hub=True`**：启用推送功能。
*   **`hub_model_id`**：指定在Hub上创建的仓库名称。如果留空，则使用 `output_dir` 的名称。
*   **`hub_organization`**：只要你是该组织的成员，就可以将模型推送到指定的组织下。这里通常应填写你的Hugging Face用户名。

配置完成后，训练器会在训练结束时自动调用推送功能。未来的版本可能会增加在每个训练周期（epoch）结束或每隔一定训练步数（step）后自动推送的功能。

该命令执行后会返回一个特定提交的URL。你可以复制此URL并在浏览器中打开，以检查推送结果。

## 手动推送模型与分词器 🛠️

如果你没有使用Trainer API，也可以直接使用 `push_to_hub` 方法手动推送模型和分词器。

通过访问提交URL，我可以进入名为“Fine-tuned-MRPC”的仓库，并看到已添加的所有文件。

仓库中通常包含以下文件：
*   `pytorch_model.bin` 或 `tf_model.h5`：模型权重文件。
*   `config.json`：模型配置文件。
*   `training_args.bin`：训练参数。
*   `runs/`：训练日志文件（如使用TensorBoard）。
*   `vocab.json`, `tokenizer_config.json` 等：分词器所需的所有文件。
*   `README.md`：模型卡（Model Card）。

训练器会自动草拟模型卡，其中包含了在评估集上的最终结果、训练输出参数、中间训练结果以及所使用的深度学习框架信息。

如果点击编辑模型卡以查看原始内容，可以看到训练器还生成了一张元数据表。Hugging Face网站会利用这些数据为模型正确分类应用领域。你还可以通过模型卡中的链接直接访问训练指标。

## 使用与更新已推送的模型 🔄

现在模型已经成功推送，我们可以从任何地方使用 `from_pretrained` 方法加载它。只需使用Hub上的仓库标识符，模型配置和权重就会自动下载。

我们可以像使用任何Transformer模型一样使用这个模型，例如将其放入管道（Pipeline）中。由于MRPC数据集涉及句子对，任务是判断两个句子是否互为释义，我们可以在两个句子之间用分隔符连接并进行预测。

有时可能会遇到预测标签不正确的情况，例如始终预测为标签0。这通常是因为在创建模型配置时没有正确指定标签映射。

修复这个问题并更新到Hub非常简单。首先，我们可以在本地修复配置，通过设置 `id2label` 和 `label2id` 映射来添加适当的标签信息。然后，我们可以再次使用 `push_to_hub` 方法，将修复后的配置推送到仓库中。

命令会再次返回一个提交URL。由于使用的是相同的本地文件夹，这次推送会非常快。一旦完成，新的配置会自动下载（得益于缓存系统），并且预测会显示正确的标签。

你也可以直接在Hugging Face网站上的模型卡页面与模型进行互动。通过使用“Hosted inference API”部件，输入文本并点击“Compute”，等待模型加载后即可看到推理结果。你可以在此验证得到的结果是否与本地测试一致。
