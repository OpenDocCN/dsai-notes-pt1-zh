在本节课中，我们将要学习一个在自然语言处理中至关重要的预处理技术——动态填充。我们将了解它的工作原理、优势、劣势以及如何在实践中应用它。

## 概述

在将文本数据输入模型进行批量训练时，一个常见的问题是每个句子的长度各不相同。为了将它们组合成一个规整的批次张量，我们需要处理这些长度差异。本节课将详细介绍两种主要的填充策略：固定填充和动态填充，并重点解释动态填充的概念及其应用场景。

## 什么是动态填充？

在之前的课程中，我们看到了如何将不同长度的输入句子分组到同一个批次中。为了实现这一点，我们需要为所有较短的句子添加特殊的“填充标记”，直到批次内所有句子的长度变得相同。

例如，假设一个批次中有四个句子，长度分别为5、7、10和8个词。其中最长的句子是第三个，长度为10。为了使所有句子长度一致，我们需要为第一个句子添加5个填充标记，为第二个句子添加3个，为第四个句子添加2个。

## 两种填充策略

在处理文本数据集时，我们可以采用两种有价值的策略来处理长度不一致的问题。

### 策略一：固定填充

固定填充策略是指在预处理阶段，将整个数据集的所有样本都填充到同一个固定长度，这个长度通常是数据集中最长样本的长度。

**公式表示**：`填充后长度 = max(数据集中所有样本的长度)`

这样做的优点是，我们得到的每个数据批次都具有完全相同的形状，由最大序列长度决定，这简化了数据处理流程。

然而，它的缺点是显而易见的。对于由许多短句组成的批次，我们会引入大量无意义的填充标记。这不仅浪费存储空间，还会在模型前向传播和反向传播过程中引入大量不必要的计算，降低效率。

### 策略二：动态填充（批量填充）

为了避免固定填充的缺点，我们可以采用动态填充策略。这种策略不是在整个数据集层面进行填充，而是在**每个批次准备时**，仅将该批次内的样本填充到该批次中最长样本的长度。

**公式表示**：`批次内填充后长度 = max(该批次内所有样本的长度)`

这样，每个批次的形状就由该批次内最长的句子决定。其优点是显著减少了填充标记的总量，从而在CPU和GPU上能带来更快的训练速度。

但动态填充也有其局限性。由于每个批次的形状可能不同，这会在一些需要固定张量形状的硬件加速器（如TPU）上造成性能下降，或与某些需要固定批次大小的优化库不兼容。

## 实践应用

上一节我们介绍了两种策略的理论，本节中我们来看看如何在代码中实际应用它们。我们以Hugging Face `transformers` 库为例进行说明。

### 应用固定填充

我们实际上已经在处理GLUE数据集中的MRPC任务时应用过固定填充。在对数据集进行分词后，我们对整个数据集应用了带填充的分词器，将所有样本的长度统一设置为128。

```python
# 示例：固定填充
from transformers import AutoTokenizer
tokenizer = AutoTokenizer.from_pretrained(“bert-base-uncased”)
# 对整个数据集进行分词和填充
encoded_dataset = tokenizer(dataset[“sentence1”], dataset[“sentence2”], padding=“max_length”, truncation=True, max_length=128)
```
如果我们将此数据集传递给数据加载器，得到的每个批次形状都是固定的，例如 `[批次大小, 128]`。

### 应用动态填充

要应用动态填充，我们必须将填充步骤推迟到批次准备的过程中。因此，我们首先从分词函数中移除填充参数。

```python
# 示例：动态填充准备
# 分词时不进行填充，只进行截断（处理超长句）
encoded_dataset = tokenizer(dataset[“sentence1”], dataset[“sentence2”], truncation=True)
```
然后，我们使用 `DataCollatorWithPadding` 这个类来实现动态填充。`transformers` 库中的这些“数据整理器”负责在形成批次之前应用所有最终的预处理操作。

```python
# 示例：使用DataCollatorWithPadding
from transformers import DataCollatorWithPadding
data_collator = DataCollatorWithPadding(tokenizer=tokenizer)

# 在DataLoader中使用
from torch.utils.data import DataLoader
train_dataloader = DataLoader(encoded_dataset[“train”], batch_size=16, collate_fn=data_collator)
```
在这里，`DataCollatorWithPadding` 会将每个批次内的所有样本填充到该批次中的最大长度。我们将其传递给PyTorch的 `DataLoader` 后，可以观察到每个批次生成的长度各不相同，并且平均长度远低于之前固定填充的128。

## 策略选择建议

以下是选择填充策略时需要考虑的关键点：

*   **追求速度**：动态填充在CPU和GPU上几乎总是更快，因为它减少了不必要的计算。如果硬件允许，应优先应用它。
*   **硬件限制**：如果您在TPU上运行训练，或者某些框架/优化器严格要求固定的批次形状，那么您需要切换回固定填充。
*   **简单性**：固定填充实现更简单，数据管道更稳定，适合快速原型验证。
