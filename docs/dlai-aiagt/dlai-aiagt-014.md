# 014：创建工具 🛠️

在本节课中，我们将学习如何让大型语言模型（LLM）调用外部工具或函数。我们将通过一个简单的例子，逐步拆解其背后的工作原理，即使这个过程最初看起来有些神秘。

## 概述

大型语言模型（LLM）通常被训练来生成文本。那么，它是如何决定并执行“调用一个函数”这样的非文本操作的呢？本节将通过一个“获取当前时间”的例子，详细解释从LLM请求到函数执行并返回结果的完整流程。理解这个基础过程，将帮助我们更好地掌握现代LLM使用工具的语法。

## LLM调用工具的基本原理

上一节我们介绍了代理式AI的概念，本节中我们来看看LLM调用工具的具体机制。关键在于，LLM本身并不直接执行代码，而是通过生成特定格式的文本来“请求”开发者编写的代码去执行函数。

**核心流程**可以用以下伪代码描述：
```
1. 用户提问 -> LLM
2. LLM生成“函数调用请求”文本 -> 开发者代码
3. 开发者代码解析请求，执行真实函数 -> 获得结果
4. 开发者代码将结果反馈给LLM -> LLM
5. LLM结合结果生成最终回答 -> 用户
```

## 逐步解析：无参数的函数调用

让我们从一个最简单的例子开始：用户询问“现在几点了？”，我们需要LLM调用一个无参数的 `get_current_time` 函数。

首先，作为开发者，你需要完成以下步骤：

1.  **实现工具函数**：编写实际能获取时间的代码，例如 `get_current_time()`。
2.  **修改系统提示词**：告诉LLM这个函数的存在和使用方法。

以下是提供给LLM的提示词可能的样子：
```
你有一个可用的工具叫 `get_current_time`。
当你想使用它时，请输出以下文本：
FUNCTION: get_current_time
如果我看到“FUNCTION: get_current_time”，我就会为你调用该函数。
```

当用户提问“现在几点了？”时，流程如下：

1.  **LLM生成请求**：LLM理解需要获取时间，于是输出：`FUNCTION: get_current_time`。
2.  **开发者代码解析**：你写的程序会监控LLM的输出，一旦发现以 `FUNCTION:` 开头的文本，就提取出函数名 `get_current_time`。
3.  **执行函数**：你的程序调用真正的 `get_current_time()` 函数，假设返回结果是“8 AM”。
4.  **反馈结果给LLM**：你的程序将这个结果“8 AM”作为新消息输入给LLM。此时LLM的对话历史包括：用户的问题、它自己发出的函数请求、以及函数执行的结果“8 AM”。
5.  **LLM生成最终回复**：LLM综合所有信息，生成最终回答：“现在是早上8点”。

需要明确的是，**LLM从不直接调用函数**。它只是输出一个事先约定好的信号（如`FUNCTION: get_current_time`），由你的代码来接收这个信号并执行相应操作，然后将结果回传给LLM。

## 进阶：带参数的函数调用

理解了基础流程后，我们来看一个更复杂的场景。假设函数可以接受参数，例如 `get_current_time(timezone)`，用于获取指定时区的当前时间。

以下是修改后的提示词可能包含的指令：
```
你可以使用 `get_current_time` 工具来获取特定时区的当前时间。
使用时，请输出：FUNCTION: get_current_time(<时区参数>)
例如，对于新西兰时间，输出：FUNCTION: get_current_time(Pacific/Auckland)
```

当用户提问“新西兰现在几点了？”时，流程如下：

1.  **LLM生成请求**：LLM判断需要获取新西兰时间，并知道其对应的时区参数。它输出：`FUNCTION: get_current_time(Pacific/Auckland)`。
2.  **开发者代码解析与执行**：你的代码解析出函数名 `get_current_time` 和参数 `Pacific/Auckland`，然后调用真实函数 `get_current_time("Pacific/Auckland")`，假设返回“4 AM”。
3.  **反馈与最终回复**：将结果“4 AM”反馈给LLM。LLM据此生成最终回答：“新西兰现在是凌晨4点”。

## 总结与展望

本节课中我们一起学习了LLM调用工具的传统流程。我们来总结一下关键步骤：

1.  **提供工具**：实现功能函数，并通过提示词告知LLM该工具的存在及调用格式。
2.  **LLM发出请求**：LLM在需要时，按照约定格式输出一个字符串信号。
3.  **执行与反馈**：你的代码解析该信号，执行对应函数，并将结果返回给LLM。
4.  **LLM继续处理**：LLM利用返回的结果生成最终回答或决定下一步行动。

需要注意的是，上述使用全大写`FUNCTION:`作为信号的方法是一种早期、略显笨拙的实现方式。在现代，LLM已经被直接训练成使用一种更清晰、更结构化的语法来请求工具调用。
