## 为什么需要NumPy？🤔

上一节我们介绍了机器学习的基本概念，本节中我们来看看为什么需要一个专门的库来处理数据。你可能会问：我们不是已经有Python列表了吗？为什么NumPy在机器学习中如此流行且必不可少？

使用NumPy主要有两个核心原因。

### 1. 性能优势 ⚡

NumPy提供的许多功能底层实际上是用C语言编写的。C语言是一种执行速度非常快的编程语言。如果我们使用Python列表来完成即将进行的一些操作，速度会慢得多。

**公式/代码表示性能差异：**
```python
# NumPy 数组操作（快）
import numpy as np
np_array = np.array([1, 2, 3])
result = np_array * 2

# Python 列表操作（相对慢）
py_list = [1, 2, 3]
result = [x * 2 for x in py_list]
```

在机器学习中，我们需要进行大量的计算和处理。这就是为什么会使用GPU（图形处理单元）——因为有海量的计算需要完成。因此，我们需要尽可能获取所有性能优势，而NumPy正好提供了这一点。

### 2. 机器可理解的数据表示 🤖

以下是NumPy便于机器理解的几个关键点：

*   **数字是机器的语言**：机器通常只理解1和0这样的数字。NumPy可以很好地容纳数字，这对于机器来说非常容易理解。
*   **数据的数值化表示**：例如，一张图像可以用数字来表示，其中每个数字代表一个像素的颜色。
*   **属性转换**：在后续课程中，我们会尝试将不同的字段和属性转换为数字。例如，将患者是否患有心脏病从“是”或“否”转换为 **1** 或 **0**。

将数据转换为机器能够理解的形式（如数字、二进制、1和0），使我们能够让机器学习。这就是机器学习的方式——机器通过执行大量操作来学习，而使用NumPy这样的数组来存储信息，是将人类可理解的数据转换为计算机可理解数据的绝佳方式。

随着课程的深入，你会对这一点有更深的体会。现在，让我们开始学习吧。
