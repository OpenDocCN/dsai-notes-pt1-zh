## AlexNet架构回顾

上一节我们介绍了LeNet等基础架构，本节中我们来看看更复杂的AlexNet。其原始架构设计用于处理ImageNet数据集中的224x224x3图像。

以下是AlexNet的核心架构流程：

1.  **输入**：图像尺寸为 `224 x 224 x 3`（高 x 宽 x 颜色通道）。
2.  **卷积层**：依次经过多个卷积块，输出通道数变化为：`96 -> 256 -> 384 -> 384 -> 256`。
3.  **全连接层**：最后是三个大型全连接层，神经元数量为 `4096 -> 4096 -> 1000`（对应ImageNet的1000个类别）。

在本次实现中，我们对第一层进行了微调，将输出通道从96减少到64，并整体保持了相同的架构设计。需要注意的是，我们并非在ImageNet上训练，而是在图像尺寸更小的CIFAR-10数据集上进行训练。

## 针对CIFAR-10的调整

![](img/97a6f73018002494c1e22440ea60e04c_11.png)

![](img/97a6f73018002494c1e22440ea60e04c_13.png)

由于CIFAR-10的原始图像尺寸仅为32x32，直接输入AlexNet会导致特征图在网络的早期阶段就变得过小甚至消失。因此，我们必须对输入和网络进行一些调整。

![](img/97a6f73018002494c1e22440ea60e04c_14.png)

![](img/97a6f73018002494c1e22440ea60e04c_16.png)

以下是主要的调整步骤：

![](img/97a6f73018002494c1e22440ea60e04c_18.png)

![](img/97a6f73018002494c1e22440ea60e04c_19.png)

*   **图像预处理**：首先将CIFAR-10图像从32x32上采样到70x70，以确保网络能够正常处理。
*   **随机裁剪**：在训练时，从70x70的图像中随机裁剪出64x64的区域。这作为一种数据增强手段，可以提高模型对微小扰动的鲁棒性，有助于减轻过拟合。
*   **中心裁剪**：在验证和测试阶段，我们使用中心裁剪而不是随机裁剪。这是因为在评估模型或进行实际预测时，我们需要确定性的结果。
*   **标准化**：对图像的三个颜色通道进行标准化，使像素值以0为中心，范围在-1到1之间。公式如下：
    `normalized_image = (original_image - mean) / std`
    其中，对于每个通道，我们简单地将 `mean` 设为0.5，`std` 设为0.5。

## 网络结构实现

现在，让我们深入代码，看看如何在PyTorch中构建AlexNet。

网络可以分为两大模块：特征提取器（卷积部分）和分类器（全连接部分）。

![](img/97a6f73018002494c1e22440ea60e04c_21.png)

![](img/97a6f73018002494c1e22440ea60e04c_22.png)

![](img/97a6f73018002494c1e22440ea60e04c_24.png)

**特征提取器** 由多个卷积块组成。每个卷积块通常包含卷积层、激活函数（如ReLU）和可选的池化层。在AlexNet中，并非每个卷积层后都紧跟池化层。

![](img/97a6f73018002494c1e22440ea60e04c_25.png)

**分类器** 由全连接层构成。在特征提取器之后，我们添加了一个自适应平均池化层（`AdaptiveAvgPool2d`）。这个层非常关键，它可以将任意尺寸的输入特征图池化为一个固定的尺寸（例如6x6）。这样，无论输入图像的分辨率如何变化，只要特征图尺寸不小于目标尺寸，其后全连接层的输入维度都能保持一致，增加了网络的灵活性。

以下是分类器部分的核心代码逻辑示意：

```python
self.classifier = nn.Sequential(
    nn.Dropout(p=0.5), # 添加Dropout防止过拟合
    nn.Linear(256 * 6 * 6, 4096), # 全连接层1
    nn.ReLU(inplace=True),
    nn.Dropout(p=0.5), # 添加Dropout防止过拟合
    nn.Linear(4096, 4096), # 全连接层2
    nn.ReLU(inplace=True),
    nn.Linear(4096, num_classes) # 输出层
)
```

需要说明的是，原始的AlexNet论文中可能没有使用Dropout。我在这里添加了Dropout层是为了应对在CIFAR-10上训练时出现的严重过拟合问题。

## 训练过程与观察

我们使用随机梯度下降（SGD）优化器与动量，并配合学习率调度器进行训练。在CIFAR-10上完整训练200个epoch大约需要3小时。

![](img/97a6f73018002494c1e22440ea60e04c_27.png)

训练过程中观察到了一个有趣的现象——“双下降”现象。具体表现为：
1.  训练损失先下降，然后上升，接着再次下降。
2.  验证准确率也先提升，后下降，再提升。

这属于模型训练动态中的一种现象。尽管最终模型在训练集上达到了很高的准确率（约80%），但在验证集上的准确率只有约60%，这表明模型存在明显的过拟合。

![](img/97a6f73018002494c1e22440ea60e04c_29.png)

![](img/97a6f73018002494c1e22440ea60e04c_31.png)

以下是缓解过拟合的潜在方法：

*   **增加数据增强**：除了随机裁剪，还可以尝试颜色抖动、随机旋转等方法。
*   **调整Dropout率**：尝试更高的Dropout比率。
*   **使用更小的模型**：对于CIFAR-10这样的小数据集，庞大的AlexNet可能过于复杂。

![](img/97a6f73018002494c1e22440ea60e04c_33.png)

![](img/97a6f73018002494c1e22440ea60e04c_35.png)

## 结果可视化与分析

训练完成后，我们需要对结果进行分析和可视化。

![](img/97a6f73018002494c1e22440ea60e04c_37.png)

![](img/97a6f73018002494c1e22440ea60e04c_39.png)

首先，为了正确显示图像，我们需要对标准化后的数据进行“反标准化”，将其转换回原始的像素值范围。这可以通过以下公式的逆运算完成：
`original_image = normalized_image * std + mean`
