在本节课中，我们将学习一系列实用的技巧和窍门，以帮助生成对抗网络（GAN）更好地工作。这些技巧基于研究者Soumith Chintala整理的一份清单，虽然清单已不再维护，但其中大部分建议在今天依然非常有效。我们将逐一探讨这些技巧，并回顾之前的代码示例，看看哪些技巧已经被应用。

## 技巧清单详解

![](img/4ae3e73bebc6d8b163ba66d1228219a6_8.png)

![](img/4ae3e73bebc6d8b163ba66d1228219a6_10.png)

以下是Soumith Chintala总结的17条技巧。我们将逐步分析它们。

![](img/4ae3e73bebc6d8b163ba66d1228219a6_12.png)

### 1. 输入归一化
将图像归一化到 **-1到1** 的范围，并在生成器（Generator）的最后一层使用 **tanh** 激活函数。这有助于稳定训练过程。

![](img/4ae3e73bebc6d8b163ba66d1228219a6_14.png)

![](img/4ae3e73bebc6d8b163ba66d1228219a6_15.png)

![](img/4ae3e73bebc6d8b163ba66d1228219a6_17.png)

![](img/4ae3e73bebc6d8b163ba66d1228219a6_18.png)

![](img/4ae3e73bebc6d8b163ba66d1228219a6_19.png)

**代码示例**：
```python
# 将图像数据归一化到 [-1, 1]
images = (images - 0.5) * 2
# 生成器输出层使用 tanh
output = torch.tanh(self.fc_out(x))
```

![](img/4ae3e73bebc6d8b163ba66d1228219a6_21.png)

### 2. 修改损失函数
在原始GAN论文中，生成器的损失函数是 `min log(1 - D(G(z)))`。但在实践中，我们通常将其改为 **最大化 `log(D(G(z)))`**，并翻转标签。这相当于最小化 `-log(D(G(z)))`，可以缓解训练初期梯度消失的问题。

![](img/4ae3e73bebc6d8b163ba66d1228219a6_23.png)

**公式**：
原始目标：`min_G max_D V(D, G) = E[log D(x)] + E[log(1 - D(G(z)))]`
实用技巧：训练生成器时，最小化 `-log(D(G(z)))`。

![](img/4ae3e73bebc6d8b163ba66d1228219a6_25.png)

### 3. 使用球形潜在向量
从**高斯分布（正态分布）**中采样噪声向量 `z`，而不是从均匀分布中采样。这通常能产生更好的结果。

**代码示例**：
```python
# 从标准正态分布采样噪声
z = torch.randn(batch_size, latent_dim)
```

![](img/4ae3e73bebc6d8b163ba66d1228219a6_27.png)

### 4. 使用批归一化（BatchNorm）
在生成器和判别器中使用批归一化层，但要注意**不要将真实图像和生成图像的批次混合在一起**。应分别用真实图像批次和生成图像批次来训练判别器。

![](img/4ae3e73bebc6d8b163ba66d1228219a6_28.png)

### 5. 避免稀疏梯度：使用Leaky ReLU
使用 **Leaky ReLU** 代替标准的ReLU激活函数，可以防止“神经元死亡”问题，确保梯度能够有效流动。

**代码示例**：
```python
# 使用Leaky ReLU，负斜率设为0.2
layer = nn.LeakyReLU(0.2, inplace=True)
```

### 6. 下采样与上采样
对于下采样，考虑使用**平均池化（Average Pooling）**。对于上采样，**转置卷积（ConvTranspose）** 是标准做法，也可以探索 **PixelShuffle** 等技巧。

![](img/4ae3e73bebc6d8b163ba66d1228219a6_30.png)

### 7. 使用软标签和噪声标签
为标签引入一些“软性”和随机噪声，可以帮助稳定训练并防止判别器变得过于自信。
*   **真实图像标签**：使用 `0.7` 到 `1.2` 之间的随机数，而不是固定的 `1`。
*   **生成图像标签**：使用 `0.0` 到 `0.3` 之间的随机数，而不是固定的 `0`。
*   **偶尔翻转标签**：以很小的概率故意给判别器错误的标签，可以防止其过强。

![](img/4ae3e73bebc6d8b163ba66d1228219a6_32.png)

![](img/4ae3e73bebc6d8b163ba66d1228219a6_33.png)

### 8. 架构选择：使用DCGAN
当处理图像数据时，**深度卷积GAN（DCGAN）** 架构是一个强大且经过验证的基线。它使用卷积层代替全连接层，能更好地捕捉图像的空间结构。

![](img/4ae3e73bebc6d8b163ba66d1228219a6_35.png)

![](img/4ae3e73bebc6d8b163ba66d1228219a6_37.png)

![](img/4ae3e73bebc6d8b163ba66d1228219a6_39.png)

### 9. 借鉴强化学习的稳定性技巧
*   **保存检查点**：定期保存生成器和判别器的模型状态。如果训练崩溃，可以回滚到之前的稳定版本。
*   **交换旧版本**：在训练不稳定时，尝试换回旧的生成器或判别器参数。

![](img/4ae3e73bebc6d8b163ba66d1228219a6_41.png)

### 10. 优化器选择
*   **Adam优化器** 通常效果很好，可以作为默认选择。
*   有建议称，对判别器使用 **SGD**，对生成器使用 **Adam** 可能更佳，因为SGD没有动量，能让判别器反应更快。但在实践中，两者都使用Adam通常也能取得好效果，需要根据具体任务尝试。

![](img/4ae3e73bebc6d8b163ba66d1228219a6_43.png)

### 11. 监控训练失败
*   **判别器损失趋近于0**：这是一个危险信号，意味着判别器过强，生成器无法学习。需要调整（例如降低判别器的学习能力或频率）。
*   **检查梯度范数**：如果梯度变得非常大，训练可能不稳定。可以考虑使用**梯度惩罚（Gradient Penalty）** 等技术。

![](img/4ae3e73bebc6d8b163ba66d1228219a6_45.png)

![](img/4ae3e73bebc6d8b163ba66d1228219a6_46.png)

### 12. 观察损失曲线
一个健康的GAN训练过程中：
*   判别器损失**方差较低**且缓慢下降或围绕一个值波动（例如，对于二分类，随机猜测的交叉熵损失约为 `-log(0.5) ≈ 0.69`）。
*   生成器损失**不会持续单调下降**。如果生成器损失持续快速下降，可能意味着它在“欺骗”一个无用的判别器。

![](img/4ae3e73bebc6d8b163ba66d1228219a6_48.png)

### 13. 不要人为平衡损失
不要试图通过固定更新次数（例如，训练k次判别器再训练1次生成器）来机械地平衡两者。更好的方法是**根据损失值动态调整**：如果某一方的损失过大，就多训练它几次直到损失降下来。

### 14. 利用标签信息（如果可用）
如果你拥有数据的标签（例如，图像类别），训练一个**辅助分类器（Auxiliary Classifier）** 通常很有帮助。这被称为 **AC-GAN**。判别器不仅需要判断真假，还需要预测样本的类别，这为生成器提供了额外的学习信号。

### 15. 评估GAN的挑战
评估GAN生成图像的质量是一个开放的研究问题。常用的指标包括：
*   **初始分数（Inception Score, IS）**：使用在ImageNet上预训练的分类器，评估生成图像的多样性和清晰度。
*   **Fréchet初始距离（Fréchet Inception Distance, FID）**：计算真实图像和生成图像在特征空间中的分布距离，值越低越好。

![](img/4ae3e73bebc6d8b163ba66d1228219a6_50.png)

![](img/4ae3e73bebc6d8b163ba66d1228219a6_51.png)

### 16. 随时间衰减的输入噪声
在训练过程中，向判别器的输入或生成器的层中**添加逐渐衰减的噪声**，类似于去噪自编码器的思想，有时能提高鲁棒性。

### 17. 在训练和测试时都使用Dropout
通常Dropout只在训练时使用。但有建议称，在**测试时也对生成器使用Dropout**，可以增加生成样本的多样性，这是一个值得尝试的思路。

![](img/4ae3e73bebc6d8b163ba66d1228219a6_53.png)

![](img/4ae3e73bebc6d8b163ba66d1228219a6_55.png)

## 总结

![](img/4ae3e73bebc6d8b163ba66d1228219a6_57.png)

本节课我们一起学习了让GAN更稳定、更有效工作的17个实用技巧。这些技巧涵盖了数据预处理（归一化）、网络架构（使用DCGAN、Leaky ReLU）、训练策略（修改损失函数、使用软标签、动态平衡）、优化器选择以及训练监控等多个方面。虽然GAN的训练仍然充满挑战，被称为“一门艺术”，但掌握这些基础技巧能为我们提供一个坚实的起点。在接下来的课程中，我们将应用这些知识，构建更复杂的GAN模型来处理真实世界的数据集。
