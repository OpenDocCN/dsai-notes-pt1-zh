# 109：PyTorch中的模型保存与加载 💾

在本节课中，我们将要学习如何在PyTorch中保存和加载训练好的模型。掌握这项技能对于管理长期训练任务、分享模型以及在项目中断后恢复训练至关重要。

上一节我们介绍了模型训练的基本流程，本节中我们来看看如何将训练成果持久化保存，并在需要时重新加载。

## 概述

模型保存与加载的核心是处理模型的**状态字典**。状态字典是一个Python字典对象，它将模型每一层的参数（权重和偏置）映射到其对应的张量值。通过保存和加载这个字典，我们可以完整地保留或恢复模型的学习成果。

## 保存模型

![](img/98c1e29d78c0232f82d3f9c26bb9d961_5.png)

保存模型不仅包括保存模型参数，通常还包括优化器和学习率调度器的状态，以便能够无缝地继续训练。

![](img/98c1e29d78c0232f82d3f9c26bb9d961_7.png)

以下是保存模型、优化器和调度器状态的关键步骤：

![](img/98c1e29d78c0232f82d3f9c26bb9d961_9.png)

![](img/98c1e29d78c0232f82d3f9c26bb9d961_10.png)

![](img/98c1e29d78c0232f82d3f9c26bb9d961_12.png)

```python
# 保存模型的状态字典
torch.save(model.state_dict(), 'model.pt')

![](img/98c1e29d78c0232f82d3f9c26bb9d961_14.png)

![](img/98c1e29d78c0232f82d3f9c26bb9d961_16.png)

# 保存优化器的状态字典
torch.save(optimizer.state_dict(), 'optimizer.pt')

![](img/98c1e29d78c0232f82d3f9c26bb9d961_18.png)

![](img/98c1e29d78c0232f82d3f9c26bb9d961_20.png)

# 保存调度器的状态字典
torch.save(scheduler.state_dict(), 'scheduler.pt')
```

![](img/98c1e29d78c0232f82d3f9c26bb9d961_22.png)

![](img/98c1e29d78c0232f82d3f9c26bb9d961_24.png)

**注意**：文件扩展名 `.pt` 是PyTorch的常用约定，但你可以使用任何你喜欢的名称。

## 加载模型

![](img/98c1e29d78c0232f82d3f9c26bb9d961_26.png)

加载模型时，必须首先初始化一个与保存时结构完全相同的模型实例。然后，才能将保存的状态字典加载到这个实例中。

以下是加载已保存状态的核心代码：

![](img/98c1e29d78c0232f82d3f9c26bb9d961_28.png)

```python
# 1. 初始化模型结构（必须与保存的模型结构相同）
model = YourModelClass()

# 2. 加载模型的状态字典
model.load_state_dict(torch.load('model.pt'))

![](img/98c1e29d78c0232f82d3f9c26bb9d961_30.png)

# 3. （可选）加载优化器和调度器状态，以继续训练
optimizer.load_state_dict(torch.load('optimizer.pt'))
scheduler.load_state_dict(torch.load('scheduler.pt'))
```

**关键点**：在调用 `load_state_dict` 之前初始化模型是必要的，因为该方法需要知道将参数加载到哪个网络结构中。

## 工作流程示例

![](img/98c1e29d78c0232f82d3f9c26bb9d961_32.png)

假设我们训练了一个简单的神经网络用于MNIST分类，但只训练了2个周期。我们可以保存此时的中间状态。

![](img/98c1e29d78c0232f82d3f9c26bb9d961_33.png)

1.  **训练后保存**：在训练循环结束后，执行上述保存代码，生成 `model.pt` 等文件。
2.  **新建环境或中断后恢复**：在新启动的Python会话或笔记本中，首先导入模型定义，然后初始化模型、优化器和调度器。最后，加载之前保存的状态字典。
3.  **继续训练**：加载完成后，你可以直接调用训练循环，模型将从第2周期结束时的状态开始，继续训练到第10周期或更久。

![](img/98c1e29d78c0232f82d3f9c26bb9d961_35.png)

![](img/98c1e29d78c0232f82d3f9c26bb9d961_36.png)

这个过程确保了训练工作的连续性，即使计算过程被中断。

![](img/98c1e29d78c0232f82d3f9c26bb9d961_38.png)

## 附加工具：混淆矩阵

![](img/98c1e29d78c0232f82d3f9c26bb9d961_40.png)

在模型评估部分，除了常规的准确率，使用混淆矩阵可以更细致地分析模型性能。混淆矩阵是一个表格，用于展示模型预测的类别与真实类别之间的关系。

通过混淆矩阵，你可以清晰地看到模型最容易混淆哪些类别（例如，将数字“9”误判为“5”，或将“2”误判为“7”），这有助于进行更有针对性的模型改进。
