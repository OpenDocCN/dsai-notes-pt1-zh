# 009：数据可视化简介

在本节课中，我们将学习数据可视化的基本概念，了解它如何融入数据密集型工程的大背景，并特别关注其在航空航天领域的应用。我们将探讨如何通过数据讲述故事，理解高维数据集和不同数据类型，最后还会学习如何识别具有误导性的图表。

## 什么是数据可视化？

上一节我们概述了课程内容，本节中我们来看看数据可视化的核心定义。

数据可视化是将数据转化为图形或图像的过程。其目的是通过直观的视觉形式，清晰地传达数据中蕴含的信息、模式和趋势。在数据密集型工程中，可视化是理解复杂数据、进行有效沟通和做出明智决策的关键工具。

## 数据可视化与讲故事

理解了基本定义后，我们来看看数据可视化的一个高级目标：用数据讲故事。

用数据讲故事意味着不仅仅是呈现数字，而是构建一个叙事，引导观众理解数据背后的含义、因果关系或重要性。一个成功的可视化图表能够像故事一样，拥有清晰的开始（背景与问题）、发展（数据展示与分析）和结尾（结论与见解）。

## 理解数据集：维度与类型

为了有效地用数据讲故事，我们首先需要理解数据的构成。本节将介绍数据集的维度和常见的数据类型。

数据集可以按其维度（即特征或变量的数量）进行分类。我们通常处理的是高维数据集。

*   **高维数据集**：指包含多个变量或特征的数据集。例如，在航空航天领域，一架飞机的飞行数据可能同时包含**时间**、**高度**、**空速**、**发动机转速**、**燃油流量**等数十个甚至上百个参数。这可以用一个包含 *n* 个样本和 *m* 个特征的数据矩阵来表示：
    `数据矩阵 X ∈ R^(n×m)`
    其中，每一行代表一个观测样本，每一列代表一个特征变量。

数据集中通常包含不同类型的数据，主要分为以下三类：

*   **数值型数据**：可以进行数学运算的数据，如高度、速度、温度。它又可分为连续型（如温度：20.5°C）和离散型（如乘客人数：150人）。
*   **分类数据**：表示类别或组别的数据，如飞机型号（A380， B787）、发动机状态（正常， 警告， 故障）。这些数据通常用文本或代码表示。
*   **时序数据**：按时间顺序记录的数据点序列，如飞行过程中每秒记录一次的飞行参数。这是航空航天领域非常常见的数据类型。

## 警惕误导性可视化

![](img/7bc63ac7a356e49cfb41ac76040e680f_1.png)

在学会了如何创建和理解可视化之后，辨别其真伪和客观性同样重要。本节将简要介绍如何识别具有欺骗性的图表。

![](img/7bc63ac7a356e49cfb41ac76040e680f_2.png)

误导性可视化可能通过扭曲比例尺、选择性展示数据、使用不恰当的图表类型等方式，刻意夸大或掩盖某些事实，以达到特定的目的（例如，推销产品、支持某个论点）。在学习后续更深入的可视化技巧时，培养批判性审视图表的能力至关重要。
