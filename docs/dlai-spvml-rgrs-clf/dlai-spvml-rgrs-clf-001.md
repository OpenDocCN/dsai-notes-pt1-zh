在本节课中，我们将要学习监督式机器学习的基本概念，特别是回归与分类问题的区别。我们将探讨为什么线性回归不适合解决分类问题，并引出逻辑回归这一重要算法。同时，我们也会初步了解多特征线性回归（多元线性回归）的概念及其表示方法。

## 线性回归与分类问题

上一周我们学习了线性回归，它用于预测一个数值。本周我们将学习分类问题，其输出变量 `y` 只能取少数几个可能的值，而不是在一个无限的数值范围内。

事实证明，线性回归并不是解决分类问题的好算法。让我们看看原因，这将引导我们学习一种称为逻辑回归的不同算法，它是当今最流行、应用最广泛的算法之一。

## 分类问题示例

以下是分类问题的一些例子。回想一下判断电子邮件是否为垃圾邮件的例子。你希望输出的答案要么是“否”，要么是“是”。

另一个例子是判断在线金融交易是否为欺诈。我曾经从事过打击在线金融欺诈的工作，这令人异常兴奋，因为我知道有外部力量试图窃取资金，而我团队的工作就是阻止他们。所以问题是，给定一笔金融交易，你的学习算法能否判断出这笔交易是否是欺诈性的，例如这张信用卡是否被盗？

我们之前接触过的另一个例子是试图将肿瘤分类为恶性或非恶性。

## 二元分类

在每一个这些问题中，你想要预测的变量只能是两个可能值之一：“否”或“是”。这种只有两种可能输出的分类问题称为**二元分类**，其中“二元”指的是只有两个可能的类别或范畴。

在这些问题中，我将相对互换地使用术语“类”和“类别”，它们基本上意思相同。

按照惯例，我们可以用几种常见的方式来指代这两个类或类别。我们通常将类指定为“否”或“是”，或者有时等价地称为“假”或“真”，或者非常常见地使用数字 `0` 或 `1`。遵循计算机科学中的常见惯例，`0` 表示假，`1` 表示真。

我通常将使用数字 `0` 和 `1` 来表示答案 `y`，因为这将最方便地适应我们想要实现的学习算法类型。但当我们谈论时，我们也经常说“否”或“是”，或者“假”或“真”。

一项常用技术是将假或 `0` 类称为**负类**，将真或 `1` 类称为**正类**。例如，对于垃圾邮件分类，一封非垃圾邮件可能被称为**负例**，因为对于“是垃圾邮件吗？”这个问题的输出是“否”或 `0`。相反，一封垃圾邮件可能被称为**正例**，因为对于“是垃圾邮件吗？”的答案是“是”或“真”或 `1`。

通常，“负”和“正”不一定意味着坏与好或邪恶与善良，它们只是用来传达你所寻找的某种属性的**不存在**（或 `0` 或假）与**存在**（或真或 `1`）的概念，例如电子邮件中垃圾属性的不存在或存在，欺诈活动的不存在或存在，或者肿瘤中恶性肿瘤的不存在或存在。

在非垃圾邮件和垃圾邮件之间，你将哪一个称为假或 `0`，哪一个称为真或 `1` 是有点随意的。通常任何一种选择都可能有效，因此不同的工程师实际上可能会互换，将正类定义为好邮件的存在，或者将正类定义为真实金融交易或健康患者的存在。

## 构建分类算法

以下是一个用于分类肿瘤是否为恶性（1类、正类、是类）或良性（0类或负类）的训练集示例。我在横轴上绘制了肿瘤大小，在纵轴上绘制了标签 `y`。

顺便说一下，在第一周，当我们第一次讨论分类时，我们是这样在数轴上可视化的，只是现在我们将类称为 `0` 和 `1`，并将它们绘制在纵轴上。

现在，你可以在这个训练集上尝试的一件事是应用你已经知道的算法——线性回归，并尝试将直线拟合到数据。如果你这样做，也许直线看起来像这样，这就是你的 `f(x)`。

线性回归不仅预测值 `0` 和 `1`，还预测 `0` 和 `1` 之间的所有数字，甚至小于 `0` 或大于 `1` 的数字。但这里我们想要预测类别。

你可以尝试的一件事是选择一个阈值，比如 `0.5`，这样如果模型输出值低于 `0.5`，则预测 `y` 等于 `0`（非恶性）；如果模型输出的数字等于或大于 `0.5`，则预测 `y` 等于 `1`（恶性）。请注意，这个 `0.5` 的阈值在这一点与最佳拟合直线相交，因此如果你在这里画这条垂直线，左边的一切最终预测为 `y` 等于 `0`，右边的一切最终预测为 `y` 等于 `1`。

对于这个特定的数据集，看起来线性回归可以做一些合理的事情。

但现在让我们看看如果你的数据多了一个训练样本会发生什么，这个样本在右边很远的地方。让我们也扩展横轴。请注意，这个训练样本不应该真正改变你对数据点的分类方式。我们刚才画的这条垂直分界线作为截止点仍然有意义，即肿瘤小于此值应分类为 `0`，大于此值应分类为 `1`。

但是一旦你在右边添加了这个额外的训练样本，线性回归的最佳拟合线将像这样移动。如果你继续使用 `0.5` 的阈值，你现在会注意到，该点左侧的所有内容都被预测为 `0`（非恶性），而该点右侧的所有内容都被预测为 `1`（恶性）。

这不是我们想要的。因为在右边添加那个样本不应该改变我们关于如何分类恶性与良性肿瘤的任何结论。但如果你尝试用线性回归来做这件事，添加这个感觉不应该改变任何东西的样本，最终会导致我们为这个分类问题学习到一个更差的函数。

显然，当肿瘤很大时，我们希望算法将其分类为恶性。所以我们刚才看到的是，当我们在右边添加一个样本时，线性回归导致最佳拟合线移动，因此分界线（也称为决策边界）也向右移动。

## 逻辑回归的引入

你将在下一个视频中了解更多关于决策边界的信息。你还将学习一种称为逻辑回归的算法，其中结果的输出值将始终在 `0` 和 `1` 之间，并且该算法将避免我们在幻灯片上看到的这些问题。

顺便说一下，逻辑回归这个名字令人困惑的一点是，即使它包含“回归”这个词，它实际上用于分类。不要被这个名字所迷惑，这个名字是出于历史原因给出的，它实际上用于解决输出标签 `y` 为 `0` 或 `1` 的二元分类问题。

在即将到来的可选实验中，你还可以看看当你尝试使用线性回归进行分类时会发生什么。有时你很幸运，它可能会奏效。但通常它效果不佳，这就是为什么我自己不将线性回归用于分类。在可选实验中，你将看到一个尝试对两个类别进行分类的交互式图表，你可能会注意到这通常效果不是很好，不过这没关系，因为这激发了对不同模型来完成分类任务的需求。

## 多元线性回归简介

欢迎回来。本周我们将学习如何使线性回归更快、更强大，到本周末，你将完成本第一门课程的三分之二。让我们先看一个版本的线性回归，它不仅看一个特征，而且看许多不同的特征。

在原始版本的线性回归中，你有一个单一特征 `x`（房屋大小），并且能够预测 `y`（房屋价格）。所以模型是 `f_wb(x) = w * x + b`。

但现在，如果你不仅拥有房屋大小作为预测价格的特征，而且还知道卧室数量、楼层数和房屋年龄，这似乎会为你提供更多信息来预测价格。

为了引入一点新符号，我们将使用变量 `x_1`、`x_2`、`x_3` 和 `x_4` 来表示这四个特征。为简单起见，让我们再引入一点符号，我们将写成 `x_j`（或者有时简称为 `x_j`）来表示特征列表，所以这里 `j` 将从 `1` 到 `4`，因为我们有四个特征。

我将使用小写 `n` 来表示特征的总数，所以在这个例子中 `n` 等于 `4`。和以前一样，我们将使用 `x^(i)` 来表示第 `i` 个训练样本，所以这里的 `x^(i)` 实际上将是一个包含四个数字的列表，或者有时我们称之为**向量**。它包含了第 `i` 个训练样本的所有特征。

作为一个具体例子，`x^(2)` 将是第二个训练样本的特征向量，所以它将等于 `[1416, 3, 2, 40]`。从技术上讲，我将这些数字写成一行，所以有时这被称为**行向量**而不是列向量，但如果你不知道区别是什么，不用担心。对于这个目的来说，这并不重要。

为了指代第 `i` 个训练样本中的特定特征，我将写成 `x^(i)_j`。例如，`x^(2)_3` 将是第二个训练样本中第三个特征的值（即楼层数），所以那将等于 `2`。

有时为了强调这个 `x^2` 不是一个数字，而实际上是一个数字列表（即一个向量），我们会在上面画一个箭头，只是为了直观地显示它是一个向量。这里也一样。但你不必在你的符号中画这个箭头。你可以将箭头视为一个可选的标志，有时仅用于强调这是一个向量而不是一个数字。

## 多特征模型

现在我们有了多个特征，让我们看看模型会是什么样子。以前，我们是这样定义模型的，其中 `x` 是单个特征，所以是一个数字。但现在有了多个特征，我们将以不同的方式定义它。相反，模型将是：

`f_wb(x) = w1 * x1 + w2 * x2 + w3 * x3 + w4 * x4 + b`

具体来说，对于房价预测，一个可能的模型可能是，我们将房屋价格估计为 `0.1 * x1`（房屋大小）加上 `4 * x2`（卧室数量）加上 `10 * x3`（楼层数）减去 `2 * x4`（房屋年龄）加上 `80`。

让我们思考一下如何解释这些参数。如果模型试图以千美元为单位预测房屋价格，你可以认为这个 `b = 80` 表示房屋的基价可能从 80,000 美元开始，假设它没有大小、没有卧室、没有楼层且没有年龄。你可以认为这个 `0.1` 表示也许每增加一平方英尺，价格将增加 0.1 千美元或 100 美元，因为我们说每平方英尺价格增加 0.1（乘以 1,000 美元，即 100 美元）。也许每增加一个浴室，价格增加 4,000 美元；每增加一层，价格可能增加 10,000 美元；房屋每增加一年房龄，价格可能减少 2,000 美元，因为参数是 `-2`。

一般来说，如果你有 `n` 个特征，那么模型将如下所示。以下是具有 `n` 个特征的模型定义。

## 向量化表示

接下来我们要做的是引入一点符号，以更简单但等效的方式重写这个表达式。

让我们将 `w` 定义为一个列出参数 `w1`、`w2`、`w3` 一直到 `wn` 的数字列表。在数学中，这称为**向量**。有时为了指定这是一个向量（仅仅意味着一个数字列表），我将在上面画一个小箭头。你并不总是必须画这个箭头，你可以在自己的符号中画或不画，所以你可以把这个小箭头看作只是一个可选的标志，提醒我们这是一个向量。

如果你以前上过线性代数课，你可能会认出这是一个**行向量**，而不是列向量，但如果你不知道这些术语是什么意思，你不需要担心。接下来和以前一样，`b` 是一个数字，而不是向量。所以这个向量 `w` 和这个数字 `b` 一起构成了模型的参数。

让我也将 `x` 写成一个列表或向量（同样是一个行向量），列出所有特征 `x1`、`x2`、`x3` 直到 `xn`。这又是一个向量，所以我将在上面加一个小箭头来表示。

在上面的符号中，我们也可以在这里和这里添加小箭头，以表示那个 `w` 和那个 `x` 实际上是这些数字列表，实际上是这些向量。

因此，有了这个符号，模型现在可以更简洁地重写为：

`f(x) = w · x + b`

这里的点指的是线性代数中的**点积**。那么点积是什么呢？两个向量（两个数字列表）`w` 和 `x` 的点积是通过取相应的数字对 `w1` 和 `x1` 相乘，`w2` 和 `x2` 相乘，`w3` 和 `x3` 相乘，一直到 `wn` 和 `xn` 相乘，然后将所有这些乘积相加来计算的。

![](img/ce03b319579ee1e75bf07bb72d508492_1.png)

写出来，这意味着点积等于：

![](img/ce03b319579ee1e75bf07bb72d508492_3.png)

`w1*x1 + w2*x2 + w3*x3 + ... + wn*xn`

然后最后，我们在上面加回 `b`。你注意到这恰好给出了与我们上面相同的表达式。所以点积符号让你可以用更紧凑的形式、更少的字符来编写模型。

这种具有多个输入特征的线性回归模型的名称是**多元线性回归**。这与只有一个特征的**单变量回归**形成对比。顺便说一下，你可能会认为这个算法被称为“多元回归”，但该术语实际上指的是我们这里不会使用的其他东西，所以我将把这个模型称为多元线性回归。
