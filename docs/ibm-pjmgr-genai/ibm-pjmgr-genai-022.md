#  022：高效提示词的最佳实践 ✍️🤖

在本节课中，我们将学习由生成式AI专家分享的编写高效提示词的最佳实践。提示词工程是与AI模型交互、获取理想结果的关键步骤。

## 概述：提示词工程的重要性

上一节我们介绍了提示词的基本概念，本节中我们来看看专家们总结的具体实践方法。编写清晰的提示词是使用生成式AI时至关重要的一步，因为这是我们与模型交互、要求其产生预期结果的主要方式。

## 核心原则：清晰与具体

首要原则是向模型提出清晰、具体的请求。与人类交流时，我们常使用开放式问题以鼓励探索，但与AI模型协作时，我们需要更加具体和直接。

以下是编写优秀提示词的三个最通用、最突出的要点：
1.  **请求清晰**：明确表达你的需求。
2.  **提供上下文**：绝对清晰地说明目标应用场景。
3.  **列出期望**：详细列出你希望回答中包含的所有特性或要点。

## 最佳实践详解

接下来，我们将逐一探讨专家们提出的各项最佳实践。想象你正在给一个极具创造力的机器人下达指令。

### 1. 保持清晰简洁
指令应清晰简洁，例如直接告诉它“写一首朗朗上口的机器人歌曲”。避免过于冗长，给模型留出发挥创意的空间。同时，需要尝试不同的提示长度，有时一个短句足够，有时则需要更多细节。

### 2. 提供背景信息
为模型提供一些上下文或背景信息，这有助于它利用这些信息来生成更符合我们需求的回答。

### 3. 使用自然语言
使用自然语言与AI模型交互，避免使用艰深晦涩的词汇或产生歧义，以确保模型能清楚理解我们的要求。

### 4. 包含示例（Few-Shot Learning）
这是一个“小样本学习”的概念。通过提供问题-答案的示例，或关于回答结构、风格的示例，模型可以参照这些例子来相应地生成结果。

### 5. 使用专业角色与语言
使用特定的语言，并指定专家角色。如果你在谈论某个特定领域，可以说“作为一名医疗保健专家”或“作为一名产品经理”。这种特定的角色或专业领域有助于大型语言模型调整并聚焦于你想要讨论的信息。

例如，一个我常与产品管理客户使用的提示词是：
```
作为一名产品经理，请以标准的问题陈述格式，帮我撰写一个问题陈述。
```
这样，模型通常会返回一个我认为质量合格的问题陈述输出。

### 6. 设定约束与迭代
我们可以提供一些约束条件，例如限制内容范围、避免某种结构、提供指导方针、限制生成内容的长度（令牌数）或提供上下文窗口。此外，**迭代**是关键。在提出问题后，根据获得的结果评估并优化提示词。同时，需要在模型的**创造力（随机性）** 与**遵循我们提供的上下文相关性**之间找到平衡。

### 7. 利用模型记忆与主动询问
较新的模型拥有更强的记忆能力，能记住对话上下文。如果生成的回答缺少某些信息，可以要求它添加或重写。不断迭代，直到获得正确的提示词或你想要的答案。另一个好方法是，如果你不确定如何提问，可以直接询问AI。例如：
```
要获得一个流程图，向你提问的最佳方式是什么？
```
它实际上会告诉你应该使用哪种类型的提示词。

## 总结与鼓励

本节课中我们一起学习了编写高效提示词的核心原则与具体实践。记住，**熟能生巧**。尝试不同的指令表达方式，直到找到最合适的一种。不同的模型（机器人）有不同的特长，因此要根据其优势调整你的提示词。最后，赋予它上下文，就像故事或氛围一样，让它理解你的目标。
