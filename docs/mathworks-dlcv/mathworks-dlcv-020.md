# 工程与科学计算机视觉：20：在MATLAB中评估分类模型

在本节课中，我们将学习如何超越简单的准确率，使用召回率、误报率等指标以及混淆矩阵和ROC曲线来全面评估分类模型。我们还将学习如何通过成本矩阵来调整模型训练，以应对不同错误具有不同严重性的应用场景。

![](img/42b427c9b6ef0170c81130c12bdada51_2.png)

## 概述

![](img/42b427c9b6ef0170c81130c12bdada51_4.png)

准确率是评估模型在所有类别上正确标记图像能力的有效指标。然而，为了满足特定应用的需求，我们通常需要考虑其他指标，例如召回率和误报率。

![](img/42b427c9b6ef0170c81130c12bdada51_6.png)

## 应用场景：混凝土裂缝检测

假设我们正在训练一个模型来对混凝土图像进行分类，以找出需要修复的区域。在这个场景中，不同类型的错误会产生不同的后果。

![](img/42b427c9b6ef0170c81130c12bdada51_8.png)

*   错误地断言存在裂缝（误报）会导致不必要的修复工作，造成资源浪费。
*   但漏掉一个裂缝（漏报）则可能带来安全隐患。

![](img/42b427c9b6ef0170c81130c12bdada51_10.png)

因此，当并非所有错误都同等严重时，我们需要调整训练方法。由于误报的后果比漏报轻，我们的目标不应是追求最高的整体准确率，而应强调“裂缝”类别的召回率。

![](img/42b427c9b6ef0170c81130c12bdada51_12.png)

![](img/42b427c9b6ef0170c81130c12bdada51_14.png)

具体目标设定为：正确识别99%的裂缝，同时将误报率控制在10%以下。

![](img/42b427c9b6ef0170c81130c12bdada51_16.png)

![](img/42b427c9b6ef0170c81130c12bdada51_17.png)

## 使用分类学习器进行可视化评估

![](img/42b427c9b6ef0170c81130c12bdada51_19.png)

在分类学习器应用中，我们已经训练了一个CNN模型，用于将图像分类为“有裂缝”或“无裂缝”。该模型具有较高的准确率，但我们需要检查它是否满足应用目标。

![](img/42b427c9b6ef0170c81130c12bdada51_21.png)

![](img/42b427c9b6ef0170c81130c12bdada51_23.png)

### 分析混淆矩阵

![](img/42b427c9b6ef0170c81130c12bdada51_25.png)

打开混淆矩阵，检查模型识别裂缝的能力。选择显示“真正例率”和“假负例率”，以查看每个类别的召回率。

![](img/42b427c9b6ef0170c81130c12bdada51_27.png)

*   **真正例率** 等同于 **召回率**。
*   **假发现率** 是 **误报率** 的另一种说法。

![](img/42b427c9b6ef0170c81130c12bdada51_29.png)

观察“裂缝”类别的真正例率，可以发现其召回率低于99%的目标，这意味着漏掉了太多裂缝。

![](img/42b427c9b6ef0170c81130c12bdada51_31.png)

### 分析ROC曲线

![](img/42b427c9b6ef0170c81130c12bdada51_33.png)

打开ROC曲线，并将正类设置为“裂缝”。ROC曲线允许我们在单一图表中同时可视化召回率和误报率。

![](img/42b427c9b6ef0170c81130c12bdada51_35.png)

*   **曲线下面积** 是一个易于解释的指标，用于衡量模型在召回率和误报率之间的平衡。AUC分数越高，模型性能越好。

![](img/42b427c9b6ef0170c81130c12bdada51_37.png)

从混淆矩阵可知，当前模型性能未达目标。但在训练新模型之前，我们需要确认期望的结果是否可能实现。

通过检查ROC曲线，我们可以查看是否存在召回率至少为99%且误报率低于10%的点。放大并点击曲线上的点查看其数值，可以发现确实存在满足要求的点。

![](img/42b427c9b6ef0170c81130c12bdada51_39.png)

## 使用成本矩阵训练定制化模型

![](img/42b427c9b6ef0170c81130c12bdada51_41.png)

![](img/42b427c9b6ef0170c81130c12bdada51_43.png)

当前模型将所有错误视为同等严重。但在本例中，漏报裂缝比误报裂缝更严重。我们可以使用**成本矩阵**来训练一个新的CNN模型，使其倾向于获得更高的召回率。

![](img/42b427c9b6ef0170c81130c12bdada51_45.png)

成本矩阵通过改变错误分类的代价来训练模型，使其偏向于避免某些特定错误。换句话说，它告诉模型在训练期间应对每种错误施加多大的惩罚。

成本矩阵的对角线值始终为零，因为这些分类是正确的，无需惩罚。默认情况下，分类学习器为所有错误分类分配相同的惩罚。但我们可以通过增加特定错误的代价，来强调避免这些错误的重要性。

在本例中，我们将把漏报（假负例）的代价设置为误报（假正例）的5倍，使新模型对漏报的偏见是误报的5倍。此时，新模型的代价设置已显示为“自定义”。

开始训练新模型。

![](img/42b427c9b6ef0170c81130c12bdada51_47.png)

![](img/42b427c9b6ef0170c81130c12bdada51_49.png)

## 评估新模型性能

![](img/42b427c9b6ef0170c81130c12bdada51_51.png)

正如预期，新模型的整体准确率略有下降，但其裂缝类别的召回率却大幅提高。

![](img/42b427c9b6ef0170c81130c12bdada51_53.png)

![](img/42b427c9b6ef0170c81130c12bdada51_55.png)

检查新模型的ROC曲线（记得绘制“裂缝”类别），可以发现新分类器满足了召回率至少99%且误报率低于10%的要求。至此，我们已成功定制了一个满足需求的模型。

![](img/42b427c9b6ef0170c81130c12bdada51_57.png)

## 扩展到多类别分类

![](img/42b427c9b6ef0170c81130c12bdada51_59.png)

本节讨论的概念同样适用于多类别模型。例如，一个用于区分四种不同交通标志的模型。

![](img/42b427c9b6ef0170c81130c12bdada51_61.png)

![](img/42b427c9b6ef0170c81130c12bdada51_62.png)

通过查看混淆矩阵，我们可以描述每个类别之间的差异。例如，“道路封闭”标志最常被误认为是“让行”标志，这是一个严重的错误。

![](img/42b427c9b6ef0170c81130c12bdada51_64.png)

在多类别情况下，整体的假负例率是各个类别假负例率的总和。与二分类情况类似，每个类别的召回率和误报率都反映在各自的ROC曲线中。

![](img/42b427c9b6ef0170c81130c12bdada51_66.png)

## 总结

![](img/42b427c9b6ef0170c81130c12bdada51_68.png)

在本节课中，我们一起学习了如何根据应用需求选择合适的指标来评估分类模型。

*   当所有错误被视为同等严重时，**准确率**是一个很好的指标。
*   但如果你的应用强调某个特定类别（如避免漏报危险裂缝），则应利用**召回率**、**误报率**以及**成本矩阵**来训练和评估你所需的模型。

![](img/42b427c9b6ef0170c81130c12bdada51_70.png)

![](img/42b427c9b6ef0170c81130c12bdada51_71.png)

通过混淆矩阵和ROC曲线进行可视化分析，并运用成本矩阵调整训练侧重点，是构建满足实际工程与科学需求的计算机视觉模型的关键步骤。
