## 理解代码注释

在代码中，你可能会看到以 `#` 号开头的文本行，这被称为**注释**。

例如，在单元格中输入以下内容并运行：
```python
# 这是一个注释，Python会忽略它
```
按下 **Shift + Enter** 后，你会发现没有任何输出。因为计算机会完全忽略注释行。

![](img/11d124235a3d3dc466429500756797b8_22.png)

开发者使用注释来为自己或他人解释代码的用途。这与 `print` 语句（也称为打印命令）不同，`print` 语句是让计算机执行输出操作的指令。

![](img/11d124235a3d3dc466429500756797b8_24.png)

请看一个更复杂的单元格示例：
```python
# 第一行是注释
# 第二行也是注释
print("Hello, Andrew")
# 另一条注释
print("How‘s your day going?”)
# 最后一条注释
```
当你运行这个单元格时，只有两个 `print` 语句会生效并输出结果，所有注释行都会被忽略。

![](img/11d124235a3d3dc466429500756797b8_26.png)

![](img/11d124235a3d3dc466429500756797b8_28.png)

输出结果将是：
```
Hello, Andrew
How‘s your day going?
```

现在，请暂停视频，亲自点击并运行这个单元格，观察结果。如果你有灵感，也可以尝试添加自己的注释行。

## 处理代码错误（Bug）

![](img/11d124235a3d3dc466429500756797b8_30.png)

![](img/11d124235a3d3dc466429500756797b8_31.png)

![](img/11d124235a3d3dc466429500756797b8_32.png)

编程时，所有人都会经常犯错误，比如打错字，这非常正常。代码中的错误通常被称为 **Bug**。

![](img/11d124235a3d3dc466429500756797b8_33.png)

![](img/11d124235a3d3dc466429500756797b8_34.png)

当你运行有错误的代码时，有时会收到错误信息。例如，运行下面这行有问题的代码：
```python
print("Hello, Andrew)
```
你可能会看到一个关于“字符串字面量”未正确终止的错误信息。起初你可能看不懂这些信息，但没关系，你可以求助AI助手。

![](img/11d124235a3d3dc466429500756797b8_36.png)

以下是使用AI助手调试的步骤：
1.  向AI助手提问：“这段代码有什么问题？如何修复它？” 然后粘贴有错误的代码。
2.  AI助手会分析问题（例如，指出字符串的引号不匹配）并提供修正后的代码。
3.  将修正后的代码粘贴回笔记本并运行。

AI助手也能帮你解释错误信息。你可以问：“这个错误信息是什么意思？” 然后粘贴错误信息。它会给出相当不错的解释。

![](img/11d124235a3d3dc466429500756797b8_38.png)

随着课程的深入，你将学习更多术语（如Python中的**字符串**是什么），从而更好地理解这类信息。

![](img/11d124235a3d3dc466429500756797b8_40.png)
