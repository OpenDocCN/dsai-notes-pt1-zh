## 一个更复杂的待办事项列表示例

上一节我们介绍了布尔变量的概念，本节中我们来看看如何将其应用于实际场景。我们将通过一个更复杂的待办事项列表（或任务列表）来演示。

首先，我定义了一个任务列表 `tasks`：

```python
tasks = [
    {"description": "给老板写一封简短的邮件", "time_to_complete": 3},
    {"description": "为远程工作主题的演示撰写大纲", "time_to_complete": 60},
    {"description": "为电影《The 30》写一篇评论", "time_to_complete": 30},
    {"description": "为与朋友Tommy、Isabel和Daniel的豆腐橄榄炒饭准备购物清单", "time_to_complete": 5}
]
```

结构化任务列表能让我们对其进行更复杂的操作。假设现在快到午餐时间，你正准备出门，但还有一点空闲时间可以完成一个快速任务。下面，我们将使用一种新的编程模式——**条件语句**——来遍历你的任务，并决定是否执行每一项。

## 使用条件语句做出决策

首先，让我们设置 `task` 等于列表中的第一个任务（索引为0），并打印出来看看。

```python
task = tasks[0]
print(task)
```

不出所料，这打印出了你任务列表上的第一个任务。

现在，如果你正准备出门吃午饭，你可能想检查一个任务是否能在5分钟内完成。我们可以这样判断：

![](img/51349d7703ea583ad3404480ff30c761_3.png)

```python
# 检查任务完成时间是否小于等于5分钟
can_be_done_quickly = task["time_to_complete"] <= 5
print(can_be_done_quickly)  # 对于第一个任务（3分钟），这将输出 True
```

![](img/51349d7703ea583ad3404480ff30c761_5.png)

![](img/51349d7703ea583ad3404480ff30c761_6.png)

如果这是一个快速任务，我们也许可以在出门前快速完成它。对于第一个任务，结果是 `True`。

接下来，让我们看看如何编写一个程序，让它只在任务耗时5分钟或更短时才去执行。以下是关键代码：

![](img/51349d7703ea583ad3404480ff30c761_8.png)

```python
if task["time_to_complete"] <= 5:
    task_to_do = task["description"]
    # 假设我们调用一个大语言模型来处理这个任务
    response = f"正在处理任务：{task_to_do}"
    print(response)
```

在Python中，`if` 语句后面跟着一个布尔条件（本例中是 `task["time_to_complete"] <= 5`），然后是一个冒号 `:`。冒号后的下一行必须**缩进**（通常是4个空格）。这段代码的意思是：如果任务完成时间小于等于5分钟，那么就将任务描述赋值给 `task_to_do`，然后（模拟）调用大语言模型处理该任务并打印响应。

![](img/51349d7703ea583ad3404480ff30c761_10.png)

对于第一个任务，因为其完成时间是3分钟（小于等于5），条件为 `True`，所以会执行缩进块内的代码。

![](img/51349d7703ea583ad3404480ff30c761_12.png)

## 遍历列表并应用条件

我们刚刚手动检查了第一个任务。为了检查所有任务，我们可以手动重复这个过程，但更高效的方法是使用 **`for` 循环**。

以下是使用 `for` 循环遍历整个任务列表的代码：

```python
for task in tasks:
    if task["time_to_complete"] <= 5:
        task_to_do = task["description"]
        response = f"正在处理任务：{task_to_do}"
        print(response)
```

![](img/51349d7703ea583ad3404480ff30c761_16.png)

![](img/51349d7703ea583ad3404480ff30c761_17.png)

这段代码做了以下事情：
1.  `for task in tasks:` 开始一个循环，依次将 `tasks` 列表中的每个字典赋值给变量 `task`。
2.  对于每个 `task`，检查其 `"time_to_complete"` 是否 `<= 5`。
3.  如果条件为 `True`，则执行缩进块内的代码：设置 `task_to_do` 并打印响应。
4.  如果条件为 `False`，则跳过缩进块，继续检查下一个任务。

运行这段代码，你会发现它只处理了第一个（3分钟）和第四个（5分钟）任务，而跳过了需要60分钟和30分钟的任务。这正是我们想要的效果。

## 使用 `else` 处理条件为假的情况

![](img/51349d7703ea583ad3404480ff30c761_19.png)

目前，我们的程序只对能在5分钟内完成的任务采取了行动。如果我们也想对那些需要更长时间的任务做点别的事情呢？这时就可以使用 `else` 语句。

`else` 语句与 `if` 配对使用，用于指定当 `if` 条件为 `False` 时要执行的代码块。

以下是修改后的代码：

```python
for task in tasks:
    if task["time_to_complete"] <= 5:
        task_to_do = task["description"]
        response = f"正在处理任务：{task_to_do}"
        print(response)
    else:
        # 当条件为 False 时执行
        print(f"任务‘{task['description']}’将稍后完成。")
```

现在，运行程序后，对于每个任务，你都会得到一个明确的输出：要么是“正在处理任务...”，要么是“任务‘...’将稍后完成。”这使程序的逻辑更加清晰完整。

## 更高效地组织未来任务

我们目前对于稍后完成的任务只是打印一条消息。实际上，我们可以利用之前学过的列表知识，将这些未来任务更有效地组织起来。

我们可以创建一个空列表来专门存放这些“稍后完成”的任务：

![](img/51349d7703ea583ad3404480ff30c761_28.png)

![](img/51349d7703ea583ad3404480ff30c761_29.png)

```python
# 创建一个空列表来存放稍后完成的任务
tasks_for_later = []

![](img/51349d7703ea583ad3404480ff30c761_31.png)

for task in tasks:
    if task["time_to_complete"] <= 5:
        task_to_do = task["description"]
        response = f"正在处理任务：{task_to_do}"
        print(response)
    else:
        # 将需要更长时间的任务添加到“稍后完成”列表中
        tasks_for_later.append(task)

# 循环结束后，打印出所有稍后完成的任务
print("\n稍后完成的任务列表：")
for later_task in tasks_for_later:
    print(f"- {later_task['description']} (预计耗时：{later_task['time_to_complete']}分钟)")
```

![](img/51349d7703ea583ad3404480ff30c761_33.png)

![](img/51349d7703ea583ad3404480ff30c761_34.png)

在这段代码中：
*   `tasks_for_later = []` 初始化一个空列表。
*   在 `else` 块中，我们使用 `tasks_for_later.append(task)` 将当前任务字典添加到这个列表中。
*   循环结束后，我们再使用一个 `for` 循环来打印 `tasks_for_later` 列表中的所有任务及其描述。

![](img/51349d7703ea583ad3404480ff30c761_35.png)

这样，我们就成功地将需要长时间完成的任务筛选出来并存储到了一个独立的列表中，便于后续管理和处理。

## 总结

![](img/51349d7703ea583ad3404480ff30c761_37.png)

本节课中我们一起学习了如何让AI（或程序）根据数据做出决策。核心要点如下：

1.  **条件语句 (`if`/`else`)**：这是控制程序流程的基本结构。`if` 语句检查一个**布尔条件**（结果为 `True` 或 `False`）。如果条件为 `True`，则执行其缩进块内的代码；如果配合 `else`，则当条件为 `False` 时执行 `else` 块内的代码。
    *   **公式/结构**：
        ```python
        if 布尔条件:
            # 条件为 True 时执行的代码
        else:
            # 条件为 False 时执行的代码
        ```

2.  **结合循环使用**：将 `if`/`else` 语句放入 `for` 循环中，可以自动化地对数据集（如列表）中的每个元素应用决策逻辑。

3.  **组织数据**：我们可以利用列表的 `append()` 方法，根据条件将数据分类存储到不同的列表中，从而实现更复杂的数据处理逻辑。
