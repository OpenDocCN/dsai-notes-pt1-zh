在本节课中，我们将开始学习 Stim，这是一个由 Craig Gidney 编写的软件，用于模拟量子电路，特别是为量子纠错生成输出数据，以便评估特定纠错协议的性能。

上一节我们介绍了表面码及其复杂性。本节中，我们来看看如何利用 Stim 工具来高效模拟这些电路。

## Stim 简介

Stim 是一款软件，它模拟量子电路的速度比其他现有工具快 1000 倍以上。以一个极端的例子来说明：屏幕上显示的是距离为 5 的表面码电路。想象一下距离为 100 的版本，其宽度和高度是它的 20 倍。Stim 可以在 15 秒内模拟该电路（距离 100，100 轮），并以每秒 100 次的速度对该实验进行采样。这是一个非常强大且实用的工具，我们的目标是让你在今天就能开始使用它。

## Stim 的输出与应用

使用 Stim 并输入正确的代码（定义了不同距离的表面码电路和你希望使用的错误模型）后，你可以使用配套工具包绘制逻辑错误率随物理错误率和码距变化的图表，得到类似下图的结果。

![](img/80ad311f3237619d02904379213fcfdd_0.png)

图中包含大量信息。最常讨论的两个数字是：
*   **阈值错误率**：在此错误率下，增加纠错资源（增大码距）不会带来逻辑错误的抑制。这对理论研究很有用。
*   **实际目标错误率**：在实践中，你更想知道的是类似下图的数字，即达到多大的物理错误率时，每次将码距增加 2，逻辑错误就能被抑制 10 倍。

我们通常将这个抑制因子称为 **λ**。实现具有良好 λ 值的物理硬件是量子计算硬件研究的主要目标。而使用 Stim 等工具的目的，就是找出何种物理错误率、架构和纠错码可以减轻硬件的负担，以更少的物理努力实现高 λ 值。

## Stim 语法与重复码示例

首先，我们将讨论 Stim 的语法。和之前一样，我们通常避免讨论整个表面码电路，因为它太复杂。我们将只关注表面码的一部分，它看起来像重复码。这是一个我们反复使用的经典示例，因为它简单且能很好地代表表面码的工作原理。

右侧的电路正是表面码中这一小块量子比特。它只关注比特翻转错误，但包含了所有必要的操作流程。让我们看看具体细节：我们想使用 Stim 的符号来定义这个重复码电路。

从根本上说，Stim 的输入是一个文本文件，它指定了所有的门以及你希望施加在这些门上的所有错误过程。它从简单地重置所有门开始。

以下是 Stim 文件的基本结构示例：

```
# 重置量子比特
R 0 1 2 3 4

# 施加单量子比特错误（比特翻转，概率 p）
DEPOLARIZE1(p) 0 1 2 3 4

# 应用 CNOT 门
CX 0 1
CX 0 2
CX 0 3
CX 0 4

# 对双量子比特门施加噪声
DEPOLARIZE2(p) 0 1 0 2 0 3 0 4

# 对空闲量子比特施加单量子比特噪声
DEPOLARIZE1(p) 1 2 3 4

# 测量（测量前施加错误）
DEPOLARIZE1(p) 1 3
M 1 3

# 定义探测器（期望测量值为 0）
DETECTOR(1, 0) rec[-1]  # 最近一次测量（M 1）
DETECTOR(3, 0) rec[-2]  # 倒数第二次测量（M 3）
```

**符号说明**：
*   `R` 后跟数字列表表示重置这些量子比特。
*   `DEPOLARIZE1(p)` 表示以概率 `p` 施加单量子比特 depolarizing 噪声（等概率发生 X 或 Z 错误）。
*   `DEPOLARIZE2(p)` 表示以概率 `p` 施加双量子比特 depolarizing 噪声（等概率发生 15 种非平凡泡利算子乘积错误）。
*   `CX` 表示 CNOT 门。
*   `M` 表示测量。
*   `DETECTOR(x, t)` 定义了一个探测器，其坐标为 `(x, t)`，它检查括号内指定的测量结果是否与预期（通常为 0）相符。`rec[-1]` 表示最近一次的测量结果，`rec[-2]` 表示上一次的，依此类推。
*   你可以添加 `TICK` 指令来划分电路阶段，便于显示，但不影响模拟。

## 构建多轮电路与逻辑错误

现在，让我们继续构建下一轮电路。表面上它与第一轮相似，是重复的。但它没有初始化门。我们希望能够重复多轮，当然不希望每次都显式地写出这些门。Stim 有 `REPEAT` 块的概念。

此外，还有一些新内容：
*   `SHIFT_COORDS` 指令意味着未来所有探测器的坐标将按此处指定的量进行偏移。例如，`SHIFT_COORDS 0, 1` 表示空间坐标 `x` 不变，时间坐标 `t` 增加 1。这样，后续轮的探测器将与之前轮的探测器在时间上区分开。
*   在定义多轮探测器时，你需要比较当前轮的测量结果与上一轮的对应测量结果。在 Stim 中，最近的测量结果存储在数组中，使用 `rec[-1]`（最近）、`rec[-2]`（上一次）等方式访问。

最后一部分是最终轮的测量。这个电路比之前简单。然而，需要记住的关键点是，现在我们需要取最终轮中的成对测量，并与倒数第二轮中的单个测量进行比较。因此，最大的新特点是，我们现在有了一个包含三次测量的探测器来构成它。

我们还需要定义实验是否成功。如果我们将数据量子比特初始化为全零，并且实验成功，那么在经过所有校正后，最终状态应为零。这意味着任何一次校正后的测量值都应为 0。如果校正后这些测量值为 1，则说明我们的校正在某个时间点失败了，数据未被保存，这将被记录为发生了逻辑错误。我们通过 `OBSERVABLE_INCLUDE` 指令告诉 Stim 跟踪哪个可观测量（例如，最终测量中的任何一个）。

## 动手练习

现在是练习时间。建议你暂停视频，按照以下步骤操作：

1.  **组装与修改**：前往 Stim 的 GitHub 仓库，下载并开始使用一些入门笔记本。首先，将本幻灯片中的所有代码组装成一个单独的文件。添加 `TICK` 指令以便清晰划分门组。然后修改它，让最后一轮的测量与倒数第二轮在同一时间进行（因为物理上没有必要延迟）。
2.  **参数化生成**：Stim 的真正用途不是单个文件对应单个码距。你需要创建一个 Python 脚本，为任意码距 `d` 和任意错误率 `p` 输出相应的 Stim 文本文件。脚本应包含输入变量，并输出具有相应生成电路和错误模型的 Stim 文件。
3.  **使用内置函数**：完成上述练习后，你可能会问，Stim 是否有内置函数？答案是肯定的。例如，有一个函数可以生成重复码内存实验。但内置函数使用的错误模型与我们刚才构建的不同（例如，默认不在空闲量子比特上施加噪声）。我们刚才构建的练习匹配了文献中一个非常常见的错误模型，即每个量子比特（无论是否执行门操作）都受到噪声影响。在 Stim 中，你必须自己编码实现这一点。现在你已经做到了。
4.  **运行模拟与分析**：在 Stim 的入门笔记本中找到相关函数（例如第 7 节），并用你编写的函数替换它。这样，你将能够使用你的电路和错误模型生成不同码距和物理错误率的模拟输出。你会得到类似下图的曲线：在低错误率下，对于较大的码距，你将看到错误被快速抑制。

![](img/80ad311f3237619d02904379213fcfdd_4.png)

继续完成入门笔记本，并特别注意：对于更大的码，你必须将错误率控制在阈值以下才能获得改进。这是所有量子纠错形式的共同点：总存在一个阈值错误率。这个阈值越高，就越容易构建满足该约束的设备，也越容易从纠错中获益。

同样，如前所述，虽然阈值错误率是一个很好的数字，但更好的数字是每次将码距增加 2 就能获得 10 倍抑制的错误率。请花时间找出你编写的重复码以及 Stim 中表面码示例的这两个数字。一旦你有了这些数字，就为未来大量的工作奠定了基础。当你想计算算法的开销时，可以利用内存实验中错误随码距的抑制率，来计算在整个算法体积中可能累积多少错误。这将告诉你需要多强的纠错能力、需要多大的码距，进而算出算法所需的量子比特数和时间。这一切都始于为简单内存实验创建此类图表。

## 总结与预告

本节课中，我们一起学习了 Stim 工具的基本概念、语法和文件结构，并通过重复码示例实践了如何构建和模拟一个量子纠错电路。我们了解了如何定义错误模型、探测器和逻辑可观测量，并强调了通过参数化脚本生成不同规模电路的重要性。最后，我们探讨了如何利用模拟结果分析阈值错误率和逻辑错误抑制因子 λ，这是评估纠错协议性能的关键。
