让我们从 JupyterLab 的简短介绍开始，回顾一下 JupyterLab 的起源。但首先，我要感谢幻灯片左侧列出的所有为 JupyterLab 做出贡献的人。它最初是 Anaconda、Bloomberg 和 Project Jupyter 之间的合作项目，后来发展成为由许多个人共同完成的工作。Chris Colbert、Steve Sylvester 和 Afton Darien 早期投入了大量工作，Ian Rose 和我，以及 Brian Granger 等许多人也为此付出了巨大努力，此后项目吸引了更多人参与。

让我们回到 Jupyter Notebook。Jupyter Notebook 可以说已经席卷全球，成为进行数据科学和科学计算的事实标准平台。其理念是提供一个基于浏览器的交互式计算环境，让你可以交互式地探索数据。它既提供了一种交互式工作流，也提供了一种文档格式，使得在一个笔记本文档中轻松共享代码输入、代码输出以及叙述性文字成为可能。

![](img/622f19a468aaf6142bd2cc4d598e2730_27.png)

![](img/622f19a468aaf6142bd2cc4d598e2730_29.png)

Jupyter 背后的一个基本理念是，这类工作流是与语言无关的。因此，我们有许多针对不同编程语言的内核，目前数量超过 100 个。当然，所有内容都是 BSD 许可的。

这是一个 Jupyter Notebook 的例子，我想根据举手情况，大家对此都比较熟悉。

那么，我们今天处于什么位置？我们仍然拥有数百万用户，甚至比去年更多。仅在 GitHub 上就有数百万个笔记本，我相信在 GitHub 之外还有更多。我们一直在支持可重复的科学。我们喜欢的一个例子是 LIGO 合作项目，他们将所有数据以 Jupyter Notebook 的形式在线发布，你可以通过下载 Jupyter Notebook 并能够在他们的数据上运行分析，来重现上个世纪最重要的物理学发现之一。我们允许你，并且这是使用 Binder 项目的一个例子，能够下载他们的笔记本，底部有一个链接，可以下载构成近期最重要物理学发现之一基础的科学计算。

我们一直在支持开放数据新闻业。例如，BuzzFeed 将他们所有文章的支持性 Jupyter Notebook 都发布出来，包括这篇关于网球拍的文章。交互式教科书平台也基于 Jupyter Notebooks 来编写交互式教科书。当然，你们中的许多人一直在工作中使用它。

然而，我们意识到，在人们使用 Jupyter Notebook 的工作流中，他们需要的不仅仅是一个笔记本。我们有笔记本，但也需要文件浏览器、终端、文本编辑器，当然还有笔记本本身。我们看到，你当然需要比笔记本多得多的组件。因此，我们开始将这些组件放入笔记本中，我们有一系列这些组件，它们基本上是独立的工具。我们称这些为工作流的构建模块，科学计算工作流的构建模块。JupyterLab 背后的理念之一就是将所有这些东西整合到一个集成的体验中。

因此，作为 Jupyter Notebook 的下一代 Web 界面，JupyterLab 将这些工具和其他工具整合到一个集成的界面中。基于这种架构，你拥有无限的可能性，各种工具可以协同工作。

这是一张 JupyterLab 的截图。左边是一个笔记本，右边是一个 Python 文件。笔记本正在导入 Python 文件，我们可以同时处理两者。

JupyterLab 包含了许多流行词汇。它满足了所有流行词汇的要求，但更重要的是超越了这些词汇。我们努力构建了一个灵活的、可扩展的、高性能的架构。我们早期非常注重设计，试图让 UI 和 UX 正确。我们努力让扩展 JupyterLab 的 API 非常清晰，使其可以成为构建在其上的许多不同工作流和工具的基础。

今天，JupyterLab 已经开发了大约四年。我们有大约 150 位贡献者。构成 JupyterLab 的独立组件大约有 60 个。截至上次统计，我们发布了大约 2600 个版本。这个数字有些虚高，因为这 60 个组件每次发布新版本时，每个组件都会有一个版本号，所以一次 JupyterLab 发布实际上涉及这 60 个组件各自独立的发布。但到目前为止，我们总共发布了大约 2600 个版本，提交了大约 13000 次。为了对比，经典 Notebook 仓库目前的提交数大约是 11000 次。所以在提交数量上，我们与经典 Notebook 相当。

我们目前处于所谓的“测试版”，但我想你应该把它理解为用户 1.0 版本。这大约是在一月或二月发布的。我们在二月发布了我们称之为 JupyterLab 测试版的系列版本。我们说，现在它已经准备好供用户使用了。你可以将其视为用户 1.0 版本，可以在日常工作中使用。我们已经使用了很长时间。我们仍在稳定扩展 API，所以我们称之为测试版。因此，我们说它适合所有用户，但适合喜欢冒险的扩展开发者。我们今天稍后会详细讨论扩展开发。计划是在今年发布 1.0 版本，为扩展开发者提供稳定的 API。最终，经典 Notebook 将被淘汰，并由 JupyterLab 取代。
