## 概述：Python 数据可视化的现状

上一节我们介绍了本教程的主题。本节中，我们来看看 Python 数据可视化领域的现状。

![](img/6d59443486e38d65a520b93a4dcf76ab_4.png)

Python 拥有众多优秀的数据可视化库，每个库都能做出令人惊叹的图表。然而，对于大多数用户而言，他们的主要目标是完成工作，而非成为数据可视化专家。面对琳琅满目的工具选择，用户常常感到困惑，不知从何学起、从何用起。

PyViz 项目正是为了解决这一问题而生。它并非一个全新的库，而是一个协调现有工具（如 HoloViews、Bokeh、Datashader）的生态系统，旨在为用户提供一个统一、连贯且易于上手的交互式可视化体验。

## PyViz 的核心设计哲学 🧠

上一节我们提到了工具选择的困境。本节中，我们来深入探讨 PyViz 为解决这些问题而确立的核心设计原则。

PyViz 的构建基于一系列明确的目标，这些目标决定了其工具的选择和功能的侧重。

以下是指导 PyViz 发展的关键原则：
1.  **面向浏览器**：所有工具都能在网页浏览器中良好运行，便于分享和协作，避免绑定到特定的桌面 GUI 工具包。
2.  **支持完整交互性**：特别是支持图表之间的联动，使用户能够通过点击、悬停等方式深入探索数据。
3.  **面向普通 Python 用户**：假设用户是数据分析师或科学家，而非专业 Web 开发者。目标是让用户用 Python 代码就能完成工作，无需深入 Web 技术底层。
4.  **数据驱动**：工具为探索和沟通数据服务，编码是实现目的的手段，而非目的本身。
5.  **处理大规模数据**：工具应能平滑处理不断增长的数据集，避免因数据量增大而被迫更换工具。
6.  **可组合性**：利用各个领域专家构建的优秀库，将它们组合起来解决特定问题，而非重新发明轮子或构建一个庞大的一体化框架。
7.  **专注于 2D 可视化**：从 2D 图表的基础需求出发，这是大多数数据分析的起点。
8.  **分离关注点**：确保数据、业务逻辑代码和可视化呈现彼此独立。这样，为特定领域编写的代码可以运行在任何环境（如 Jupyter Notebook、服务器、超级计算机）中，而不受前端展示技术的束缚。

## 从数据到交互式图表：一个简单案例 📊

上一节我们了解了 PyViz 的设计理念。本节中，我们通过一个具体案例，看看如何应用这些理念。

我们假设你有一个 Pandas DataFrame 格式的数据集。本例使用的是美国各州按月统计的麻疹发病率数据。

### 使用 Pandas 进行静态绘图

首先，我们可以直接使用 Pandas 内置的绘图功能，它基于 Matplotlib。

```python
import pandas as pd
# 假设 df 是包含‘year’， ‘state’， ‘measles’等列的数据框
df_year = df.groupby(‘year’)[‘measles’].sum().reset_index()
df_year.plot(x=‘year’, y=‘measles’)
```

执行上述代码会生成一张静态 PNG 图片。这张图显示了麻疹病例总数随年份的变化趋势，我们可以观察到在 1963 年左右病例数急剧下降（这与麻疹疫苗的推广时间吻合）。然而，静态图片无法交互，如果我们想查看某个具体数据点的值或放大某一区域，就必须返回修改代码并重新运行。

### 使用 hvPlot 进行交互式绘图

为了解决静态图的限制，PyViz 生态系统提供了 `hvplot` 库。它的 API 设计模仿了 Pandas 的 `.plot()` 方法，但返回的是一个交互式的 Bokeh 图表对象。

```python
import hvplot.pandas
interactive_plot = df_year.hvplot(x=‘year’, y=‘measles’)
interactive_plot
```

现在生成的图表支持鼠标悬停查看数值、缩放和平移。这使用户无需修改代码就能进行初步的数据探索。

### 引入 HoloViews：数据即对象

`hvplot` 生成的实际上是一个 HoloViews 对象。HoloViews 的核心思想不是直接生成图片，而是**为你的数据添加可视化注解**，使其成为一个“可可视化”的数据对象。

这个对象包含了原始数据以及如何绘制它的声明式信息。它的强大之处在于可组合性和数据可追溯性。

```python
import holoviews as hv
# 创建一个曲线图对象
curve = hv.Curve(df_year, ‘year’， ‘measles’)
# 添加一条标记1963年的竖线和文本
vline = hv.VLine(1963)
text = hv.Text(1963， df_year[df_year[‘year’]==1963][‘measles’].values[0]*1.05， ‘Vaccine Intro’)
# 组合这些对象
composite_plot = curve * vline * text
composite_plot
```

`composite_plot` 是一个 HoloViews 对象，它是数据（曲线）与注解（竖线、文本）的叠加。你始终可以从中提取出原始的数据组件。

```python
# 获取叠加层中的曲线对象
extracted_curve = composite_plot.Curve.I
# 从曲线对象中获取原始数据
original_data = extracted_plot.data
```

### 自动探索多维数据

当你的数据具有多个维度（例如，除了年份，还有“州”这个维度）时，HoloViews 可以自动生成控件（如下拉菜单）来让你探索这些维度。

```python
# 假设 df 有 ‘year’， ‘state’， ‘measles’ 三列
# 我们按年份和州进行分组，但没有指定如何处理‘state’维度
df_state_year = df.groupby([‘year’， ‘state’])[‘measles’].sum().reset_index()
exploration_plot = df_state_year.hvplot(x=‘year’， y=‘measles’， by=‘state’)
exploration_plot
```

执行上述代码后，你会得到一个图表和一个“state”下拉菜单。选择不同的州，图表会动态显示该州的数据。这一切都是自动完成的，无需手动编写控件逻辑。

上一节我们学会了如何探索多维数据。本节中，我们来看看 PyViz 如何应对海量数据的挑战。

对于包含数亿甚至数十亿数据点的数据集，传统的将全部数据发送到浏览器进行渲染的方法行不通。PyViz 通过整合 `Datashader` 和 `Dask` 库来解决这个问题。

![](img/6d59443486e38d65a520b93a4dcf76ab_30.png)

![](img/6d59443486e38d65a520b93a4dcf76ab_32.png)

![](img/6d59443486e38d65a520b93a4dcf76ab_34.png)

**核心原理**：
1.  **数据驻留服务器端**：原始海量数据始终保留在 Python 后端（或分布式集群中），不会传输到浏览器。
2.  **按需栅格化**：当用户在浏览器中交互（如平移、缩放）时，后端根据当前视图范围，动态地对落入该范围的数据点进行聚合计算（例如，计算每个像素内的点数）。
3.  **传输图像**：将聚合结果（一个二维像素数组）发送到浏览器，由浏览器着色并显示为一张图片。
4.  **利用并行计算**：通过 `Dask` 框架，聚合计算可以并行分布在多个 CPU 核心或多个计算节点上，从而高效处理超大规模数据。
5.  **Numba 加速**：使用 `Numba` 对底层的数值计算循环进行即时编译，极大提升聚合速度。

![](img/6d59443486e38d65a520b93a4dcf76ab_36.png)

![](img/6d59443486e38d65a520b93a4dcf76ab_37.png)

![](img/6d59443486e38d65a520b93a4dcf76ab_39.png)

![](img/6d59443486e38d65a520b93a4dcf76ab_41.png)

![](img/6d59443486e38d65a520b93a4dcf76ab_43.png)

这个过程使得用户能够交互式地探索远超本地内存限制的数据集，体验上就像在操作一张本地图片，而实际上每次交互都在触发后端的一次高效计算。
