# 67：稀疏数组库介绍 🧮

在本课程中，我们将学习一个名为“sparse”的现代稀疏数组库。该库旨在提供比传统SciPy稀疏矩阵更通用、更符合NumPy习惯的N维稀疏数组操作。

---

## 概述

稀疏数组是包含大量零值（或空值）元素的数组。与存储所有元素的密集数组相比，稀疏数组仅存储非零元素及其位置信息，从而在特定场景下显著节省内存并提升计算性能。

上一节我们概述了稀疏数组的基本概念，本节中我们来看看为何需要新的稀疏数组库。

## 为何需要新的稀疏数组库？

传统的SciPy生态系统使用`scipy.sparse`处理稀疏矩阵，但它存在一些限制。它仅限于二维数组，而NumPy接口支持N维。此外，其设计视角从一开始就倾向于MATLAB风格。

新的`sparse`库旨在采用NumPy协议，确保与NDArray的最大兼容性。因此，许多适用于NumPy数组的函数可以开箱即用地应用于`sparse`数组。

## 核心存储格式

我们首先讨论如何存储稀疏数组。对于N维数组，最简单的格式是坐标格式（COO），它可以轻松推广到N维。

例如，考虑以下矩阵：
```
[[0, 0, 3, 0],
 [0, 0, 0, 0],
 [0, 0, 0, 7],
 [5, 0, 0, 0]]
```
其坐标格式存储为：
*   值 `3` 的坐标是 `(行=0, 列=2)`
*   值 `7` 的坐标是 `(行=2, 列=3)`
*   值 `5` 的坐标是 `(行=3, 列=0)`

该稀疏数组的密度为0.25，因为25%的元素是非空的。

## 逐元素操作

接下来，我们看看如何对两个稀疏数组执行逐元素操作，例如加法。由于只存储非零元素及其位置，我们无法通过循环传统数组来完成此操作。

以下是执行步骤：
1.  **识别重叠非零元素**：首先找出两个数组中位置相同的非零元素（在示例中高亮为黄色），并对它们执行操作。
2.  **处理单边非零元素**：然后处理在一个数组中为零、在另一个数组中非零的元素。
3.  **重组结果**：将上述结果组合起来，重新排序为规范顺序，即可得到结果数组。

在涉及NumPy广播的场景中，我们进行了优化。例如，当两个稀疏数组相乘且其中一个需要广播到更大维度时，算法会识别出与零相乘结果为零，从而避免不必要的存储和计算。

我们使用`__array_ufunc__`协议，使得通用函数（ufunc）可以直接在稀疏数组上工作。对于N个输入的ufunc，算法被泛化，我们使用`2^N - 1`批处理过程。

目前，排序操作耗时约占所有操作时间的30%到40%。我们正在研究从快速排序切换到基数排序，以提升速度。

一个挑战是，许多对稀疏数组的操作（例如计算余弦`cos(x)`，会将零映射为1）可能导致结果数组变得稠密。我们正在努力使这类操作更高效，但最终目标是将其设为错误提示，以避免意外的性能下降。

## 归约操作

归约操作（如求和、求积）通常具有挑战性，但我们已成功为稀疏数组实现了所有归约操作的子类。

求和很简单：沿行求和只需对所有非零元素求和。

求积则更有趣。我们计算非零元素的乘积，并额外存储一个标记来指示该行是否存在零元素。例如，某行全为零，我们甚至不存储它。对于包含零和非零元素的行，标记会记录零的存在。最终，通过与零进行额外的归约操作，我们得到正确的乘积结果。

目前支持`sum`、`prod`、`min`、`max`等归约操作。未来计划添加`argmin`和`argmax`。

我们同样使用`__array_ufunc__`协议来处理归约，因此`sum`等ufunc归约方法在遵守前述限制的条件下可以工作。通用归约通过转置和重塑为二维数组来实现，最后再反向操作恢复形状。

## 索引与切片

现在讨论如何索引稀疏数组。假设我们想提取某个特定元素（例如黄色高亮部分）。由于坐标已排序，我们可以先通过二分查找定位行，再在行内二分查找定位元素。

但在某些情况下，例如需要访问大量分散的元素时，为每个行和列执行二分查找的总次数可能超过数组本身的非零元素数量，这效率很低。

实际上，我们可以直接过滤坐标：先按行过滤，再按列过滤，然后应用简单的坐标变换，即可得到所需元素及其新坐标。

在实践中，我们采用混合方案：权衡预期二分查找次数与需过滤的元素数量，选择更快的方法执行。

目前，索引与NumPy并非完全兼容，主要区别在于**不提供视图**。即使是切片或整数索引，也会创建新数组。同时，由于需要执行二分查找或坐标过滤，速度目前不如NumPy。

## 转置、重塑与矩阵运算

转置操作很简单：只需交换坐标顺序并重新排序，使其保持规范格式。

重塑操作通过将坐标转换为线性索引，再转换回目标形状的坐标来实现。

对于矩阵或张量乘积，我们将两个数组重塑为二维，使用现有的BLAS实现（即使对于稀疏数组也存在优化实现）进行乘法，然后将结果转换回所需的维度。

## 库的现状与未来计划

我们鼓励更多人开始使用这个库，以便获得关于功能重要性和开发优先级的反馈。

*   **API稳定性**：API目前基本稳定，但尚未完全冻结。
*   **文档与测试**：拥有全面的文档，并进行了彻底的持续集成测试，因此错误应该很少。
*   **存储格式**：目前主要支持COO格式，也有DOK格式（更适合写操作）。计划实现一种能泛化这些格式的新格式，以适应更多场景和库（如scikit-learn）的需求。
*   **错误处理**：计划通过增加特性来减少“结果非稀疏”错误出现的频率。
*   **协议支持**：正大力推动通过协议支持更多NumPy API，这不仅对本库有益，也对Dask、Xarray等库有益。
*   **集成计划**：一旦库功能更完善、API更稳定，计划将其集成到SciPy中。

## 总结

本节课中我们一起学习了现代稀疏数组库`sparse`。我们探讨了其产生的背景、核心的坐标存储格式、高效的逐元素操作和归约算法、当前的索引策略以及转置重塑等操作。该库致力于提供N维、兼容NumPy协议的稀疏数组解决方案，以克服传统`scipy.sparse`的局限，并在未来成为SciPy生态系统的一部分。
