在本节课中，我们将学习 EarthSim 项目。EarthSim 不是一个独立的框架，而是一个集成了多种开源工具的实验平台和最佳实践集合，旨在为地球科学和模拟工作流提供灵活的解决方案。我们将通过一系列演示，了解如何利用 PyViz 生态系统的工具进行交互式数据探索、注释和可视化。

## 概述

EarthSim 项目始于两年前在 SciPy 会议上的一次启发。其核心目标不是创建一个新框架，而是作为实验场，探索如何将现有工具（如 HoloViews、Bokeh、GeoViews、DataShader）组合起来，以解决环境模拟中的实际问题。成熟的功能会被推回上游开源库中。项目目前主要在 Jupyter Notebook 中开展工作，但未来计划支持在无 Jupyter 后端的网站或仪表板中直接使用这些工具。

## 工具栈与协作

EarthSim 构建在 PyViz 技术栈之上。项目还与 AquaVayo 公司合作，将其商业产品中的高质量约束 Delaunay 网格生成器开源化，以支持三角形网格处理。

大部分开发工作已贡献回 DataShader、Bokeh、GeoViews 等上游项目。这是一个由 Anaconda 和 Arctic 团队协作完成的项目。

## 绘图工具与双向数据流

为了让交互更强大，我们添加了所谓的“绘图工具”。这些工具允许用户直接在地图上绘制点、路径和多边形，并且这些图形数据可以与 Python 后端进行双向同步。

以下是创建绘图工具并连接数据流的示例：

```python
# 创建空的点、路径、多边形数据源
point_source = hv.streams.PointDraw()
path_source = hv.streams.PathDraw()
polygon_source = hv.streams.PolygonDraw()

![](img/4f68067ad52d5c277ca19701bdde8f17_8.png)

# 创建地图并附加绘图工具
map_with_tools = tiles * hv.DynamicMap(lambda: hv.Points([]), streams=[point_source])
# ... 附加其他工具
```

用户在地图上的绘制操作会实时更新到 Python 的 `point_source.data`、`path_source.data` 等对象中。反之，在 Python 中修改这些数据，也会立即反映在地图视图上。这为实现复杂的交互式工作流奠定了基础。

![](img/4f68067ad52d5c277ca19701bdde8f17_10.png)

## 动态数据获取与处理

利用上述绘图工具，我们可以构建更实用的功能。例如，用户可以在地图上绘制一个边界框，然后程序自动从 USGS NED 高程服务器下载该区域的高程数据。

![](img/4f68067ad52d5c277ca19701bdde8f17_12.png)

![](img/4f68067ad52d5c277ca19701bdde8f17_14.png)

步骤如下：
1.  使用边界框工具在地图上绘制一个矩形区域。
2.  在 Python 中获取该边界框的经纬度坐标。
3.  使用 `quest` 或其他库从 USGS 服务器下载高程数据。
4.  将数据合并为单个栅格图层并加载。
5.  利用优化后的 HoloViews/GeoViews 功能进行快速、动态分辨率更新的可视化。

## 数据注释器

![](img/4f68067ad52d5c277ca19701bdde8f17_27.png)

上一节我们实现了数据的动态获取，本节中我们来看看如何为地理要素添加属性信息。

我们开发了“注释器”工具。用户可以先在地图上绘制点或多边形，然后在一个关联的表格中为这些要素添加或修改属性（如材料类型、网格大小、边界条件等）。地图上的选择和表格中的行选择是双向联动的。

以下是注释器功能的简要说明：

1.  定义注释器，指定要为点、顶点或多边形添加哪些数据列。
2.  用户在地图上绘制要素。
3.  在下方表格中编辑对应要素的属性值。
4.  属性变化会同步到地图要素，地图上选择要素也会高亮表格中的对应行。

这种功能对于准备模拟输入数据非常有用，例如为不同区域设置不同的网格分辨率或边界条件类型。

## 集成示例：驱动网格生成

![](img/4f68067ad52d5c277ca19701bdde8f17_34.png)

![](img/4f68067ad52d5c277ca19701bdde8f17_35.png)

现在，我们将绘图工具和注释器结合起来，演示一个完整的用例：根据用户绘制的海岸线和指定的点属性来生成自适应三角形网格。

![](img/4f68067ad52d5c277ca19701bdde8f17_37.png)

1.  用户绘制一个代表海岸线的多边形。
2.  在关键位置（如近岸和远海）添加几个点。
3.  通过注释器为这些点设置不同的“目标网格尺寸”属性（例如，近岸点设为 2000，远海点设为 50000）。
4.  调用网格生成器，它会根据这些点的属性值，在多边形区域内生成一个尺寸自适应的非结构化三角形网格。靠近小尺寸点的区域网格更密，远离的区域网格更疏。

这个例子展示了如何通过交互式操作来驱动复杂的后端计算流程。

## 大规模非结构化网格可视化

本节我们将探讨 EarthSim 解决的一个核心挑战：在浏览器中高效可视化大规模非结构化网格数据。

通过与 DataShader 的集成，我们实现了对三角网格的支持。这使得在浏览器中动态渲染数百万甚至上千万节点的网格成为可能。其原理是，DataShader 会根据当前的视图和缩放级别，动态地将覆盖在同一像素上的多个三角形要素进行聚合（如取平均值或最大值），从而保证交互的流畅性。

以下代码展示了如何加载一个包含 500 万个节点的哥伦比亚海域网格数据，并用 DataShader 进行渲染：

```python
# 读取网格数据（假设为 3DM 格式）
nodes, faces = read_3dm_file('columbia_mesh.3dm')
# 创建 HoloViews 三角网格对象
tri_mesh = hv.TriMesh((faces, nodes))
# 使用 DataShader 进行动态渲染
datashaded_mesh = dynspread(datashade(tri_mesh, aggregator='mean'))
```

加载数据可能需要几秒钟，但一旦加载完成，平移、缩放等操作都非常流畅。用户还可以在网格上绘制剖面线，实时提取地形变化数据。

## 图像分割与海岸线提取

除了处理矢量数据和网格，EarthSim 也集成了计算机视觉算法来处理栅格图像。一个典型的应用是利用 GrabCut 算法从卫星图像中半自动地提取海岸线。

步骤如下：
1.  加载一张河口区域的卫星图像。
2.  用户在地图上简单地绘制几笔，标记哪些区域是“背景”（水体）和“前景”（陆地）。
3.  调用 OpenCV 中的 GrabCut 算法，基于这些标记对图像进行分割。
4.  从分割结果中提取轮廓线，即可得到海岸线多边形。
5.  可以进行后处理（如过滤小多边形）以优化结果。

这种方法可以快速、粗略地从影像中提取地理特征，用于构建初始的模拟几何模型。

## 未来工作与社区参与

EarthSim 是一个持续开发的项目。未来的工作重点包括：
*   **完善文档**：提供更清晰的使用指南和示例。
*   **提升用户体验**：不断优化注释器等工具的交互细节。
*   **脱离 Notebook 环境**：使构建的组件能够直接嵌入到独立网站或仪表板中。
*   **增强 DataShader**：支持输出 WMTS 瓦片、进一步优化三角网格渲染速度、集成空间索引（如四叉树）等。
*   **探索工作流构建器**：原型化类似 ModelBuilder 的可视化工作流连接工具。
*   **支持复杂图形**：例如支持带孔洞的多边形绘制。

项目欢迎社区参与。其代码仓库不仅是开发场所，也旨在收集和展示用户利用这些工具解决实际问题的 Notebook 和案例，供他人参考和复用。
