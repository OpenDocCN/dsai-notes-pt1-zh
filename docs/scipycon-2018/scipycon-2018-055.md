# 55：SciPy 1.0 及其未来 🚀

在本节课中，我们将学习 SciPy 库的核心概念、发展历史、社区治理以及未来展望。SciPy 是科学计算领域的基础工具库，其 1.0 版本的发布标志着一个重要的里程碑。

---

## 什么是 SciPy？🔬

SciPy 是一个用于科学计算的基础工具库。它并非单一功能的库，而是集成了多种适用于不同科学领域的算法。其创始人之一 Travis 曾形容：“SciPy 是一个伪装成库的发行版。” 这意味着它最初是将围绕 NumPy 数组对象的一系列科学计算工具捆绑在一起的产物。

更简单的定义是：**SciPy = NumPy（数据结构） + 算法（科学计算）**。它提供了进行科学计算所需的基本构建模块。

---

## SciPy 包含什么？📦

SciPy 官网列出了 16 个子模块，我们可以将其归纳为五大类别。

以下是其主要组成部分：

1.  **数学基础构建模块**
    *   线性代数（包括密集和稀疏数组）
    *   傅里叶变换
    *   特殊函数
    *   这是 SciPy 中最底层的部分。

2.  **数值算法**
    *   数值优化（寻找函数最优值）
    *   积分（函数积分和常微分方程）
    *   插值（根据数据点估算中间值）
    *   聚类（数据分组的基础算法）

3.  **图算法与计算几何**
    *   图算法（例如解决最短路径问题）
    *   计算几何（例如计算 3D 物体的体积）

4.  **数据分析算法**
    *   统计（约 100 个概率分布、假设检验）
    *   信号处理（滤波等）
    *   图像处理（基础滤波、边缘检测）

5.  **实用工具**
    *   最重要的功能是支持多种科学数据格式（如 MATLAB `.mat` 文件），这在跨平台、跨语言协作中至关重要。

除了 NumPy 数组，SciPy 还有两个重要的数据结构：
*   **稀疏矩阵**：用于高效存储和处理大部分元素为零的矩阵。其数据结构只存储非零值及其位置，公式可简化为：`稀疏矩阵 = {(行索引， 列索引): 值, ...}`。
*   **KD 树**：用于快速进行空间搜索，例如查找每个点的最近邻。

## SciPy 的发展历程 📜

上一节我们介绍了 SciPy 的构成，本节中我们来看看它的发展历程。SciPy 拥有悠久的历史，其第一个版本发布于 2001 年。

*   **2001年**：首次发布。
*   **2007年**：从单一代码库中分离出多个“scikit”项目（如 scikit-learn, scikit-image）。
*   **2008年**：文档马拉松活动极大地改善了 NumPy/SciPy 的文档，使其更易于接近和使用。
*   **2010年**：开始采用六个月的发布周期。
*   **后续发展**：迁移到 GitHub、早期支持 Python 3、采用持续集成工具 Travis CI。
*   **2018年**：经过 16 年，终于发布了 **1.0 版本**。

为何耗时如此之久？原因并非技术不成熟（SciPy 早已被 NASA、英特尔、苹果等机构用于生产环境），更多是社区和流程方面的考量。

---

## 通往 1.0 之路：社区与治理 👥

发布 1.0 版本的关键因素并非全是技术升级，社区建设同样至关重要。

以下是达成 1.0 里程碑的几个核心社区要素：

1.  **明确的治理结构**：制定了“终身仁慈独裁者”（BDFL）加指导委员会的决策模式。这明确了决策者和冲突解决机制，尽管目前大多决策仍通过共识达成。
2.  **行为准则**：建立了包含报告机制的行为准则。制定过程注重广泛讨论和吸纳不同意见，以确保其被社区接受。
3.  **发展路线图**：公开路线图有助于引导开发、吸引新贡献者，并向潜在赞助方展示项目的严肃性和规划。
4.  **Windows 安装包**：解决了在 Windows 上通过 `pip install scipy` 直接安装预编译二进制包（wheel）的难题，极大地降低了用户的使用门槛。

这些努力塑造了一个健康的社区：拥有数百万用户、每版本约有 100-120 位贡献者，以及约 15 位活跃维护者。正如 Nadia Eghbal 所言：“维护者是软件开发的基石物种。” 他们是项目可持续发展的关键。

## SciPy 1.0 的代码亮点 💻

在稳固社区的基础上，SciPy 1.0 在代码层面也有重要进步，主要体现在用户友好性和性能上。

1.  **用户友好的接口**
    *   目标是将强大的算法封装成简洁易用的函数。例如，`griddata` 函数用一行代码即可完成空间数据匹配。
    *   新增的 `solve_ivp` 函数统一了多种常微分方程求解器的接口，用户可以轻松尝试并比较不同算法：
        ```python
        for method in [RK45, Radau, BDF, LSODA]:
            sol = solve_ivp(func, t_span, y0, method=method)
            # 比较和分析结果
        ```

2.  **性能提升**
    *   为大量 C/C++/Fortran 代码提供了 **Cython 绑定**，允许在 SciPy 内部直接调用，避免 Python 开销，在某些情况下可提速一两个数量级。
    *   这些绑定还能被 Numba 等即时编译器利用。例如，一个调用 `np.linalg.norm` 的函数，在添加 `@jit(nopython=True)` 装饰器后，由于利用了 SciPy 的底层 Cython 绑定，速度可提升约 5 倍。

## 未来展望与挑战 🔮

展望未来，SciPy 面临技术选型、项目边界和可持续发展等挑战。

1.  **技术栈选择**：目前 SciPy 混合使用了 Python、Cython、C、C++ 和 Fortran。未来会优先考虑**可读性、可维护性**和**可移植性**。Cython 因其易读性仍是首选，但 Numba 因其易用性（仅需一个装饰器）也备受关注。
2.  **项目边界**：SciPy 倾向于保持甚至缩小范围，专注于核心算法，确保代码质量和可维护性。新的、高层次的领域（如机器学习）更适合以独立的 scikit 项目发展。
3.  **技术路线**：
    *   支持更新的线性代数库（如放弃对 macOS Accelerate 的依赖）。
    *   改进稀疏数组（使其行为更像 NumPy 数组而非矩阵）。
    *   集成更先进的全局优化器。
4.  **可持续发展**：仅靠志愿者难以支撑项目的长远发展。NumPy 已开始尝试聘请全职开发者。**NumFOCUS** 基金会作为非营利组织，正在为 SciPy 等核心项目筹集资金，其赞助商名单中已包括多家大型科技和金融公司，这为项目的长期稳定发展带来了希望。
