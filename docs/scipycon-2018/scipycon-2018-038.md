默认情况下，当你简单地说 `%matplotlib` 时，它会打开 Jupyter 或 IPython 后端的交互式版本。这意味着 `plt.ion()` 也被触发，也就是说，一旦你发出绘图命令，图形就会立即显示。这不是通过 Python 脚本执行 Matplotlib 代码时的默认行为。

我发现这对很多人来说是一个困惑点，因为他们通常先用 Jupyter 学习 Matplotlib，然后去编写 Python 脚本，却发现行为不像 Jupyter。这是因为 Jupyter 是特例，IPython 是特例，因为它们是为交互性而设计的，而 Python 脚本和 Python REPL 不是为非交互式接口设计的。

我想让这里的感觉更像 Python 脚本和 Python REPL，而不是 Jupyter 接口。所以，当你在 Jupyter 中执行 `%matplotlib` 时，你不需要调用 `plt.show()` 来最终显示图形，而在普通的 Matplotlib 中，直到你调用 `plt.show()` 之前，什么都不会显示。

我们这里有一个图形窗口，当你调用 `plt.show()` 时会出现。它看起来像一个普通的 GUI 窗口，有一个工具栏，底部有一些东西（不同的后端可能不同）。它由轴（Axes）子图组成，可能有一个或多个轴子图。有时我们称它们为轴（axes），有时称为子图（subplots），基本上可以互换。官方名称是“axes”（复数）。我知道即使这里只有一个子图，它也是复数“axes”，这是因为它由两个或多个轴（axis）组成，比如 y 轴和 x 轴。所以一个子图由两个或多个轴组成，因此我们称它为“axes”对象，因为它是一个包含多个轴的对象。

对于一个图形窗口（GUI 窗口），你有一个图形（Figure），所以图形对象是包含一个或多个子图的东西。这非常像 MATLAB。在 MATLAB 中，你创建一个图形，然后创建一个轴（在 MATLAB 中，`subplot` 和 `axes` 是同一个东西）。Matplotlib 的创始人 John Hunter 最初编写这个库时，他是 MATLAB 的“难民”，所以 Matplotlib 的很多设计灵感来自 MATLAB，有好有坏。希望那些让我烦恼的东西不会压倒这里真正好的东西。

现在，轴（Axes）。你实际上想要进入实际绘图的核心。所有的绘图都发生在一个轴对象上。现在，你不会实际调用轴的构造函数（对于那些知道那是什么的人），我们不会让你这么做。你可以在图形上调用一个方法，或者在 pyplot 上调用一个函数来做这件事。

这是一个典型的例子，还有 `fig.add_axes()`，那是更底层的，不用担心，你很少会实际使用它。通常使用的是 `fig.add_subplot(1,1,1)`，对于来自 MATLAB 的人来说，这应该看起来很熟悉。是的，我们“复制”了这个功能。你也可以写成 `(1,1,1)`，意思是一行，一列，第一个子图。如果你写成 `(1,2,1)`，就是一行两列的第一个子图。

这里我们要添加一个子图，只是一个单子图。我们要设置 x 轴范围，设置 y 轴范围，给它一个标题，给 y 轴一个标签，给 x 轴一个标签，一大堆东西都在一个命令里，然后显示它。看，顶部有一个标题。注意，你不需要告诉它用什么字体，什么大小，你只需要给出相关的信息。你可以改变字体，改变颜色等，但里面有很多很好的默认值，大多数时候你根本不需要关心。

这是 y 轴，注意范围从 -2 到 8，如果你看命令 `ylim(-2, 8)`，就像魔法一样。x 轴，那里有标签，范围从 0.5 到 4.5，这是我指定的限制。长宽比确实改变了，因为我没有指定图形大小，所以它使用了默认值，这和我使用 `figaspect` 时长宽比不同，因为我说我想要两倍高。我们有图形大小的默认值，大概是 6.5 x 5.5 左右，我记不清确切值了。

因为我调用了 `fig = plt.figure()`，那是一个全新的图形，所以它忘记了旧的。我可能有个一般性问题：我总是困惑为什么要调用 `plt.show()`？我总是以为我会在图形或绘图上调用 `show`。你明白这个问题吗？当你真正想要一个图形窗口出现时，你调用 `plt.show()`。你可以提前创建多个图形，直到你告诉它显示之前，什么都不会显示，除非你调用了 `plt.ion()`（交互模式开启），这使得它像 MATLAB 的行为。`plt.ion()` 是全局的。

Matplotlib 会跟踪所有创建的图形，当你调用 `plt.show()` 时，它们全部出现。这引出了一个非常重要的点：当你调用 `plt.show()` 时，如果交互模式关闭（这是 Matplotlib 的默认行为），执行会停止。你不会执行任何更多的 Python 代码，它会停在那个 `plt.show()`。如果你在那之后还有任何代码，它会等待直到所有图形窗口被关闭。关闭那些图形窗口，你就销毁了那些图形。之后你可以创建更多图形，但下次你调用 `plt.show()` 时，只有新图形会出现，旧的已经没了。

快速问题：如果你调用 `fig.show()`，它是只显示当前图形还是显示所有图形？如果我没记错的话，有一个 `fig.show()` 只限于那个图形。然而，那样做有点棘手，那是更高级的用法，专门显示某个图形，它确实会搞乱一些内部的东西。所以，为了安全起见，总是用 `plt.show()`。另外，如果你碰巧使用非交互式后端，比如 `Agg` 后端，`plt.show()` 基本上是一个空操作，它不是阻塞调用，所以它会继续执行。因此，即使你在非交互式后端中，调用 `plt.show()` 也是完全安全的，比如你在无头服务器上运行代码时。

再次强调，这就是 Matplotlib 的美妙之处之一：同样的代码，`fig = plt.figure()`，`add_subplot`，设置所有那些，在 Windows、Linux、Mac 上都能工作，在你的网络服务器上能工作，在 Jupyter Notebook 上也能工作。无论你使用哪个 GUI 工具包，它都能工作。这就是 Matplotlib 的美妙之处：你们不必关心这些细节。你们只想要图形显示，只想要生成 PNG 图片（我们稍后会讲到如何制作 PNG），你们不关心所有这些细节，但你们也不希望这些细节影响你们的代码。你们不想写一堆 `if` 语句，比如“如果我用的是 QT，就这样调整大小；如果我用的是 WX，就那样做”。或者如果你在像 Jupyter Notebook 这样的 JavaScript 环境中，你根本不在乎这些。你只想绘制你的数据。我们是科学家，我们认识到你们想达到目的，不想担心那些细节。这就是 Matplotlib 的力量。
