# 059：如何为物理模拟器实现自动微分？🚀

在本教程中，我们将学习如何为物理模拟器（特别是刚体和可变形体模拟）实现自动微分（Autograd），以便将其轻松集成到神经网络训练流程中。我们将探讨标准自动微分假设在模拟场景中如何被打破，并提供一系列实用的解决方案和技巧。

---

![](img/dbb3d8805499080dc2590ce18d038c40_1.png)

## 概述：模拟与自动微分

![](img/dbb3d8805499080dc2590ce18d038c40_3.png)

![](img/dbb3d8805499080dc2590ce18d038c40_5.png)

物理模拟器可用于模拟流体、刚体、可变形体、机器人、音频、电磁学等多种现象。诸如 Newton、Warp、Tai Chi 等技术使用不同的方法构建可微分模拟器。本教程将聚焦于在 PyTorch 中构建可微分模拟器，并找到利用 Autograd 的方法。

## 1. 模拟的基本流程与目标 🎯

![](img/dbb3d8805499080dc2590ce18d038c40_7.png)

模拟可以看作一个迭代过程：从初始位置 `X` 和速度 `V` 开始，通过一系列计算步骤，得到新的 `X'` 和 `V'`，如此反复，构成完整的模拟。

![](img/dbb3d8805499080dc2590ce18d038c40_9.png)

**公式表示**：
```
初始状态: (X0, V0)
步骤1: (X1, V1) = Step(X0, V0)
步骤2: (X2, V2) = Step(X1, V1)
...
```

![](img/dbb3d8805499080dc2590ce18d038c40_11.png)

![](img/dbb3d8805499080dc2590ce18d038c40_12.png)

模拟可以基于多种参数运行，产生不同的结果。一个核心问题是：如何判断哪个模拟结果更接近真实世界（例如与真实视频对比）？或者，如何让模拟产生我们期望的行为（例如控制球体的运动）？

![](img/dbb3d8805499080dc2590ce18d038c40_14.png)

一种朴素的方法是运行大量参数组合的模拟，然后从网格中挑选最佳结果。但这存在效率低下、难以可靠用于复杂任务等问题。

![](img/dbb3d8805499080dc2590ce18d038c40_16.png)

## 2. 集成自动微分的理想愿景 🤖

更优的方案是将模拟器集成到一个可微分的计算图中。我们可以在模拟器前端添加一个神经网络（负责生成参数），在模拟器输出端连接一个损失函数，然后利用自动微分来训练这个网络。

![](img/dbb3d8805499080dc2590ce18d038c40_18.png)

**理想计算图**：
```
神经网络(参数) -> 模拟器(参数) -> 输出 -> 损失函数
```
通过 `loss.backward()`，梯度可以一路反向传播回神经网络，从而优化模拟参数。

![](img/dbb3d8805499080dc2590ce18d038c40_20.png)

![](img/dbb3d8805499080dc2590ce18d038c40_21.png)

然而，现实并非如此简单。自动微分机制内建了一些假设，而这些假设在物理模拟中常常被违反。

![](img/dbb3d8805499080dc2590ce18d038c40_23.png)

## 3. 违反的假设与应对策略 ⚙️

上一节我们介绍了集成自动微分的理想架构，本节中我们来看看具体有哪些假设被违反，以及如何解决。

![](img/dbb3d8805499080dc2590ce18d038c40_25.png)

自动微分要求所有操作是平滑且可微的。物理模拟中的某些环节打破了这一假设。

### 应对碰撞处理

![](img/dbb3d8805499080dc2590ce18d038c40_27.png)

![](img/dbb3d8805499080dc2590ce18d038c40_28.png)

如果使用流行的 IPC（增量势能接触）方法配合连续碰撞检测（CCD），可微性假设并未被直接打破，因为它使用了可微的对数障碍势能。

![](img/dbb3d8805499080dc2590ce18d038c40_30.png)

**问题**：当物体间距离 `d` 趋近于0时，对数障碍势能的梯度会变得非常大。
**解决方案**：借鉴机器学习中训练早期的经验，可以使用平滑的 `Albumentation` 函数替代对数障碍函数，以避免梯度爆炸。

### 应对库伦摩擦模型

库伦摩擦模型违反了平滑性假设，其“滑动”或“粘滞”的二元条件导致次梯度未定义。
**解决方案**：使用整个摩擦模型的平滑近似来替代原有的不连续模型。这样我们就无需单独为滑动和粘滞参数建模。

![](img/dbb3d8805499080dc2590ce18d038c40_32.png)

### 应对材料模型

![](img/dbb3d8805499080dc2590ce18d038c40_34.png)

对于常用的 `corotated` FEM 模型，可以使用稳定的 `Neo-Hookean` 材料。当雅可比行列式 `J` 趋近于 `0+` 时，可以通过钳制（clamp）雅可比行列式，使得平滑钳制后的二次体积应变能保持有界，从而保证数值稳定性。

![](img/dbb3d8805499080dc2590ce18d038c40_36.png)

![](img/dbb3d8805499080dc2590ce18d038c40_37.png)

## 4. 模拟步骤内部的挑战：隐式求解器 🔄

![](img/dbb3d8805499080dc2590ce18d038c40_39.png)

每个模拟步骤（图中箭头部分）通常包含多次牛顿迭代、共轭梯度求解和线搜索。线搜索会改变有效的映射关系，使得梯度依赖于迭代次数、容忍度等。此外，边界条件通常在求解后通过“后投影”进行钳制，这隐藏了真实的依赖关系。

在隐式积分步中，下一步状态由如下方程定义（例如后向欧拉法）：
```
F(X_{t+1}, V_{t+1}, ...) = 0
```
如果直接展开牛顿迭代或共轭梯度循环进行自动微分，Autograd 会学习到依赖于具体迭代路径的梯度，这不稳定。

![](img/dbb3d8805499080dc2590ce18d038c40_41.png)

![](img/dbb3d8805499080dc2590ce18d038c40_42.png)

![](img/dbb3d8805499080dc2590ce18d038c40_43.png)

![](img/dbb3d8805499080dc2590ce18d038c40_44.png)

**解决方案：隐式微分**
在解处，系统处于平衡态 `F(z*) = 0`。我们可以利用这个平衡条件，通过求解一个伴随系统来获取梯度，而无需展开内部求解循环。这只需要一次雅可比向量积和一次转置求解，并且可以复用前向求解时构建的矩阵和预处理器。

![](img/dbb3d8805499080dc2590ce18d038c40_46.png)

![](img/dbb3d8805499080dc2590ce18d038c40_47.png)

**优势**：
*   **内存友好**：只需在共轭梯度迭代中计算刚度矩阵向量积，无需显式构建雅可比或海森矩阵。
*   **高效**：伴随迭代的代价与前向求解的代价成比例，且复用预处理器。
*   **稳定**：满足线搜索和提前终止情况下的局部计算图稳定性要求。

## 5. 确保良好的数值条件 📈

![](img/dbb3d8805499080dc2590ce18d038c40_49.png)

为了让雅可比矩阵处于良好条件数，我们需要采取以下措施：

1.  **边界条件编码**：不应在求解后条件性地进行钳制，而应将边界条件（如固定某些点的位置）编码到线性系统本身中。
2.  **参数设置**：正确设置时间步长和前向替换次数。在非微分应用中，时间步长的影响可能仅通过视觉结果判断；但在微分应用中，必须同时关注雅可比矩阵的条件数。
3.  **梯度检查点**：模拟包含大量时间步，每个时间步又有许多子步骤，保存所有中间状态内存开销巨大。可以采用梯度检查点技术，定期保存状态，在反向传播时重新计算局部前向过程。

![](img/dbb3d8805499080dc2590ce18d038c40_51.png)

## 6. 前向与反向传播的一致性 🔄

![](img/dbb3d8805499080dc2590ce18d038c40_53.png)

自动微分另一个核心假设是：反向传播应与收敛后的前向传播相匹配。

**解决方案**：
*   **紧密收敛**：通过固定绝对/相对容差，仅在定点残差足够小时才停止求解。
*   **精确切线**：使用前向求解收敛后的精确切线进行反向传播，即梯度应基于收敛值计算，而不是基于达到收敛的路径。
*   **处理接触**：接触发生时，障碍刚度、激活距离和接触点集都会变化。我们可以在单个牛顿求解内“冻结”接触点集和障碍参数，仅在求解间更新它们。这允许我们将每个阶段视为独立的前向求解，并仅通过最终阶段进行微分。
